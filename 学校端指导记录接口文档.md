# 学校端指导记录接口文档

## 1. 获取指导记录列表

**接口地址**: `GET /api/school/guidance/list`

**请求头**:
```
Authorization: Bearer {token}
```

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| pageNum | int | 是 | 页码 |
| pageSize | int | 是 | 每页数量 |
| teacherName | string | 否 | 教师姓名（模糊查询） |
| projectName | string | 否 | 项目名称（模糊查询） |
| startTime | string | 否 | 开始时间（格式：YYYY-MM-DD） |
| endTime | string | 否 | 结束时间（格式：YYYY-MM-DD） |

**响应示例**:

### SQL查询示例

```sql
-- 1. 获取指导记录列表（带分页和筛选）
SELECT 
  pc.id,
  pc.teacher_id,
  pc.teacher_name,
  t.employee_no,
  t.college_name,
  t.title,
  pc.project_id,
  pc.project_name,
  s.student_name,
  pc.content,
  TO_CHAR(pc.create_time, 'YYYY-MM-DD HH24:MI:SS') AS create_time
FROM biz_project_comment pc
JOIN biz_teacher t ON pc.teacher_id = t.teacher_id
JOIN biz_project p ON pc.project_id = p.project_id
JOIN biz_student s ON p.user_id = s.user_id
WHERE t.school_id = #{schoolId}  -- 只查询本校教师的指导记录
  AND (#{teacherName} IS NULL OR pc.teacher_name LIKE CONCAT('%', #{teacherName}, '%'))
  AND (#{projectName} IS NULL OR pc.project_name LIKE CONCAT('%', #{projectName}, '%'))
  AND (#{startTime} IS NULL OR DATE(pc.create_time) >= #{startTime})
  AND (#{endTime} IS NULL OR DATE(pc.create_time) <= #{endTime})
ORDER BY pc.create_time DESC
LIMIT #{pageSize} OFFSET #{offset};

-- 2. 统计总数
SELECT COUNT(*) 
FROM biz_project_comment pc
JOIN biz_teacher t ON pc.teacher_id = t.teacher_id
WHERE t.school_id = #{schoolId}
  AND (#{teacherName} IS NULL OR pc.teacher_name LIKE CONCAT('%', #{teacherName}, '%'))
  AND (#{projectName} IS NULL OR pc.project_name LIKE CONCAT('%', #{projectName}, '%'))
  AND (#{startTime} IS NULL OR DATE(pc.create_time) >= #{startTime})
  AND (#{endTime} IS NULL OR DATE(pc.create_time) <= #{endTime});

-- 3. 获取指导记录详情
SELECT 
  pc.id,
  pc.teacher_id,
  pc.teacher_name,
  t.employee_no,
  t.college_name,
  t.title,
  pc.project_id,
  pc.project_name,

  s.student_no,

JOIN biz_teacher t ON pc.teacher_id = t.teacher_id
JOIN biz_project p ON pc.project_id = p.project_id
```

### Java Controller示例

```java
@RestController
@RequestMapping("/api/school/guidance")
public class SchoolGuidanceController {

    @Autowired
    private SchoolGuidanceService guidanceService;

    @GetMapping("/list")
    public Result<PageResult<GuidanceVO>> getGuidanceList(GuidanceQuery query) {
        // 从token获取当前学校ID
        String schoolId = SecurityUtils.getSchoolId();
        query.setSchoolId(schoolId);
        
        PageResult<GuidanceVO> result = guidanceService.getGuidanceList(query);
        return Result.success(result);
    }

    @GetMapping("/detail/{id}")
    public Result<GuidanceDetailVO> getGuidanceDetail(@PathVariable String id) {
        String schoolId = SecurityUtils.getSchoolId();
        GuidanceDetailVO detail = guidanceService.getGuidanceDetail(id, schoolId);
        
        if (detail == null) {
            return Result.error("指导记录不存在或无权访问");
        }
        
        return Result.success(detail);
    }
}
```

### Service实现示例

```java
@Service
public class SchoolGuidanceServiceImpl implements SchoolGuidanceService {
    
    @Autowired
    private BizProjectCommentMapper commentMapper;
    
    @Override
    public PageResult<GuidanceVO> getGuidanceList(GuidanceQuery query) {
        // 计算分页偏移量
        int offset = (query.getPageNum() - 1) * query.getPageSize();
        query.setOffset(offset);
        
        // 查询列表
        List<GuidanceVO> list = commentMapper.selectGuidanceList(query);
        
        // 查询总数
        int total = commentMapper.countGuidanceList(query);
        
        return new PageResult<>(list, total);
    }
    
    @Override
    public GuidanceDetailVO getGuidanceDetail(String id, String schoolId) {
        return commentMapper.selectGuidanceDetail(id, schoolId);
    }
}
```

## 注意事项

1. **权限控制**: 学校端只能查看本校教师的指导记录，需要在SQL中通过 `teacher.school_id` 进行过滤
2. **数据隔离**: 从token中获取当前登录用户所属的学校ID，确保数据隔离
3. **关联查询**: 需要关联 `biz_project_comment`、`biz_teacher`、`biz_project`、`biz_student` 四张表
4. **日期筛选**: 支持按指导时间范围筛选，使用 `DATE()` 函数进行日期比较
5. **内容显示**: 指导内容可能较长，列表页使用 `show-overflow-tooltip` 显示省略号，详情页完整展示
6. **排序规则**: 按指导时间倒序排列，最新的指导记录在前
7. **数据来源**: 
   - 指导记录来自 `biz_project_comment` 表
   - 教师信息来自 `biz_teacher` 表
   - 项目信息来自 `biz_project` 表
   - 学生信息来自 `biz_student` 表（通过 `project.user_id` 关联）
8. **字段冗余**: `biz_project_comment` 表中已经冗余了 `teacher_name` 和 `project_name`，可以直接使用，但为了获取更多信息（如工号、学院、职称等），仍需要关联查询
9. **性能优化**: 
   - 在 `biz_project_comment` 表的 `teacher_id` 和 `project_id` 字段上已有索引
   - 如果查询量大，建议在 `create_time` 字段上添加索引
10. **业务说明**: 
    - 教师在学生端查看创业项目时可以发表指导意见
    - 指导记录会自动记录教师姓名和项目名称（冗余字段）
    - 学校端可以查看所有本校教师的指导记录，用于统计和管理

## 扩展功能建议

如果后续需要扩展功能，可以考虑：

1. **导出功能**: 导出指导记录为Excel文件
2. **统计分析**: 统计每个教师的指导次数、指导的项目数量等
3. **评价功能**: 学生对教师指导进行评价和反馈
4. **通知功能**: 教师发表指导意见后，通知学生查看
5. **筛选优化**: 增加按学院、职称等维度筛选
