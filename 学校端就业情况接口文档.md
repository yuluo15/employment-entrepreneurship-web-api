# 学校端就业情况接口文档

## 概述

本接口用于管理学生就业信息。就业状态只有两种：
- **0**: 待就业
- **1**: 已就业（企业发offer时自动设置）

**说明**:
- 创业项目通过 `biz_project` 表单独管理，不影响就业状态
- 学生可以同时就业和参与创业项目
- 就业状态由企业端在发放offer时自动设置

---

## 1. 获取就业列表

**接口地址**: `GET /api/school/employment/list`

**请求头**:
```
Authorization: Bearer {token}
```

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| pageNum | int | 是 | 页码 |
| pageSize | int | 是 | 每页数量 |
| studentName | string | 否 | 学生姓名（模糊查询） |
| studentNo | string | 否 | 学号（模糊查询） |
| employmentStatus | string | 否 | 就业状态（0=待就业, 1=已就业） |
| graduationYear | int | 否 | 毕业年份 |

**响应示例**:
```json
{
  "message": "success",
  "data": {
    "total": 100,
    "data": [
      {
        "studentId": "stu123",
        "studentName": "张三",
        "studentNo": "2021001",
        "collegeName": "计算机学院",
        "majorName": "软件工程",
        "graduationYear": 2025,
        "employmentStatus": "1",
        "companyName": "阿里巴巴集团",
        "position": "Java开发工程师",
        "salary": "15-20K",
        "employmentDate": "2025-07-01",
        "workLocation": "杭州",
        "remark": ""
      }
    ]
  }
}
```

**字段说明**:
- `employmentStatus`: 0=待就业, 1=已就业
- 已就业字段：`companyName`, `position`, `salary`, `employmentDate`, `workLocation`
- **就业状态设置**: 企业端发放offer时自动设置为已就业
- **创业项目**: 通过 `biz_project` 表单独统计，不影响就业状态

## 3. 更新就业信息

**接口地址**: `PUT /api/school/employment/update`

**请求头**:
```
Authorization: Bearer {token}
Content-Type: application/json
```

**请求体**:
```json
{
  "studentId": "stu123",
  "employmentStatus": "1",
  "companyName": "阿里巴巴集团",
  "position": "Java开发工程师",
  "salary": "15-20K",
  "employmentDate": "2025-07-01",
  "workLocation": "杭州",
  "remark": "通过校招录用"
}
```

**字段说明**:
- `studentId`: 必填，学生ID
- `employmentStatus`: 必填，就业状态（0=待就业, 1=已就业）
- 根据不同的就业状态，需要填写对应的字段：
  - **已就业(1)**: `companyName`(必填), `position`(必填), `salary`, `employmentDate`, `workLocation`
  - **待就业(0)**: 无需额外字段

**说明**:
- 通常情况下，就业状态由企业端在发放offer时自动设置
- 学校端也可以手动修改学生的就业信息

**响应示例**:
```json
{
  "message": "success"
}
```

**接口地址**: `GET /api/school/employment/stats`

**请求头**:
```
Authorization: Bearer {token}
```

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| studentName | string | 否 | 学生姓名（模糊查询） |
| studentNo | string | 否 | 学号（模糊查询） |
| employmentStatus | string | 否 | 就业状态（0=待就业, 1=已就业） |
| graduationYear | int | 否 | 毕业年份 |

**响应示例**:
```json
{
  "message": "success",
  "data": {
    "totalCount": 500,
    "employedCount": 450,
    "employmentRate": 90.0
  }
}
```

**字段说明**:
- `totalCount`: 总人数
- `employedCount`: 已就业人数（employmentStatus=1）
- `employmentRate`: 就业率（已就业/总人数 * 100）

**说明**:
- 就业率 = 已就业人数 / 总人数 × 100
- 创业项目不计入就业统计，通过 `biz_project` 表单独统计
- 学生可以同时就业和参与创业项目

## 后端实现参考

### 数据库表设计

**方案一：使用 `biz_student` 表的 `employment_status` 字段（推荐）**

由于就业状态只有两种（待就业/已就业），且就业信息可以从 `biz_job_delivery` 表关联获取，建议直接使用 `biz_student` 表的 `employment_status` 字段，无需新建表。

```sql
-- biz_student 表已有字段
-- employment_status VARCHAR(255)  -- 0=待就业, 1=已就业

-- 就业信息可以从投递记录表关联获取
-- biz_job_delivery 表包含：job_id, company_id, status (OFFER表示已录用)
```

**方案二：新建就业信息表（如需存储额外信息）**

如果需要存储更详细的就业信息（如薪资、工作地点等），可以新建表：

```sql
-- 就业信息表（可选）
CREATE TABLE biz_student_employment (
  id VARCHAR(255) PRIMARY KEY,
  student_id VARCHAR(255) NOT NULL,
  employment_status VARCHAR(10) NOT NULL DEFAULT '0',
  
  -- 已就业字段
  company_id VARCHAR(255),  -- 关联企业ID
  company_name VARCHAR(255),
  job_id VARCHAR(255),  -- 关联职位ID
  position VARCHAR(255),
  salary VARCHAR(100),
  employment_date DATE,
  work_location VARCHAR(255),
  
  -- 通用字段
  remark TEXT,
  create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  update_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  
  CONSTRAINT uk_student_id UNIQUE (student_id),
  CONSTRAINT fk_student FOREIGN KEY (student_id) REFERENCES biz_student(student_id)
);

CREATE INDEX idx_employment_status ON biz_student_employment(employment_status);
CREATE INDEX idx_student_id ON biz_student_employment(student_id);

COMMENT ON TABLE biz_student_employment IS '学生就业信息表';
COMMENT ON COLUMN biz_student_employment.employment_status IS '就业状态：0=待就业, 1=已就业';
COMMENT ON COLUMN biz_student_employment.company_name IS '就业单位';
```

**推荐方案**: 使用方案一，直接使用 `biz_student.employment_status` 字段，就业详情从 `biz_job_delivery` 表关联获取。这样数据更一致，避免冗余。

### SQL查询示例

```sql
-- 1. 获取就业列表（方案一：直接使用 biz_student 表）
SELECT 
  s.student_id,
  s.student_name,
  s.student_no,
  s.college_name,
  s.major_name,
  s.graduation_year,
  COALESCE(s.employment_status, '0') AS employment_status,
  -- 从投递记录关联获取就业信息
  c.name AS company_name,
  j.job_name AS position,
  j.salary_range AS salary,
  TO_CHAR(d.update_time, 'YYYY-MM-DD') AS employment_date,
  j.city AS work_location
FROM biz_student s
LEFT JOIN biz_job_delivery d ON s.student_id = d.student_id AND d.status = 'OFFER'
LEFT JOIN biz_job j ON d.job_id = j.job_id
LEFT JOIN sys_company c ON d.company_id = c.id
WHERE s.school_id = #{schoolId}
  AND (#{studentName} IS NULL OR s.student_name LIKE CONCAT('%', #{studentName}, '%'))
  AND (#{studentNo} IS NULL OR s.student_no LIKE CONCAT('%', #{studentNo}, '%'))
  AND (#{employmentStatus} IS NULL OR COALESCE(s.employment_status, '0') = #{employmentStatus})
  AND (#{graduationYear} IS NULL OR s.graduation_year = #{graduationYear})
ORDER BY s.graduation_year DESC, s.student_no ASC
LIMIT #{pageSize} OFFSET #{offset};

-- 2. 获取就业详情
SELECT 
  s.student_id,
  s.student_name,
  s.student_no,
  s.college_name,
  s.major_name,
  s.graduation_year,
  COALESCE(s.employment_status, '0') AS employment_status,
  c.name AS company_name,
  j.job_name AS position,
  j.salary_range AS salary,
  TO_CHAR(d.update_time, 'YYYY-MM-DD') AS employment_date,
  j.city AS work_location,
  TO_CHAR(s.update_time, 'YYYY-MM-DD HH24:MI:SS') AS update_time
FROM biz_student s
LEFT JOIN biz_job_delivery d ON s.student_id = d.student_id AND d.status = 'OFFER'
LEFT JOIN biz_job j ON d.job_id = j.job_id
LEFT JOIN sys_company c ON d.company_id = c.id
WHERE s.student_id = #{studentId}
  AND s.school_id = #{schoolId};

-- 3. 更新就业状态（通常由企业端在发放offer时自动调用）
UPDATE biz_student 
SET employment_status = #{employmentStatus}, 
    update_time = CURRENT_TIMESTAMP
WHERE student_id = #{studentId};

-- 4. 获取就业统计
SELECT 
  COUNT(*) AS total_count,
  SUM(CASE WHEN COALESCE(s.employment_status, '0') = '1' THEN 1 ELSE 0 END) AS employed_count,
  ROUND(
    (SUM(CASE WHEN COALESCE(s.employment_status, '0') = '1' THEN 1 ELSE 0 END)::NUMERIC / 
     NULLIF(COUNT(*), 0)) * 100, 
    2
  ) AS employment_rate
FROM biz_student s
WHERE s.school_id = #{schoolId}
  AND (#{studentName} IS NULL OR s.student_name LIKE CONCAT('%', #{studentName}, '%'))
  AND (#{studentNo} IS NULL OR s.student_no LIKE CONCAT('%', #{studentNo}, '%'))
  AND (#{employmentStatus} IS NULL OR COALESCE(s.employment_status, '0') = #{employmentStatus})
  AND (#{graduationYear} IS NULL OR s.graduation_year = #{graduationYear});

-- 5. 统计创业项目数量（单独统计，不计入就业率）
SELECT COUNT(*) AS project_count
FROM biz_project
WHERE school_id = #{schoolId}
  AND status IN ('1', '2');  -- 1=审核通过, 2=已完成

-- 6. 企业端发放offer时自动设置就业状态
-- 当企业端将投递状态改为 OFFER 时，同时更新学生就业状态
UPDATE biz_student 
SET employment_status = '1', 
    update_time = CURRENT_TIMESTAMP
WHERE student_id = (
  SELECT student_id FROM biz_job_delivery WHERE id = #{deliveryId}
);
```

### Java Controller示例

```java
@RestController
@RequestMapping("/api/school/employment")
public class SchoolEmploymentController {

    @Autowired
    private SchoolEmploymentService employmentService;

    @GetMapping("/list")
    public Result<PageResult<EmploymentVO>> getEmploymentList(EmploymentQuery query) {
        String schoolId = SecurityUtils.getSchoolId();
        query.setSchoolId(schoolId);
        
        PageResult<EmploymentVO> result = employmentService.getEmploymentList(query);
        return Result.success(result);
    }

    @GetMapping("/detail/{studentId}")
    public Result<EmploymentVO> getEmploymentDetail(@PathVariable String studentId) {
        String schoolId = SecurityUtils.getSchoolId();
        EmploymentVO employment = employmentService.getEmploymentDetail(studentId, schoolId);
        
        if (employment == null) {
            return Result.error("学生不存在或无权访问");
        }
        
        return Result.success(employment);
    }

    @PutMapping("/update")
    public Result<Void> updateEmploymentInfo(@RequestBody EmploymentUpdateDTO dto) {
        String schoolId = SecurityUtils.getSchoolId();
        
        // 验证学生是否属于当前学校
        if (!employmentService.validateStudent(dto.getStudentId(), schoolId)) {
            return Result.error("无权操作该学生信息");
        }
        
        employmentService.updateEmploymentInfo(dto);
        return Result.success();
    }

    @GetMapping("/stats")
    public Result<EmploymentStatsVO> getEmploymentStats(EmploymentQuery query) {
        String schoolId = SecurityUtils.getSchoolId();
        query.setSchoolId(schoolId);
        
        EmploymentStatsVO stats = employmentService.getEmploymentStats(query);
        return Result.success(stats);
    }
}
```

## 注意事项

1. **数据库设计**: 推荐直接使用 `biz_student.employment_status` 字段，就业详情从 `biz_job_delivery` 表关联获取
2. **就业状态设置**: 企业端在发放offer（将投递状态改为OFFER）时，自动将学生就业状态设置为"已就业"
3. **权限控制**: 学校端只能查看和修改本校学生的就业信息
4. **就业率计算**: 就业率 = 已就业人数 / 总人数 * 100
5. **默认值处理**: 如果学生没有就业记录，`employment_status` 默认为 '0'（待就业）
6. **事务控制**: 更新操作需要使用事务确保数据一致性
7. **日期格式**: 日期字段统一使用 `YYYY-MM-DD` 格式
8. **统计筛选**: 统计接口支持与列表相同的筛选条件，但不需要分页参数
9. **创业项目**: 创业项目通过 `biz_project` 表单独管理和统计，不计入就业状态
10. **就业与创业**: 学生可以同时就业和参与创业项目，两者不冲突
11. **简化状态**: 就业状态只有2种：待就业(0)、已就业(1)
12. **自动化流程**: 
    - 企业发放offer → 自动设置学生就业状态为"已就业"
    - 学生发布创业项目 → 不影响就业状态
13. **数据一致性**: 就业信息优先从 `biz_job_delivery` 表获取，确保数据准确性
