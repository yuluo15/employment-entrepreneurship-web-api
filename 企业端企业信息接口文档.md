# 企业端企业信息接口文档

## 接口概述

企业端企业信息模块提供查看和编辑企业基本信息、联系方式、企业介绍等功能。

---

## 1. 获取企业信息

### 接口信息
- **接口路径**: `GET /api/company/profile/info`
- **接口描述**: 获取当前登录HR所属企业的详细信息
- **权限要求**: 企业HR角色

### 请求参数

无

### SQL 实现示例

```sql
SELECT 
  c.id,
  c.name,
  c.code,
  c.industry,
  c.scale,
  c.founded_date AS "foundedDate",
  c.registered_capital AS "registeredCapital",
  c.website,
  c.logo,
  c.address,
  c.contact_person AS "contactPerson",
  c.contact_phone AS "contactPhone",
  c.email,
  c.description,
  c.benefits,
  c.status,
  c.create_time AS "createTime",
  c.update_time AS "updateTime"
FROM sys_company c
WHERE c.id = (
  SELECT company_id 
  FROM biz_hr 
  WHERE user_id = #{userId} AND status = 1
);
```

### 业务规则

1. **权限验证**: 
   - 从登录token中获取user_id
   - 通过biz_hr表查询company_id
   - 只能查看自己所属企业的信息

2. **数据完整性**: 返回企业的所有信息，包括认证状态

## 2. 更新企业信息

### 接口信息
- **接口路径**: `PUT /api/company/profile/update`
- **接口描述**: 更新企业信息
- **权限要求**: 企业HR角色

### 请求体

```json
{
  "id": "company_001",
  "name": "成都某某科技有限公司",
  "code": "91510100MA6CQ8XJ2X",
  "industry": "互联网/电子商务",
  "scale": "100-499",
  "foundedDate": "2018-06-15",
  "registeredCapital": "1000万元",
  "website": "https://www.example.com",
  "logo": "https://example.com/logo.png",
  "address": "成都市高新区天府软件园A座10楼",
  "contactPerson": "张经理",
  "contactPhone": "13800138000",
  "email": "hr@example.com",
  "description": "我们是一家专注于互联网技术研发的创新型企业...",
  "benefits": "五险一金、带薪年假、节日福利、员工体检、弹性工作、下午茶"
}
```

### 字段验证规则

| 字段名 | 必填 | 验证规则 |
|--------|------|----------|
| name | 是 | 2-100字符 |
| code | 是 | 18位统一社会信用代码，格式：数字+大写字母 |
| industry | 是 | 从预定义列表中选择 |
| scale | 是 | 从预定义列表中选择 |
| foundedDate | 否 | 日期格式 YYYY-MM-DD |
| registeredCapital | 否 | 字符串 |
| website | 否 | URL格式 |
| logo | 否 | URL格式 |
| address | 是 | 不能为空 |
| contactPerson | 是 | 不能为空 |
| contactPhone | 是 | 手机号格式 |
| email | 是 | 邮箱格式 |
| description | 是 | 至少20字符 |
| benefits | 否 | 字符串 |

### 响应数据

```json
{
  "code": 200,
  "msg": "更新成功",
  "data": null
}
```

### SQL 实现示例

```sql
UPDATE sys_company
SET 
  name = #{name},
  code = #{code},
  industry = #{industry},
  scale = #{scale},
  founded_date = #{foundedDate},
  registered_capital = #{registeredCapital},
  website = #{website},
  logo = #{logo},
  address = #{address},
  contact_person = #{contactPerson},
  contact_phone = #{contactPhone},
  email = #{email},
  description = #{description},
  benefits = #{benefits},
  update_time = CURRENT_TIMESTAMP
WHERE id = #{id}
  AND id = (
    SELECT company_id 
    FROM biz_hr 
    WHERE user_id = #{userId} AND status = 1
  );
```

### 业务规则

1. **权限验证**: 
   - 只能修改自己所属企业的信息
   - 通过WHERE条件确保安全性

2. **字段限制**:
   - 企业名称、统一社会信用代码等核心字段修改后可能需要重新审核
   - status字段不允许HR自行修改，只能由管理员审核

3. **数据验证**:
   - 统一社会信用代码格式验证
   - 联系电话、邮箱格式验证
   - 企业简介最少20字符

4. **审核机制**（可选）:
   - 如果修改了核心信息（企业名称、统一社会信用代码），可以将status重置为0（待审核）
   - 需要管理员重新审核后才能恢复正常状态

## 3. 上传企业LOGO

### 接口信息
- **接口路径**: `POST /api/common/upload`
- **接口描述**: 上传企业LOGO图片
- **权限要求**: 企业HR角色

### 请求参数

- **Content-Type**: `multipart/form-data`
- **file**: 图片文件

### 响应数据

```json
{
  "code": 200,
  "msg": "上传成功",
  "data": {
    "url": "https://example.com/uploads/logo/20260210/abc123.png",
    "filename": "abc123.png",
    "size": 102400
  }
}
```

### 业务规则

1. **文件限制**:
   - 支持格式：JPG、PNG
   - 文件大小：不超过2MB
   - 建议尺寸：200x200像素

2. **存储路径**: 
   - 按日期分目录存储：`/uploads/logo/YYYYMMDD/`
   - 文件名使用UUID避免冲突

3. **安全验证**:
   - 验证文件类型（MIME类型）
   - 验证文件大小
   - 防止恶意文件上传

## 数据库表结构

### sys_company（企业表）

```sql
CREATE TABLE sys_company (
  id VARCHAR(64) PRIMARY KEY,
  name VARCHAR(200) NOT NULL,
  code VARCHAR(18) NOT NULL UNIQUE,  -- 统一社会信用代码
  industry VARCHAR(100),
  scale VARCHAR(50),
  founded_date DATE,
  registered_capital VARCHAR(100),
  website VARCHAR(200),
  logo VARCHAR(500),
  address VARCHAR(500),
  contact_person VARCHAR(100),
  contact_phone VARCHAR(20),
  email VARCHAR(100),
  description TEXT,
  benefits TEXT,
  status INT2 DEFAULT 0,  -- 0=待审核, 1=已认证, 2=审核未通过
  create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  update_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_company_code ON sys_company(code);
CREATE INDEX idx_company_status ON sys_company(status);

COMMENT ON TABLE sys_company IS '企业信息表';
COMMENT ON COLUMN sys_company.code IS '统一社会信用代码';
COMMENT ON COLUMN sys_company.status IS '认证状态：0=待审核, 1=已认证, 2=审核未通过';
```

## 业务流程说明

### 企业信息管理流程

```
企业注册 → status=0(待审核) → 管理员审核 → status=1(已认证)
                                    ↓
                              status=2(审核未通过) → 修改信息 → 重新审核
```

### 信息修改流程

```
HR编辑信息 → 提交保存 → 
  ├─ 修改普通信息 → 直接保存
  └─ 修改核心信息（企业名称、信用代码） → status=0 → 等待管理员审核
```

## 注意事项

1. **统一社会信用代码**:
   - 18位，由数字和大写字母组成
   - 格式验证：`/^[0-9A-Z]{18}$/`
   - 全国唯一，不可重复

2. **企业认证**:
   - 新注册企业默认status=0（待审核）
   - 管理员审核通过后status=1（已认证）
   - 修改核心信息可能需要重新审核

3. **数据安全**:
   - 所有操作都需要验证company_id
   - 确保HR只能操作自己所属企业的数据

4. **图片上传**:
   - 建议使用OSS等云存储服务
   - 需要配置CORS允许跨域上传
   - 返回的URL需要是可访问的完整路径

5. **字段长度**:
   - 企业名称：2-100字符
   - 企业简介：至少20字符，最多2000字符
   - 福利待遇：最多1000字符
