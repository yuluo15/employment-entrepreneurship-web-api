# 学校端工作台接口文档

## 接口概述

学校端工作台提供学校管理员所需的核心数据概览，包括：
- 学生、教师统计
- 就业率和创业项目数据
- 待审核创业项目列表
- 最新通知公告
- 就业率趋势图表
- 创业项目状态分布
- 各学院就业情况

---

## 1. 获取学校工作台数据

### 接口信息
- **接口路径**: `GET /api/school/dashboard`
- **接口描述**: 获取学校工作台所有数据（一次请求返回所有数据）
- **请求参数**: 无（从登录用户token中获取学校ID）

### 字段说明

#### 核心指标
| 字段名 | 类型 | 说明 |
|--------|------|------|
| schoolName | string | 学校名称 |
| studentCount | number | 学生总数 |
| teacherCount | number | 教师总数 |
| employmentRate | string | 就业率（带%符号，如"92.5%"） |
| incubatingProjectCount | number | 孵化中的创业项目数量 |

#### 待审核项目 (pendingProjects)
| 字段名 | 类型 | 说明 |
|--------|------|------|
| projectId | string | 项目ID |
| projectName | string | 项目名称 |
| logo | string | 项目LOGO |
| studentName | string | 学生姓名 |
| domain | string | 项目领域（字典标签） |
| createTime | string | 提交时间 |

**排序规则**: 按提交时间降序排列，取最新5个待审核项目

#### 最新通知 (latestNotices)
| 字段名 | 类型 | 说明 |
|--------|------|------|
| noticeId | string | 通知ID |
| noticeTitle | string | 通知标题 |
| noticeType | string | 通知类型：1=通知, 2=公告, 3=政策, 4=新闻 |
| isTop | number | 是否置顶：0=否, 1=是 |
| publishTime | string | 发布时间 |

**排序规则**: 按置顶优先、发布时间降序排列，取最新5条

#### 就业率趋势 (employmentTrend)
| 字段名 | 类型 | 说明 |
|--------|------|------|
| months | string[] | 月份数组（格式：YYYY-MM） |
| rates | number[] | 对应月份的就业率（数值，不带%） |

**统计规则**: 
- 统计近6个月的就业率
- 就业率 = 已就业学生数 / 应届毕业生总数 * 100
- 只统计当届毕业生（根据 graduation_year 判断）

#### 项目状态分布 (projectStatus)
| 字段名 | 类型 | 说明 |
|--------|------|------|
| statusName | string | 状态名称：待审核、孵化中、已驳回、已落地 |
| count | number | 该状态的项目数量 |

**状态映射**:
- status = '0' → 待审核
- status = '1' → 孵化中
- status = '2' → 已驳回
- status = '3' → 已落地

#### 各学院就业情况 (collegeStats)
| 字段名 | 类型 | 说明 |
|--------|------|------|
| collegeName | string | 学院名称 |
| employmentRate | number | 就业率（数值，不带%） |

**排序规则**: 按就业率降序排列，取前10个学院

## 后端实现参考

### SQL 示例

```sql
-- 1. 获取学校基本信息
SELECT 
  s.name AS school_name
FROM sys_school s
WHERE s.id = #{schoolId};

-- 2. 学生总数
SELECT COUNT(*) 
FROM biz_student 
WHERE school_id = #{schoolId};

-- 3. 教师总数
SELECT COUNT(*) 
FROM biz_teacher 
WHERE school_id = #{schoolId};

-- 4. 就业率（当届毕业生）
SELECT 
  CONCAT(
    ROUND(
      COUNT(CASE WHEN employment_status = '1' THEN 1 END) * 100.0 / 
      NULLIF(COUNT(*), 0), 
      1
    ), 
    '%'
  ) AS employment_rate
FROM biz_student
WHERE school_id = #{schoolId}
  AND graduation_year = EXTRACT(YEAR FROM CURRENT_DATE);

-- 5. 孵化中的创业项目数量
SELECT COUNT(*) 
FROM biz_project 
WHERE school_id = #{schoolId} 
  AND status = '1';

-- 6. 待审核项目列表（最新5个）
SELECT 
  p.project_id,
  p.project_name,
  p.logo,
  s.student_name,
  d.dict_label AS domain,
  TO_CHAR(p.create_time, 'YYYY-MM-DD HH24:MI:SS') AS create_time
FROM biz_project p
JOIN biz_student s ON p.user_id = s.user_id
LEFT JOIN sys_dict_data d ON p.domain = d.dict_value AND d.dict_type = 'project_domain'
WHERE p.school_id = #{schoolId}
  AND p.status = '0'
ORDER BY p.create_time DESC
LIMIT 5;

-- 7. 最新通知公告（最新5条）
SELECT 
  notice_id,
  notice_title,
  notice_type,
  is_top,
  TO_CHAR(publish_time, 'YYYY-MM-DD HH24:MI:SS') AS publish_time
FROM sys_notice
WHERE status = 1
  AND (
    create_by = 'admin'  -- 管理员发布的
    OR create_by = #{schoolId}  -- 本校发布的
  )
ORDER BY is_top DESC, publish_time DESC
LIMIT 5;

-- 8. 近6个月就业率趋势
WITH months AS (
  SELECT 
    TO_CHAR(
      generate_series(
        DATE_TRUNC('month', CURRENT_DATE - INTERVAL '5 months'),
        DATE_TRUNC('month', CURRENT_DATE),
        '1 month'::interval
      ),
      'YYYY-MM'
    ) AS month
)
SELECT 
  m.month,
  COALESCE(
    ROUND(
      COUNT(CASE WHEN s.employment_status = '1' THEN 1 END) * 100.0 / 
      NULLIF(COUNT(s.student_id), 0), 
      1
    ), 
    0
  ) AS rate
FROM months m
LEFT JOIN biz_student s ON 
  TO_CHAR(s.create_time, 'YYYY-MM') <= m.month
  AND s.school_id = #{schoolId}
  AND s.graduation_year = EXTRACT(YEAR FROM CURRENT_DATE)
GROUP BY m.month
ORDER BY m.month ASC;

-- 9. 创业项目状态分布
SELECT 
  CASE 
    WHEN status = '0' THEN '待审核'
    WHEN status = '1' THEN '孵化中'
    WHEN status = '2' THEN '已驳回'
    WHEN status = '3' THEN '已落地'
  END AS status_name,
  COUNT(*) AS count
FROM biz_project
WHERE school_id = #{schoolId}
GROUP BY status
ORDER BY status ASC;

-- 10. 各学院就业情况（TOP 10）
SELECT 
  college_name,
  ROUND(
    COUNT(CASE WHEN employment_status = '1' THEN 1 END) * 100.0 / 
    NULLIF(COUNT(*), 0), 
    1
  ) AS employment_rate
FROM biz_student
WHERE school_id = #{schoolId}
  AND graduation_year = EXTRACT(YEAR FROM CURRENT_DATE)
  AND college_name IS NOT NULL
GROUP BY college_name
ORDER BY employment_rate DESC
LIMIT 10;
```

### 实现要点

1. **学校ID获取**: 从登录用户的 `owner_id` 字段获取学校ID
2. **就业率计算**: 
   - 只统计当届毕业生（graduation_year = 当前年份）
   - 就业率 = (employment_status='1'的学生数 / 总学生数) * 100
   - 保留1位小数
3. **通知查询**: 
   - 包含管理员发布的公告（create_by='admin'）
   - 包含本校发布的公告（create_by=schoolId）
   - 只显示已发布的（status=1）
4. **项目状态**: 使用 CASE WHEN 将状态码转换为中文标签
5. **趋势数据**: 使用 generate_series 生成近6个月的月份序列，确保数据完整

## Java 实现示例

### Controller

```java
@RestController
@RequestMapping("/api/school")
public class SchoolDashboardController {

    @Autowired
    private SchoolDashboardService dashboardService;

    @GetMapping("/dashboard")
    public Result<SchoolDashboardVO> getDashboard() {
        // 从token获取当前用户的学校ID
        String schoolId = SecurityUtils.getSchoolId();
        
        // 调用service获取工作台数据
        SchoolDashboardVO data = dashboardService.getDashboardData(schoolId);
        
        return Result.success(data);
    }
}
```

## 注意事项

1. **统一接口**: 这是一个统一的概览接口，一次请求返回所有首页需要的数据，避免多次请求
2. **权限控制**: 所有数据都基于当前登录用户的学校ID进行过滤
3. **就业率统计**: 只统计当届毕业生（graduation_year = 当前年份）
4. **数据完整性**: 
   - 就业率趋势需要补全6个月数据，没有数据的月份rate为0
   - 项目状态分布需要包含所有4种状态，即使count为0
5. **性能优化**: 
   - 可以考虑缓存学校基本信息（名称等变化不频繁的数据）
   - 就业率趋势可以定时任务预计算
6. **字典关联**: 项目领域需要关联字典表获取中文标签
7. **时间格式**: 统一使用 `YYYY-MM-DD HH24:MI:SS` 格式

### 1. 基本数据测试
- 验证学校名称正确显示

### 3. 通知公告测试
- 验证包含管理员发布的公告

- 验证不包含其他学校发布的公告

- 验证就业率趋势包含完整的6个月数据
- 验证项目状态分布包含所有4种状态
- 验证各学院就业情况按就业率降序排列

- 学校没有待审核项目时，返回空数组
- 学校没有通知时，返回空数组

## 数据库表依赖

本接口涉及以下数据库表：
- `sys_school` - 学校基本信息
- `biz_student` - 学生信息（统计学生数、就业率）
- `biz_teacher` - 教师信息（统计教师数）
- `biz_project` - 创业项目（待审核项目、项目状态分布）
- `sys_notice` - 通知公告（最新通知）
- `sys_dict_data` - 数据字典（项目领域标签）
