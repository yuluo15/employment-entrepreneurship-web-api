# 教师端项目详情接口文档

## 1. 获取项目详情（教师视角）

### 接口信息
- **接口路径**: `/api/teacher/project/{id}`
- **请求方式**: `GET`
- **接口描述**: 获取项目详细信息，教师视角，用于查看项目并发表指导意见

### 请求参数
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | String | 是 | 项目ID（路径参数） |

### 请求头
```
Authorization: Bearer {token}
```

### 请求示例
```
GET /api/teacher/project/1
```

### 响应数据

### 响应字段说明
| 字段名 | 类型 | 说明 |
|--------|------|------|
| projectId | String | 项目ID |
| projectName | String | 项目名称 |
| logo | String | 项目Logo URL |
| description | String | 项目介绍 |
| studentId | String | 负责人学生ID |
| studentName | String | 负责人姓名 |
| studentAvatar | String | 负责人头像 |
| studentPhone | String | 负责人联系电话 |
| schoolId | String | 所属学校ID |
| schoolName | String | 学校名称 |
| collegeName | String | 学院名称 |
| domain | String | 项目领域（多个用逗号分隔） |
| teamSize | Number | 团队规模 |
| status | String | 项目状态：0=待审核，1=孵化中，2=已驳回，3=已落地 |
| tags | Array | 项目标签 |
| createTime | String | 创建时间 |

### 数据库查询示例
```sql
SELECT 
  p.project_id,
  p.project_name,
  p.logo,
  p.description,
  p.student_id,
  u.nickname as student_name,
  u.avatar as student_avatar,
  u.phone as student_phone,
  p.school_id,
  s.school_name,
  p.college_name,
  p.domain,
  p.team_size,
  p.status,
  p.tags,
  p.create_time
FROM project p
LEFT JOIN user u ON p.student_id = u.user_id
LEFT JOIN school s ON p.school_id = s.school_id
WHERE p.project_id = #{projectId}
```

## 2. 获取项目指导记录列表

### 接口信息
- **接口路径**: `/api/teacher/project/{id}/guidance`
- **请求方式**: `GET`
- **接口描述**: 获取指定项目的所有指导记录，按时间倒序排列

### 请求参数
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | String | 是 | 项目ID（路径参数） |

### 请求头
```
Authorization: Bearer {token}
```

### 请求示例
```
GET /api/teacher/project/1/guidance
```

### 响应数据
```json
{
  "code": 200,
  "message": "success",
  "data": [
    {
      "id": "1",
      "projectId": "1",
      "teacherId": "201",
      "teacherName": "王老师",
      "teacherAvatar": "https://example.com/teacher-avatar.jpg",
      "content": "项目整体思路清晰，技术方案可行。建议在物联网设备选型时考虑成本因素，同时加强数据安全防护措施。",
      "createTime": "2小时前"
    },
    {
      "id": "2",
      "projectId": "1",
      "teacherId": "202",
      "teacherName": "李老师",
      "teacherAvatar": "https://example.com/teacher-avatar2.jpg",
      "content": "商业模式需要进一步完善，建议调研市场需求，明确目标客户群体。",
      "createTime": "1天前"
    }
  ]
}
```

### 响应字段说明
| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | String | 指导记录ID |
| projectId | String | 项目ID |
| teacherId | String | 指导教师ID |
| teacherName | String | 教师姓名 |
| teacherAvatar | String | 教师头像 |
| content | String | 指导内容 |
| createTime | String | 指导时间（相对时间） |

### 数据库查询示例
```sql
SELECT 
  g.id,
  g.project_id,
  g.teacher_id,
  t.name as teacher_name,
  t.avatar as teacher_avatar,
  g.content,
  g.create_time
FROM guidance g
LEFT JOIN teacher t ON g.teacher_id = t.teacher_id
WHERE g.project_id = #{projectId}
ORDER BY g.create_time DESC
```

## 3. 发表指导意见（复用已有接口）

该功能复用已有的指导记录接口，详见 `教师端指导记录接口文档.md` 中的 **发表指导意见** 接口。

### 接口信息
- **接口路径**: `/api/teacher/guidance/add`
- **请求方式**: `POST`
- **接口描述**: 教师对项目发表指导意见

### 请求参数
```json
{
  "projectId": "1",
  "content": "项目整体思路清晰，建议加强市场调研。"
}
```

## 业务逻辑说明

### 1. 权限控制
- 教师可以查看全网所有项目的详情
- 教师可以对任何项目发表指导意见
- 需要验证用户角色为教师（role = 'ROLE_TEACHER'）

### 2. 数据展示
- 项目详情页展示完整的项目信息
- 指导记录按时间倒序排列，最新的在最上面
- 时间字段需要转换为相对时间格式（如"2小时前"、"1天前"）

### 3. 指导意见发表
- 指导内容至少10个字
- 发表成功后自动刷新指导记录列表
- 指导记录会同步到教师的指导记录页面

### 4. 与学生端项目详情的区别
| 功能 | 学生端 | 教师端 |
|------|--------|--------|
| 查看项目详情 | ✓ | ✓ |
| 收藏项目 | ✓ | ✗ |
| 申请加入项目 | ✓ | ✗ |
| 查看指导记录 | ✓ | ✓ |
| 发表指导意见 | ✗ | ✓ |
| 联系项目负责人 | ✓ | ✓ |

## 错误码说明
| 错误码 | 说明 |
|--------|------|
| 401 | 未登录或token过期 |
| 403 | 无权限访问（非教师角色） |
| 404 | 项目不存在 |
| 500 | 服务器内部错误 |

## 注意事项

1. **时间格式转换**：后端返回的时间需要转换为相对时间格式
2. **领域标签处理**：domain字段可能包含多个领域（用逗号分隔），前端需要拆分并映射为中文
3. **头像默认值**：如果教师或学生没有头像，需要显示默认头像
4. **内容长度限制**：指导内容最少10个字，最多500个字
5. **实时更新**：发表指导意见后需要立即刷新指导记录列表
