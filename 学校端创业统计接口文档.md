# 学校端创业统计接口文档

## 概述

本接口用于学校端查看本校学生的创业项目统计数据，包括各学院项目数量、项目状态分布、领域分布、团队规模分布、月度趋势等。

**与管理员端创业统计的区别**:
- 管理员端：查看所有学校的创业项目，按学校维度统计
- 学校端：只查看本校学生的创业项目，按学院维度统计

---

## 接口地址

`GET /api/school/statistics/entrepreneurship`

---

## 请求头

```
Authorization: Bearer {token}
```

---

## 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| collegeName | string | 否 | 学院名称（用于筛选特定学院的数据） |

## 字段说明

### KPI 指标

| 字段名 | 类型 | 说明 |
|--------|------|------|
| totalProjects | int | 项目总数（所有状态） |
| incubatingCount | int | 孵化中的项目数（status='1'） |
| incubatingRate | string | 孵化率（孵化中/项目总数 * 100） |
| pendingCount | int | 待审核项目数（status='0'） |
| totalJobs | int | 创造岗位总数（所有项目的jobs_created之和） |

### 学院列表 (collegeList)

返回本校所有学院名称的数组，用于前端筛选下拉框。

### 学院统计 (collegeStats)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| collegeName | string | 学院名称 |
| count | int | 该学院的项目数量 |

**说明**: 按项目数量降序排列

### 状态分布 (statusDistribution)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| statusName | string | 状态名称 |
| count | int | 该状态的项目数量 |

**状态说明**:
- 待审核：status='0'
- 孵化中：status='1'
- 已驳回：status='2'
- 已落地：status='3'

### 领域分布 (domainDistribution)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| domain | string | 项目领域 |
| count | int | 该领域的项目数量 |

**说明**: 按项目数量降序排列

### 团队规模分布 (teamSizeDistribution)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| range | string | 团队规模范围 |
| count | int | 该范围的项目数量 |

**团队规模分类**:
- 1-3人
- 4-6人
- 7-10人
- 10人以上

### 月度趋势 (monthlyTrend)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| month | string | 月份（格式：YYYY-MM） |
| newCount | int | 该月新增项目数 |
| approvedCount | int | 该月审核通过项目数 |

**说明**: 返回最近6个月的数据，按月份升序排列

### 成功率统计 (successStats)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| approvalRate | string | 审核通过率（百分比，如"82.2"） |
| approvedCount | int | 审核通过项目数（status='1'或'3'） |
| submittedCount | int | 提交审核项目数（status!='0'） |
| landingRate | string | 项目落地率（百分比） |
| landedCount | int | 已落地项目数（status='3'） |
| avgTeamSize | number | 平均团队规模（保留1位小数） |

**计算公式**:
- 审核通过率 = (审核通过项目数 / 提交审核项目数) × 100
- 项目落地率 = (已落地项目数 / 审核通过项目数) × 100
- 平均团队规模 = 所有项目team_size之和 / 项目总数

## 后端实现参考

### SQL查询示例

```sql
-- 1. KPI指标查询
SELECT 
  COUNT(*) AS total_projects,
  SUM(CASE WHEN status = '1' THEN 1 ELSE 0 END) AS incubating_count,
  ROUND(
    (SUM(CASE WHEN status = '1' THEN 1 ELSE 0 END)::NUMERIC / 
     NULLIF(COUNT(*), 0)) * 100, 
    1
  ) AS incubating_rate,
  SUM(CASE WHEN status = '0' THEN 1 ELSE 0 END) AS pending_count,
  COALESCE(SUM(jobs_created), 0) AS total_jobs
FROM biz_project p
JOIN biz_student s ON p.user_id = s.user_id
WHERE p.school_id = #{schoolId}
  AND (#{collegeName} IS NULL OR s.college_name = #{collegeName});

-- 2. 获取学院列表
SELECT DISTINCT s.college_name
FROM biz_project p
JOIN biz_student s ON p.user_id = s.user_id
WHERE p.school_id = #{schoolId}
  AND s.college_name IS NOT NULL
ORDER BY s.college_name;

-- 3. 各学院项目数量统计
SELECT 
  s.college_name,
  COUNT(*) AS count
FROM biz_project p
JOIN biz_student s ON p.user_id = s.user_id
WHERE p.school_id = #{schoolId}
  AND (#{collegeName} IS NULL OR s.college_name = #{collegeName})
  AND s.college_name IS NOT NULL
GROUP BY s.college_name
ORDER BY count DESC;

-- 4. 项目状态分布统计
SELECT 
  CASE 
    WHEN status = '0' THEN '待审核'
    WHEN status = '1' THEN '孵化中'
    WHEN status = '2' THEN '已驳回'
    WHEN status = '3' THEN '已落地'
    ELSE '未知'
  END AS status_name,
  COUNT(*) AS count
FROM biz_project p
JOIN biz_student s ON p.user_id = s.user_id
WHERE p.school_id = #{schoolId}
  AND (#{collegeName} IS NULL OR s.college_name = #{collegeName})
GROUP BY status
ORDER BY 
  CASE status
    WHEN '0' THEN 1
    WHEN '1' THEN 2
    WHEN '2' THEN 3
    WHEN '3' THEN 4
    ELSE 5
  END;

-- 5. 项目领域分布统计
SELECT 
  p.domain,
  COUNT(*) AS count
FROM biz_project p
JOIN biz_student s ON p.user_id = s.user_id
WHERE p.school_id = #{schoolId}
  AND (#{collegeName} IS NULL OR s.college_name = #{collegeName})
  AND p.domain IS NOT NULL
GROUP BY p.domain
ORDER BY count DESC;

-- 6. 团队规模分布统计
SELECT 
  CASE 
    WHEN team_size <= 3 THEN '1-3人'
    WHEN team_size <= 6 THEN '4-6人'
    WHEN team_size <= 10 THEN '7-10人'
    ELSE '10人以上'
  END AS range,
  COUNT(*) AS count
FROM biz_project p
JOIN biz_student s ON p.user_id = s.user_id
WHERE p.school_id = #{schoolId}
  AND (#{collegeName} IS NULL OR s.college_name = #{collegeName})
GROUP BY range
ORDER BY 
  CASE range
    WHEN '1-3人' THEN 1
    WHEN '4-6人' THEN 2
    WHEN '7-10人' THEN 3
    WHEN '10人以上' THEN 4
    ELSE 5
  END;

-- 7. 月度项目趋势（最近6个月）
SELECT 
  TO_CHAR(p.create_time, 'YYYY-MM') AS month,
  COUNT(*) AS new_count,
  SUM(CASE WHEN p.status IN ('1', '3') THEN 1 ELSE 0 END) AS approved_count
FROM biz_project p
JOIN biz_student s ON p.user_id = s.user_id
WHERE p.school_id = #{schoolId}
  AND (#{collegeName} IS NULL OR s.college_name = #{collegeName})
  AND p.create_time >= CURRENT_DATE - INTERVAL '6 months'
GROUP BY TO_CHAR(p.create_time, 'YYYY-MM')
ORDER BY month ASC;

-- 8. 成功率统计
SELECT 
  -- 审核通过率
  ROUND(
    (SUM(CASE WHEN status IN ('1', '3') THEN 1 ELSE 0 END)::NUMERIC / 
     NULLIF(SUM(CASE WHEN status != '0' THEN 1 ELSE 0 END), 0)) * 100, 
    1
  ) AS approval_rate,
  SUM(CASE WHEN status IN ('1', '3') THEN 1 ELSE 0 END) AS approved_count,
  SUM(CASE WHEN status != '0' THEN 1 ELSE 0 END) AS submitted_count,
  
  -- 项目落地率
  ROUND(
    (SUM(CASE WHEN status = '3' THEN 1 ELSE 0 END)::NUMERIC / 
     NULLIF(SUM(CASE WHEN status IN ('1', '3') THEN 1 ELSE 0 END), 0)) * 100, 
    1
  ) AS landing_rate,
  SUM(CASE WHEN status = '3' THEN 1 ELSE 0 END) AS landed_count,
  
  -- 平均团队规模
  ROUND(AVG(team_size), 1) AS avg_team_size
FROM biz_project p
JOIN biz_student s ON p.user_id = s.user_id
WHERE p.school_id = #{schoolId}
  AND (#{collegeName} IS NULL OR s.college_name = #{collegeName});
```

### Java Controller示例

```java
@RestController
@RequestMapping("/api/school/statistics")
public class SchoolStatisticsController {

    @Autowired
    private SchoolStatisticsService statisticsService;

    @GetMapping("/entrepreneurship")
    public Result<EntrepreneurshipStatsVO> getEntrepreneurshipStats(
            @RequestParam(required = false) String collegeName) {
        
        String schoolId = SecurityUtils.getSchoolId();
        
        EntrepreneurshipStatsVO stats = statisticsService.getEntrepreneurshipStats(
            schoolId, collegeName
        );
        
        return Result.success(stats);
    }
}
```

## 注意事项

1. **权限控制**: 学校端只能查看本校（school_id）的创业项目数据
2. **数据来源**: 
   - 项目数据从 `biz_project` 表获取
   - 学生信息从 `biz_student` 表关联获取（通过user_id）
   - 学院信息从 `biz_student.college_name` 字段获取
3. **学院筛选**: 当传入 `collegeName` 参数时，所有统计数据都只统计该学院的数据
4. **项目状态**: 
   - 0 = 待审核（学生刚发布）
   - 1 = 孵化中（审核通过）
   - 2 = 已驳回（审核不通过）
   - 3 = 已落地（项目成功）
5. **孵化率计算**: 孵化率 = 孵化中项目数 / 项目总数 × 100
6. **审核通过率**: 审核通过率 = (孵化中 + 已落地) / (项目总数 - 待审核) × 100
7. **项目落地率**: 项目落地率 = 已落地项目数 / 审核通过项目数 × 100
8. **团队规模分类**: 根据team_size字段分为4个区间
9. **月度趋势**: 统计最近6个月的数据，包括新增项目和审核通过项目
10. **数据排序**: 
    - 学院统计按项目数量降序
    - 领域分布按项目数量降序
    - 状态分布按状态顺序（待审核、孵化中、已驳回、已落地）
11. **空值处理**: 对于 NULL 值要做适当处理，避免统计错误
12. **性能优化**: 建议在 `biz_project` 表的 `school_id`, `status`, `create_time` 字段上建立索引

## 与管理员端创业统计的对比

| 维度 | 管理员端 | 学校端 |
|------|---------|--------|
| 数据范围 | 所有学校 | 仅本校 |
| 主要统计维度 | 按学校统计 | 按学院统计 |
| 学院筛选 | 无 | 支持按学院筛选 |
| 项目审核 | 不涉及 | 可查看待审核数量 |
| 成功率统计 | 无 | 有（审核通过率、落地率） |
| 团队规模 | 不统计 | 统计分布情况 |
| 创造岗位 | 不统计 | 统计总数 |
