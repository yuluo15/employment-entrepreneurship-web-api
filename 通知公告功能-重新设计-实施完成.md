## 📝 数据流转

### 管理员发布公告
```
管理员发布公告
  ├─ publisher_type = 'admin'
  ├─ publisher_id = NULL
  ├─ target_audience = 'all' 或 'student'
  └─ 可见范围：
      ├─ all: 所有学校、所有学生
      └─ student: 所有学生
```

### 学校发布公告
```
学校发布公告
  ├─ publisher_type = 'school'
  ├─ publisher_id = 学校ID
  ├─ target_audience = 'student' (固定)
  └─ 可见范围：
      ├─ 本校学生
      └─ 本校教师（暂不开发）
```

### 学生查看公告
```
学生查看公告
  ├─ 管理员发布的公告
  │   ├─ publisher_type = 'admin'
  │   └─ target_audience IN ('all', 'student')
  │
  └─ 本校发布的公告
      ├─ publisher_type = 'school'
      └─ publisher_id = 学生的学校ID
```

### 1. 简化权限
- 使用 `publisher_type` 和 `publisher_id` 明确发布者身份
- 管理员：publisher_type='admin', publisher_id=NULL

- 管理员公告：publisher_type='admin' AND target_audience IN ('all', 'student')
- 学校公告：publisher_type='school' AND publisher_id=学生的学校ID

## ⚠️ 注意事项

### 1. 数据库字段
确保数据库表 `sys_notice` 中已有以下字段：
- `publisher_type` VARCHAR(255) - 发布者类型：'admin' 或 'school'
- `publisher_id` VARCHAR(255) - 发布者ID（学校ID）
- `target_audience` VARCHAR(255) - 目标受众：'all' 或 'student'

### 2. 大小写一致性
- 数据库中 `publisher_type` 使用小写：'admin', 'school'
- 代码中也使用小写进行比较和设置

### 3. 权限控制
- 所有接口都需要相应的角色权限
- 学校端接口需要验证 publisher_type 和 publisher_id
- 学生端接口需要验证学生的学校ID

### 4. 前端修改
前端需要相应修改：
- 管理员端：添加目标受众选择（'all' 或 'student'）
- 学校端：移除目标受众选择，修改权限判断逻辑
- 学生端：确认接口调用正确
