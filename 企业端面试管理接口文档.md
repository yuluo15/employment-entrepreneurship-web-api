# 企业端面试管理接口文档

## 📋 接口概述

面试管理模块用于管理企业的面试日程，包括面试列表查询、完成面试、取消面试、面试评价、发放Offer、归档候选人等功能。

---

## 1. 获取面试列表（分页）

### 接口信息
- **接口路径**: `GET /api/company/interview/list`
- **接口描述**: 分页查询面试日程列表
- **权限要求**: 企业HR角色

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| pageNum | number | 是 | 页码，从1开始 |
| pageSize | number | 是 | 每页条数 |
| studentName | string | 否 | 学生姓名（模糊查询） |
| status | string | 否 | 面试状态：SCHEDULED/COMPLETED/CANCELLED |
| startDate | string | 否 | 开始日期 YYYY-MM-DD |
| endDate | string | 否 | 结束日期 YYYY-MM-DD |

### 响应数据

```json
{

  "msg": "success",
  "data": {
```

## 数据库表结构说明

### biz_interview（面试记录表）

```sql
CREATE TABLE biz_interview (
  id VARCHAR(64) PRIMARY KEY,
  delivery_id VARCHAR(64) NOT NULL,
  company_id VARCHAR(64) NOT NULL,
  student_id VARCHAR(64) NOT NULL,
  job_id VARCHAR(64) NOT NULL,
  interview_time TIMESTAMP NOT NULL,
  duration INT NOT NULL,
  type VARCHAR(20) NOT NULL,  -- ONSITE/VIDEO/PHONE
  location VARCHAR(500),
  notes TEXT,
  status VARCHAR(20) NOT NULL,  -- SCHEDULED/COMPLETED/CANCELLED
  interview_score INT,  -- 1-5
  interview_comment TEXT,
  interview_result VARCHAR(20),  -- PASS/FAIL
  cancel_reason TEXT,
  create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  update_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_interview_company ON biz_interview(company_id, status);
CREATE INDEX idx_interview_student ON biz_interview(student_id);
CREATE INDEX idx_interview_delivery ON biz_interview(delivery_id);
CREATE INDEX idx_interview_time ON biz_interview(interview_time);
```

**status 字段说明**：
- `SCHEDULED`: 已安排（待面试）
- `COMPLETED`: 已完成
- `CANCELLED`: 已取消

**type 字段说明**：
- `ONSITE`: 现场面试
- `VIDEO`: 视频面试
- `PHONE`: 电话面试

**interview_result 字段说明**：
- `PASS`: 面试通过
- `FAIL`: 面试不通过

## SQL 实现示例

### 1. 获取面试列表

```sql
SELECT 
  i.id,
  i.delivery_id AS "deliveryId",
  i.student_id AS "studentId",
  s.name AS "studentName",
  s.phone AS "studentPhone",
  i.job_id AS "jobId",
  j.job_name AS "jobName",
  i.interview_time AS "interviewTime",
  i.duration,
  i.type,
  i.location,
  i.notes,
  i.status,
  i.interview_score AS "interviewScore",
  i.interview_comment AS "interviewComment",
  i.interview_result AS "interviewResult"
FROM biz_interview i
LEFT JOIN biz_student s ON i.student_id = s.student_id
LEFT JOIN biz_job j ON i.job_id = j.job_id
WHERE i.company_id = #{companyId}
  AND (#{studentName} IS NULL OR s.name LIKE CONCAT('%', #{studentName}, '%'))
  AND (#{status} IS NULL OR i.status = #{status})
  AND (#{startDate} IS NULL OR i.interview_time >= #{startDate})
  AND (#{endDate} IS NULL OR i.interview_time <= #{endDate})
ORDER BY i.interview_time DESC
LIMIT #{pageSize} OFFSET (#{pageNum} - 1) * #{pageSize};
```

### 2. 完成面试

```sql
-- 更新面试记录
UPDATE biz_interview
SET 
  status = 'COMPLETED',
  interview_score = #{score},
  interview_comment = #{comment},
  interview_result = #{result},
  update_time = CURRENT_TIMESTAMP
WHERE id = #{interviewId}
  AND company_id = #{companyId}
  AND status = 'SCHEDULED';

-- 更新投递状态为面试中（如果还不是）
UPDATE biz_job_delivery
SET 
  status = 'INTERVIEW',
  update_time = CURRENT_TIMESTAMP
WHERE id = (SELECT delivery_id FROM biz_interview WHERE id = #{interviewId})
  AND status = 'DELIVERED';
```

### 3. 取消面试

```sql
UPDATE biz_interview
SET 
  status = 'CANCELLED',
  cancel_reason = #{reason},
  update_time = CURRENT_TIMESTAMP
WHERE id = #{interviewId}
  AND company_id = #{companyId}
  AND status = 'SCHEDULED';
```

### 4. 面试评价

```sql
UPDATE biz_interview
SET 
  interview_score = #{score},
  interview_comment = #{comment},
  update_time = CURRENT_TIMESTAMP
WHERE id = #{interviewId}
  AND company_id = #{companyId}
  AND status = 'COMPLETED';
```

### 5. 发放Offer

```sql
-- 1. 更新投递状态为已录用
UPDATE biz_job_delivery
SET 
  status = 'OFFER',
  update_time = CURRENT_TIMESTAMP
WHERE id = #{deliveryId}
  AND company_id = #{companyId};

-- 2. 创建Offer记录
INSERT INTO biz_offer (
  id,
  delivery_id,
  company_id,
  student_id,
  job_id,
  entry_date,
  salary,
  notes,
  status,
  create_time
) VALUES (
  #{offerId},
  #{deliveryId},
  #{companyId},
  (SELECT student_id FROM biz_job_delivery WHERE id = #{deliveryId}),
  (SELECT job_id FROM biz_job_delivery WHERE id = #{deliveryId}),
  #{entryDate},
  #{salary},
  #{notes},
  'PENDING',
  CURRENT_TIMESTAMP
);

-- 3. 发送消息通知学生
INSERT INTO sys_message (
  id,
  receiver_id,
  title,
  content,
  type,
  is_read,
  ref_id,
  create_time
) VALUES (
  #{messageId},
  (SELECT student_id FROM biz_job_delivery WHERE id = #{deliveryId}),
  'Offer通知',
  CONCAT('恭喜您获得Offer！入职时间：', #{entryDate}, '，薪资：', #{salary}),
  'OFFER',
  0,
  #{deliveryId},
  CURRENT_TIMESTAMP
);
```

### 6. 归档候选人

```sql
-- 更新投递状态为已拒绝
UPDATE biz_job_delivery
SET 
  status = 'REJECTED',
  reject_reason = '面试不通过',
  update_time = CURRENT_TIMESTAMP
```

## 业务流程说明

### 面试完整流程

```
新简历投递 (DELIVERED)
    ↓
安排面试 → 创建面试记录 (SCHEDULED)
    ↓
投递状态变更为 INTERVIEW
    ↓
完成面试 → 记录结果和评价 (COMPLETED)
    ↓
    ├─ 面试通过 (PASS)
    │     ↓
    │  发放Offer → 状态变为 OFFER → 进入人才库
    │
    └─ 面试不通过 (FAIL)
          ↓
       归档 → 状态变为 REJECTED → 进入人才库
```

### 状态流转规则

1. **投递状态流转**:
   - `DELIVERED` → `INTERVIEW` (安排面试)
   - `INTERVIEW` → `OFFER` (发放Offer)
   - `INTERVIEW` → `REJECTED` (归档)

2. **面试状态流转**:
   - `SCHEDULED` → `COMPLETED` (完成面试)
   - `SCHEDULED` → `CANCELLED` (取消面试)

3. **面试结果**:
   - `PASS`: 可以发放Offer
   - `FAIL`: 可以归档

---

## 业务规则

### 1. 权限控制
- 所有操作都需要验证 `company_id`
- 确保HR只能操作自己公司的面试记录
- 查询时必须加上公司ID过滤条件

### 2. 状态验证
- 只有 `SCHEDULED` 状态的面试可以完成或取消
- 只有 `COMPLETED` 状态的面试可以补充评价
- 只有面试结果为 `PASS` 的可以发放Offer
- 只有面试结果为 `FAIL` 的可以归档

### 3. 数据完整性
- 完成面试时必须填写结果、评分和评价
- 发放Offer时必须填写入职时间和薪资
- 取消面试时必须填写取消原因

### 4. 消息通知
- 发放Offer后自动发送消息通知学生
- 消息类型为 `OFFER`
- 消息内容包含入职时间和薪资信息

### Service 层

```java
package com.example.service;

import com.example.vo.InterviewVO;
import com.example.dto.CompleteInterviewDTO;
import com.example.dto.OfferDTO;
import com.example.common.PageResult;
import com.example.mapper.InterviewMapper;
import com.example.mapper.JobDeliveryMapper;
import com.example.util.SecurityUtils;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

import java.util.List;

@Service
public class InterviewService {
    
    @Autowired
    private InterviewMapper interviewMapper;
    
    @Autowired
    private JobDeliveryMapper jobDeliveryMapper;
    
    @Autowired
    private MessageService messageService;
    
    /**
     * 获取面试列表
     */
    public PageResult<InterviewVO> getInterviewList(
        Integer pageNum, Integer pageSize,
        String studentName, String status,
        String startDate, String endDate
    ) {
        String companyId = SecurityUtils.getCurrentCompanyId();
        
        List<InterviewVO> list = interviewMapper.selectInterviewList(
            companyId, studentName, status, startDate, endDate,
            (pageNum - 1) * pageSize, pageSize
        );
        
        Integer total = interviewMapper.countInterview(
            companyId, studentName, status, startDate, endDate
        );
        
        return new PageResult<>(list, total);
    }
    
    /**
     * 完成面试
     */
    @Transactional
    public void completeInterview(CompleteInterviewDTO dto) {
        String companyId = SecurityUtils.getCurrentCompanyId();
        
        // 验证权限
        Interview interview = interviewMapper.selectById(dto.getInterviewId());
        if (!interview.getCompanyId().equals(companyId)) {
            throw new BusinessException("无权操作");
        }
        
        // 更新面试记录
        interviewMapper.completeInterview(
            dto.getInterviewId(),
            dto.getResult(),
            dto.getScore(),
            dto.getComment()
        );
        
        // 更新投递状态为面试中
        jobDeliveryMapper.updateStatus(interview.getDeliveryId(), "INTERVIEW");
    }
    
    /**
     * 取消面试
     */
    @Transactional
    public void cancelInterview(String interviewId, String reason) {
        String companyId = SecurityUtils.getCurrentCompanyId();
        
        Interview interview = interviewMapper.selectById(interviewId);
        if (!interview.getCompanyId().equals(companyId)) {
            throw new BusinessException("无权操作");
        }
        
        interviewMapper.cancelInterview(interviewId, reason);
    }
    
    /**
     * 面试评价
     */
    @Transactional
    public void evaluateInterview(String interviewId, Integer score, String comment) {
        String companyId = SecurityUtils.getCurrentCompanyId();
        
        Interview interview = interviewMapper.selectById(interviewId);
        if (!interview.getCompanyId().equals(companyId)) {
            throw new BusinessException("无权操作");
        }
        
        interviewMapper.evaluateInterview(interviewId, score, comment);
    }
    
    /**
     * 发放Offer
     */
    @Transactional
    public void sendOfferFromInterview(OfferDTO dto) {
        String companyId = SecurityUtils.getCurrentCompanyId();
        
        // 验证权限
        JobDelivery delivery = jobDeliveryMapper.selectById(dto.getDeliveryId());
        if (!delivery.getCompanyId().equals(companyId)) {
            throw new BusinessException("无权操作");
        }
        
        // 更新投递状态为已录用
        jobDeliveryMapper.updateStatus(dto.getDeliveryId(), "OFFER");
        
        // 创建Offer记录
        Offer offer = new Offer();
        offer.setDeliveryId(dto.getDeliveryId());
        offer.setCompanyId(companyId);
        offer.setStudentId(delivery.getStudentId());
        offer.setJobId(delivery.getJobId());
        offer.setEntryDate(dto.getEntryDate());
        offer.setSalary(dto.getSalary());
        offer.setNotes(dto.getNotes());
        offer.setStatus("PENDING");
        offerMapper.insert(offer);
        
        // 发送消息通知
        messageService.sendOfferNotification(
            delivery.getStudentId(),
            dto.getEntryDate(),
            dto.getSalary()
        );
    }
    
    /**
     * 归档候选人
     */
    @Transactional
    public void archiveCandidate(String deliveryId) {
        String companyId = SecurityUtils.getCurrentCompanyId();
        
        JobDelivery delivery = jobDeliveryMapper.selectById(deliveryId);
        if (!delivery.getCompanyId().equals(companyId)) {
            throw new BusinessException("无权操作");
        }
        
        // 更新投递状态为已拒绝
        jobDeliveryMapper.updateStatusWithReason(
            deliveryId,
            "REJECTED",
            "面试不通过"
        );
    }
}
```

### 1. 数据权限
- 所有操作都需要验证 `company_id`
- 确保HR只能操作自己公司的面试记录

### 2. 状态管理
- 完成面试后，根据结果显示不同操作按钮
- 面试通过显示"发放Offer"按钮
- 面试不通过显示"归档"按钮

### 3. 业务逻辑
- 发放Offer后，候选人进入人才库（OFFER状态）
- 归档后，候选人进入人才库（REJECTED状态）
- 人才库可以查看所有已录用和已拒绝的候选人

### 4. 消息通知
- 发放Offer后自动发送消息通知学生
- 消息类型为 `OFFER`

## 测试用例

### 1. 获取面试列表
```bash
GET /api/company/interview/list?pageNum=1&pageSize=10
GET /api/company/interview/list?pageNum=1&pageSize=10&status=SCHEDULED
GET /api/company/interview/list?pageNum=1&pageSize=10&studentName=张三
```

### 2. 完成面试
```bash
POST /api/company/interview/complete
Content-Type: application/json

{
  "interviewId": "interview_123456",
  "result": "PASS",
  "score": 5,
  "comment": "候选人表现优秀"
}
```

### 3. 发放Offer
```bash
POST /api/company/interview/offer
Content-Type: application/json

{
  "deliveryId": "delivery_001",
  "entryDate": "2026-03-01",
  "salary": "8000-10000元/月",
  "notes": "欢迎加入"
}
```

### 4. 归档候选人
```bash
POST /api/company/interview/archive
Content-Type: application/json

{
  "deliveryId": "delivery_001"
}
```
