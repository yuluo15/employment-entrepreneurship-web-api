# 学生端消息通知接口文档

## 接口概述

学生端消息通知功能，用于展示企业发送的面试通知、Offer通知和系统通知。

**基础路径**: `/api/mobile/messages`

---

## 1. 获取消息列表

### 接口信息

- **接口地址**: `GET /api/mobile/messages`
- **接口描述**: 分页获取学生的消息列表
- **权限要求**: 学生登录

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| pageNum | Integer | 是 | 页码，从1开始 |
| pageSize | Integer | 是 | 每页数量，建议10-20 |
| type | Integer | 否 | 消息类型：1=系统通知, 2=面试通知, 3=Offer通知 |
| isRead | Integer | 否 | 阅读状态：0=未读, 1=已读 |

### 请求示例

```http
GET /api/mobile/messages?pageNum=1&pageSize=10&type=2
Authorization: Bearer {token}
```

| companyName | String | 公司名称（从关联投递记录获取） |
| positionName | String | 职位名称（从关联投递记录获取） |

### SQL实现

```sql
SELECT 
  m.id,
  m.title,
  m.content,
  m.type,
  m.is_read AS "isRead",
  m.ref_id AS "refId",
  TO_CHAR(m.create_time, 'YYYY-MM-DD HH24:MI:SS') AS "createTime",
  c.name AS "companyName",
  p.position_name AS "positionName"
FROM sys_message m
LEFT JOIN biz_job_application ja ON m.ref_id = ja.id
LEFT JOIN biz_position p ON ja.position_id = p.id
LEFT JOIN sys_company c ON p.company_id = c.id
WHERE m.receiver_id = #{studentId}
  AND (#{type} IS NULL OR m.type = #{type})
  AND (#{isRead} IS NULL OR m.is_read = #{isRead})
ORDER BY m.create_time DESC
LIMIT #{pageSize} OFFSET (#{pageNum} - 1) * #{pageSize};

-- 查询总数
SELECT COUNT(*) 
FROM sys_message m
WHERE m.receiver_id = #{studentId}
  AND (#{type} IS NULL OR m.type = #{type})
  AND (#{isRead} IS NULL OR m.is_read = #{isRead});
```

### Java实现示例

```java
@RestController
@RequestMapping("/api/mobile/messages")
public class MessageController {
    
    @Autowired
    private MessageService messageService;
    
    @GetMapping
    public Result<PageResult<MessageVO>> getMessages(
            @RequestParam Integer pageNum,
            @RequestParam Integer pageSize,
            @RequestParam(required = false) Integer type,
            @RequestParam(required = false) Integer isRead) {
        
        // 获取当前登录学生ID
        String studentId = SecurityUtils.getCurrentUserId();
        
        MessageQuery query = new MessageQuery();
        query.setPageNum(pageNum);
        query.setPageSize(pageSize);
        query.setType(type);
        query.setIsRead(isRead);
        query.setReceiverId(studentId);
        
        PageResult<MessageVO> result = messageService.getMessages(query);
        return Result.success(result);
    }
}
```

## 2. 获取消息详情

### 接口信息

- **接口地址**: `GET /api/mobile/messages/{id}`
- **接口描述**: 获取消息详情，并自动标记为已读
- **权限要求**: 学生登录

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | String | 是 | 消息ID（路径参数） |

### 请求示例

```http
GET /api/mobile/messages/msg001
Authorization: Bearer {token}
```

### 响应数据

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": "msg001",
    "title": "您投递的职位已安排面试",
    "content": "您投递的职位已安排面试\n\n职位名称：前端开发工程师\n公司名称：腾讯科技\n面试时间：2026-02-28 14:00\n面试地点：深圳市南山区腾讯大厦\n联系方式：0755-12345678\n\n备注：请携带简历和作品集，提前10分钟到达。",
    "type": 2,
    "isRead": 1,
    "refId": "app001",
    "createTime": "2026-02-21 14:30:00",
    "companyName": "腾讯科技",
    "positionName": "前端开发工程师",
    "positionId": "pos001",
    "applicationId": "app001"
  }

```

### 响应字段说明

| 字段名 | 类型 | 说明 |
|--------|------|------|
| positionId | String | 职位ID（用于跳转职位详情） |
| applicationId | String | 投递记录ID（用于跳转投递记录） |
| 其他字段 | - | 同消息列表接口 |

### SQL实现

```sql
-- 查询消息详情
SELECT 
  m.id,
  m.title,
  m.content,
  m.type,
  m.is_read AS "isRead",
  m.ref_id AS "refId",
  TO_CHAR(m.create_time, 'YYYY-MM-DD HH24:MI:SS') AS "createTime",
  c.name AS "companyName",
  p.position_name AS "positionName",
  p.id AS "positionId",
  ja.id AS "applicationId"
FROM sys_message m
LEFT JOIN biz_job_application ja ON m.ref_id = ja.id
LEFT JOIN biz_position p ON ja.position_id = p.id
LEFT JOIN sys_company c ON p.company_id = c.id
WHERE m.id = #{id}
  AND m.receiver_id = #{studentId};

-- 标记为已读
UPDATE sys_message
SET is_read = 1
WHERE id = #{id}
  AND receiver_id = #{studentId}
  AND is_read = 0;
```

### Java实现示例

```java
@GetMapping("/{id}")
public Result<MessageDetailVO> getMessageDetail(@PathVariable String id) {
    String studentId = SecurityUtils.getCurrentUserId();
    
    // 查询消息详情
    MessageDetailVO detail = messageService.getMessageDetail(id, studentId);
    
    if (detail == null) {
        return Result.error("消息不存在");
    }
    
    // 自动标记为已读
    messageService.markAsRead(id, studentId);
    
    return Result.success(detail);
}
```

## 3. 获取未读消息数量

### 接口信息

- **接口地址**: `GET /api/mobile/messages/unread-count`
- **接口描述**: 获取各类型未读消息数量
- **权限要求**: 学生登录

### 请求参数

无

### 请求示例

```http
GET /api/mobile/messages/unread-count
Authorization: Bearer {token}
```

### 响应数据

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "total": 5,
    "systemCount": 1,
    "interviewCount": 3,
    "offerCount": 1
  }
}
```

### 响应字段说明

| 字段名 | 类型 | 说明 |
|--------|------|------|
| total | Integer | 总未读消息数 |
| systemCount | Integer | 系统通知未读数 |
| interviewCount | Integer | 面试通知未读数 |
| offerCount | Integer | Offer通知未读数 |

### SQL实现

```sql
SELECT 
  COUNT(*) AS total,
  SUM(CASE WHEN type = 1 THEN 1 ELSE 0 END) AS "systemCount",
  SUM(CASE WHEN type = 2 THEN 1 ELSE 0 END) AS "interviewCount",
  SUM(CASE WHEN type = 3 THEN 1 ELSE 0 END) AS "offerCount"
FROM sys_message
WHERE receiver_id = #{studentId}
  AND is_read = 0;
```

### Java实现示例

```java
@GetMapping("/unread-count")
public Result<UnreadCountVO> getUnreadCount() {
    String studentId = SecurityUtils.getCurrentUserId();
    UnreadCountVO count = messageService.getUnreadCount(studentId);
    return Result.success(count);
}
```

## 5. 批量标记已读

### 接口信息

- **接口地址**: `PUT /api/mobile/messages/read-all`
- **接口描述**: 批量标记消息为已读，可指定消息类型
- **权限要求**: 学生登录

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| type | Integer | 否 | 消息类型：1=系统通知, 2=面试通知, 3=Offer通知。不传则标记全部 |

### 请求示例

```http
PUT /api/mobile/messages/read-all
Content-Type: application/json
Authorization: Bearer {token}

{
  "type": 2
}
```

### 响应数据

```json
{
  "code": 200,
  "message": "已标记3条消息为已读"
}
```

### SQL实现

```sql
UPDATE sys_message
SET is_read = 1
WHERE receiver_id = #{studentId}
  AND is_read = 0
  AND (#{type} IS NULL OR type = #{type});
```

### Java实现示例

```java
@PutMapping("/read-all")
public Result<Void> markAllAsRead(@RequestBody(required = false) ReadAllRequest request) {
    String studentId = SecurityUtils.getCurrentUserId();
    Integer type = request != null ? request.getType() : null;
    
    int rows = messageService.markAllAsRead(studentId, type);
    
    return Result.success("已标记" + rows + "条消息为已读");
}
```

## 数据模型

### MessageEntity (sys_message表)

```java
@Data
@TableName("sys_message")
public class MessageEntity {
    @TableId
    private String id;
    
    private String receiverId;  // 接收人ID（学生ID）
    private String title;        // 标题
    private String content;      // 内容
    private Integer type;        // 消息类型（1:系统通知, 2:面试通知, 3:offer通知）

    private String refId;        // 关联业务ID（投递ID）
    private Timestamp createTime; // 创建时间
}
```

## 企业端创建消息示例

### 安排面试时创建消息

```java
public void scheduleInterview(InterviewDTO dto) {
    // 1. 保存面试信息
    Interview interview = new Interview();
    interview.setId(UUID.randomUUID().toString());
    interview.setApplicationId(dto.getApplicationId());
    interview.setInterviewTime(dto.getInterviewTime());
    interview.setInterviewAddress(dto.getInterviewAddress());
    interview.setContactPhone(dto.getContactPhone());
    interview.setRemark(dto.getRemark());
    interviewMapper.insert(interview);
    
    // 2. 获取投递记录，获取学生ID
    JobApplication application = applicationMapper.selectById(dto.getApplicationId());
    
    // 3. 获取职位和公司信息
    Position position = positionMapper.selectById(application.getPositionId());
    Company company = companyMapper.selectById(position.getCompanyId());
    
    // 4. 构建消息内容
    String content = String.format(
        "您投递的职位已安排面试\n\n" +
        "职位名称：%s\n" +
        "公司名称：%s\n" +
        "面试时间：%s\n" +
        "面试地点：%s\n" +
        "联系方式：%s\n\n" +
        "备注：%s",
        position.getPositionName(),
        company.getName(),
        dto.getInterviewTime(),
        dto.getInterviewAddress(),
        dto.getContactPhone(),
        dto.getRemark() != null ? dto.getRemark() : "无"
    );
    
    // 5. 创建消息
    MessageEntity message = new MessageEntity();
    message.setId(UUID.randomUUID().toString());
    message.setReceiverId(application.getStudentId());
    message.setTitle("您投递的职位已安排面试");
    message.setContent(content);
    message.setType(2);  // 面试通知
    message.setIsRead(0);
    message.setRefId(dto.getApplicationId());
    message.setCreateTime(new Timestamp(System.currentTimeMillis()));
    
    messageMapper.insert(message);
}
```

## 注意事项

1. **权限校验**: 所有接口都需要验证学生登录状态，并且只能操作自己的消息
2. **SQL注入防护**: 使用参数化查询，避免SQL注入
3. **数据关联**: 消息列表和详情需要关联查询投递记录、职位、公司信息
4. **自动已读**: 查看消息详情时自动标记为已读
5. **软删除**: 可以考虑使用软删除（添加deleted字段）而不是物理删除
6. **性能优化**: 添加适当的索引，优化查询性能
