# 企业注册接口文档

## 概述

本接口用于企业端自主注册账号，包括发送邮箱验证码和提交注册信息两个接口。

**注册流程**:
1. 用户输入邮箱，点击发送验证码
2. 后端发送6位验证码到邮箱，存储在Redis（5分钟有效）
3. 用户填写完整注册信息（包括验证码）
4. 后端验证邮箱验证码，创建企业和HR账号
5. 企业状态为待审核（status=0），管理员审核通过后才能登录

## 接口1: 发送邮箱验证码

### 接口地址

`POST /api/sendEmail/code`

### 请求头

无需认证（公开接口）

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| email | string | 是 | 邮箱地址 |

### 请求示例

```json
{
  "email": "hr@company.com"
}
```

### 响应示例

```json
{
  "message": "success",
  "data": null
}
```

### 后端实现要点

1. **验证邮箱格式**: 使用正则表达式验证邮箱格式
2. **检查邮箱是否已注册**: 查询 `sys_user` 表，如果邮箱已存在则返回错误
3. **生成6位随机验证码**: 使用 `Random` 或 `UUID` 生成
4. **存储到Redis**: 
   - Key: `email:code:{email}`
   - Value: 验证码
   - 过期时间: 5分钟（300秒）
5. **发送邮件**: 调用邮件服务发送验证码

### Java实现示例

```java
@RestController
@RequestMapping("/api/sendEmail")
public class EmailController {

    @Autowired
    private RedisTemplate<String, String> redisTemplate;
    
    @Autowired
    private JavaMailSender mailSender;
    
    @Autowired
    private UserService userService;
    
    @Value("${spring.mail.username}")
    private String fromEmail;

    @PostMapping("/code")
    public Result<Void> sendEmailCode(@RequestBody EmailCodeReq req) {
        String email = req.getEmail();
        
        // 1. 验证邮箱格式
        if (!email.matches("^[^\\s@]+@[^\\s@]+\\.[^\\s@]+$")) {
            return Result.error("邮箱格式不正确");
        }
        
        // 2. 检查邮箱是否已注册
        if (userService.existsByEmail(email)) {
            return Result.error("该邮箱已被注册");
        }
        
        // 3. 生成6位验证码
        String code = String.format("%06d", new Random().nextInt(999999));
        
        // 4. 存储到Redis（5分钟有效）
        String redisKey = "email:code:" + email;
        redisTemplate.opsForValue().set(redisKey, code, 5, TimeUnit.MINUTES);
        
        // 5. 发送邮件
        try {
            SimpleMailMessage message = new SimpleMailMessage();
            message.setFrom(fromEmail);
            message.setTo(email);
            message.setSubject("【高校就业平台】企业注册验证码");
            message.setText("您的验证码是：" + code + "，有效期5分钟，请勿泄露给他人。");
            mailSender.send(message);
        } catch (Exception e) {
            log.error("发送邮件失败", e);
            return Result.error("发送邮件失败，请稍后重试");
        }
        
        return Result.success();
    }
}

@Data
class EmailCodeReq {
    private String email;
}
```

## 接口2: 企业注册

### 接口地址

`POST /api/company/register`

### 请求头

无需认证（公开接口）

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| adminAccount | string | 是 | 管理员邮箱（登录账号） |
| emailCode | string | 是 | 邮箱验证码（6位） |
| password | string | 是 | 登录密码 |
| name | string | 是 | 企业全称 |
| code | string | 是 | 统一社会信用代码（18位） |
| industry | string | 是 | 所属行业 |
| scale | string | 是 | 人员规模 |
| licenseUrl | string | 是 | 营业执照图片URL |
| contactPerson | string | 是 | 联系人姓名 |
| contactPhone | string | 是 | 联系电话 |

### 请求示例

```json
{
  "adminAccount": "hr@company.com",
  "emailCode": "123456",
  "password": "password123",
  "name": "某某科技有限公司",
  "code": "91110000123456789X",
  "industry": "互联网/IT",
  "scale": "100-499人",
  "licenseUrl": "https://example.com/license.jpg",
  "contactPerson": "张三",
  "contactPhone": "13800138000"
}
```

### 响应示例

```json
{
  "message": "success",
  "data": null
}
```

### 后端实现要点

1. **验证邮箱验证码**: 从Redis读取验证码，比对是否正确
2. **检查邮箱是否已注册**: 再次检查邮箱是否已存在
3. **检查信用代码是否重复**: 查询 `sys_company` 表
4. **创建企业记录**: 插入 `sys_company` 表，status=0（待审核）
5. **创建HR账号**: 插入 `sys_user` 表，role='ROLE_COMPANY'
6. **创建HR信息**: 插入 `biz_hr` 表
7. **删除Redis验证码**: 验证成功后删除，防止重复使用
8. **事务处理**: 使用 `@Transactional` 确保数据一致性

### 数据库表结构

#### sys_company 表

```sql
CREATE TABLE sys_company (
  id              varchar(64)  PRIMARY KEY,
  name            varchar(200) NOT NULL,           -- 企业全称
  code            varchar(18)  UNIQUE NOT NULL,    -- 统一社会信用代码
  industry        varchar(50),                     -- 所属行业
  scale           varchar(50),                     -- 人员规模
  license_url     varchar(500),                    -- 营业执照URL
  contact_person  varchar(50),                     -- 联系人
  contact_phone   varchar(20),                     -- 联系电话
  status          int2         DEFAULT 0,          -- 审核状态 (0:待审核, 1:已通过, 2:已驳回)
  reason          text,                            -- 驳回原因
  create_time     timestamp    DEFAULT CURRENT_TIMESTAMP,
  update_time     timestamp    DEFAULT CURRENT_TIMESTAMP
);
```

#### sys_user 表

```sql
CREATE TABLE sys_user (
  user_id         varchar(64)  PRIMARY KEY,
  login_name      varchar(100) UNIQUE NOT NULL,    -- 登录账号（邮箱）
  password        varchar(200) NOT NULL,           -- 密码（加密）
  show_name       varchar(100),                    -- 显示名称
  role            varchar(50)  NOT NULL,           -- 角色 (ROLE_COMPANY)
  company_id      varchar(64),                     -- 关联企业ID
  status          int2         DEFAULT 1,          -- 账号状态 (0:禁用, 1:正常)
  create_time     timestamp    DEFAULT CURRENT_TIMESTAMP,
  update_time     timestamp    DEFAULT CURRENT_TIMESTAMP
);
```

#### biz_hr 表

```sql
CREATE TABLE biz_hr (
  hr_id           varchar(64)  PRIMARY KEY,
  user_id         varchar(64)  NOT NULL,           -- 关联用户ID
  company_id      varchar(64)  NOT NULL,           -- 关联企业ID
  name            varchar(50),                     -- HR姓名
  phone           varchar(20),                     -- 联系电话
  email           varchar(100),                    -- 邮箱
  create_time     timestamp    DEFAULT CURRENT_TIMESTAMP,
  update_time     timestamp    DEFAULT CURRENT_TIMESTAMP
);
```

### Java实现示例

```java
@RestController
@RequestMapping("/api/company")
public class CompanyRegisterController {

    @Autowired
    private RedisTemplate<String, String> redisTemplate;
    
    @Autowired
    private CompanyService companyService;
    
    @Autowired
    private UserService userService;
    
    @Autowired
    private HrService hrService;
    
    @Autowired
    private PasswordEncoder passwordEncoder;

    @PostMapping("/register")
    @Transactional(rollbackFor = Exception.class)
    public Result<Void> register(@RequestBody CompanyRegisterReq req) {
        // 1. 验证邮箱验证码
        String redisKey = "email:code:" + req.getAdminAccount();
        String cachedCode = redisTemplate.opsForValue().get(redisKey);
        
        if (cachedCode == null) {
            return Result.error("验证码已过期，请重新获取");
        }
        
        if (!cachedCode.equals(req.getEmailCode())) {
            return Result.error("验证码错误");
        }
        
        // 2. 检查邮箱是否已注册
        if (userService.existsByEmail(req.getAdminAccount())) {
            return Result.error("该邮箱已被注册");
        }
        
        // 3. 检查信用代码是否重复
        if (companyService.existsByCode(req.getCode())) {
            return Result.error("该信用代码已被注册");
        }
        
        // 4. 创建企业记录
        String companyId = UUID.randomUUID().toString().replace("-", "");
        Company company = new Company();
        company.setId(companyId);
        company.setName(req.getName());
        company.setCode(req.getCode());
        company.setIndustry(req.getIndustry());
        company.setScale(req.getScale());
        company.setLicenseUrl(req.getLicenseUrl());
        company.setContactPerson(req.getContactPerson());
        company.setContactPhone(req.getContactPhone());
        company.setStatus(0); // 待审核
        companyService.save(company);
        
        // 5. 创建HR账号
        String userId = UUID.randomUUID().toString().replace("-", "");
        User user = new User();
        user.setUserId(userId);
        user.setLoginName(req.getAdminAccount());
        user.setPassword(passwordEncoder.encode(req.getPassword()));
        user.setShowName(req.getContactPerson());
        user.setRole("ROLE_COMPANY");
        user.setCompanyId(companyId);
        user.setStatus(1); // 正常
        userService.save(user);
        
        // 6. 创建HR信息
        String hrId = UUID.randomUUID().toString().replace("-", "");
        Hr hr = new Hr();
        hr.setHrId(hrId);
        hr.setUserId(userId);
        hr.setCompanyId(companyId);
        hr.setName(req.getContactPerson());
        hr.setPhone(req.getContactPhone());
        hr.setEmail(req.getAdminAccount());
        hrService.save(hr);
        
        // 7. 删除Redis验证码（防止重复使用）
        redisTemplate.delete(redisKey);
        
        return Result.success();
    }
}

@Data
class CompanyRegisterReq {
    private String adminAccount;    // 邮箱
    private String emailCode;       // 验证码
    private String password;        // 密码
    private String name;            // 企业全称
    private String code;            // 信用代码
    private String industry;        // 行业
    private String scale;           // 规模
    private String licenseUrl;      // 营业执照
    private String contactPerson;   // 联系人
    private String contactPhone;    // 联系电话
}
```

## 错误码说明

| 错误信息 | 说明 |
|---------|------|
| 邮箱格式不正确 | 邮箱格式验证失败 |
| 该邮箱已被注册 | 邮箱已存在于系统中 |
| 发送邮件失败，请稍后重试 | 邮件服务异常 |
| 验证码已过期，请重新获取 | Redis中验证码已过期（超过5分钟） |
| 验证码错误 | 用户输入的验证码与Redis中不匹配 |
| 该信用代码已被注册 | 企业信用代码重复 |

---

## 注意事项

1. **邮箱验证码有效期**: 5分钟，过期后需要重新获取
2. **验证码使用次数**: 验证成功后立即删除，防止重复使用
3. **企业审核状态**: 
   - 注册后企业状态为待审核（status=0）
   - 管理员审核通过后改为已通过（status=1）
   - 只有审核通过的企业才能登录系统
4. **密码加密**: 使用 `BCryptPasswordEncoder` 加密存储
5. **事务处理**: 使用 `@Transactional` 确保企业、用户、HR三张表数据一致性
6. **邮件配置**: 需要在 `application.yml` 中配置邮件服务器信息
7. **Redis配置**: 需要配置Redis连接信息
8. **唯一性约束**: 
   - 邮箱（login_name）在 `sys_user` 表中唯一
   - 信用代码（code）在 `sys_company` 表中唯一
9. **角色权限**: 企业端用户角色为 `ROLE_COMPANY`
10. **联系信息**: contactPerson 和 contactPhone 既存储在企业表，也存储在HR表

## 邮件配置示例

在 `application.yml` 中添加邮件配置：

```yaml
spring:
  mail:
    host: smtp.qq.com          # QQ邮箱SMTP服务器
    port: 587                  # 端口
    username: your@qq.com      # 发件人邮箱
    password: your-auth-code   # 授权码（不是邮箱密码）
    properties:
      mail:
        smtp:
          auth: true
          starttls:
            enable: true
            required: true
```
