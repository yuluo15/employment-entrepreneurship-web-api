# 项目申请管理功能设计

## 功能概述

项目负责人可以在"我发布的项目"页面查看每个项目收到的申请，并进行审核（通过/拒绝）。

## 设计方案

### 方案选择

我们采用**独立申请列表页面**的方案：

1. 在"我的项目"列表中，每个项目卡片底部添加"查看申请"按钮
2. 点击后跳转到独立的申请列表页面
3. 申请列表页面支持查看申请详情、通过/拒绝申请

### 页面结构

```
我的项目列表页
  ├── 项目卡片1
  │   ├── 项目信息
  │   ├── [编辑] [删除] [查看申请(3)]  ← 新增按钮，显示待审核数量
  ├── 项目卡片2
  └── ...

申请列表页（新增）
  ├── 顶部导航栏（项目名称）
  ├── Tab切换（全部/待审核/已通过/已拒绝）
  ├── 申请卡片列表
  │   ├── 申请人信息（头像、姓名、学校、专业）
  │   ├── 申请理由
  │   ├── 个人技能
  │   ├── 申请时间
  │   └── 操作按钮（通过/拒绝）或状态标签
  └── 空状态提示
```

## 后端接口设计

### 1. 获取我的项目列表（需要修改）

**接口路径**: `/api/mobile/project/my/list`  
**请求方式**: `GET`  
**接口描述**: 获取当前用户发布的项目列表，需要返回每个项目的待审核申请数量

#### 请求参数
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| pageNum | Number | 是 | 页码 |
| pageSize | Number | 是 | 每页数量 |

#### 响应数据（新增字段）
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "total": 5,
    "data": [
      {
        "id": "1",
        "name": "智能校园管理系统",
        "logo": "https://example.com/logo.jpg",
        "domain": "互联网+",
        "status": 1,
        "statusText": "孵化中",
        "teamSize": 5,
        "createTime": "2024-01-15",
        "pendingApplicationCount": 3
      }
    ]
  }
}
```

#### 新增字段说明
| 字段名 | 类型 | 说明 |
|--------|------|------|
| pendingApplicationCount | Number | 待审核申请数量 |

#### SQL查询示例
```sql
SELECT 
  p.*,
  COALESCE(pa.pending_count, 0) as pending_application_count
FROM biz_project p
LEFT JOIN (
  SELECT project_id, COUNT(*) as pending_count
  FROM biz_project_application
  WHERE status = 'PENDING'
  GROUP BY project_id
) pa ON p.project_id = pa.project_id
WHERE p.user_id = #{currentUserId}
ORDER BY p.create_time DESC
LIMIT #{offset}, #{pageSize}
```

| pageSize | Number | 是 | 每页数量 |


#### 权限验证
- 只有项目负责人可以查看申请列表
- 验证：`project.user_id = currentUserId`

#### SQL查询示例
```sql
SELECT 
  pa.*,
  st.student_name as applicant_name,
  u.avatar as applicant_avatar,
  s.name as applicant_school,
  st.major_name as applicant_major
FROM biz_project_application pa
LEFT JOIN biz_student st ON pa.applicant_id = st.student_id
LEFT JOIN sys_user u ON st.user_id = u.id
LEFT JOIN sys_school s ON st.school_id = s.id
WHERE pa.project_id = #{projectId}
  AND EXISTS (
    SELECT 1 FROM biz_project p 
    WHERE p.project_id = #{projectId} 
    AND p.user_id = #{currentUserId}
  )
  AND (#{status} IS NULL OR pa.status = #{status})
ORDER BY 
  CASE pa.status 
    WHEN 'PENDING' THEN 1 
    WHEN 'APPROVED' THEN 2 
    WHEN 'REJECTED' THEN 3 
    ELSE 4 
  END,
  pa.create_time DESC
LIMIT #{offset}, #{pageSize}
```

### 3. 处理申请（通过/拒绝）（新增）

**接口路径**: `/api/mobile/project/application/{applicationId}/handle`  
**请求方式**: `PUT`  
**接口描述**: 项目负责人处理申请

#### 请求参数
```json
{
  "action": "APPROVE",
  "replyMessage": "欢迎加入我们的团队！"
}
```

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| applicationId | String | 是 | 申请ID（路径参数） |
| action | String | 是 | 操作：APPROVE=通过，REJECT=拒绝 |
| replyMessage | String | 否 | 回复消息（建议填写） |

#### 响应数据
```json
{
  "code": 200,
  "message": "处理成功",
  "data": null
}
```

#### 权限验证
```sql
-- 验证是否是项目负责人
SELECT COUNT(*) FROM biz_project_application pa
LEFT JOIN biz_project p ON pa.project_id = p.project_id
WHERE pa.id = #{applicationId}
  AND p.user_id = #{currentUserId}
```

#### 业务逻辑
1. 验证权限（是否是项目负责人）
2. 验证申请状态（只能处理待审核的申请）
3. 更新申请状态
4. 发送系统消息通知申请人

#### SQL更新示例
```sql
UPDATE biz_project_application
SET 
  status = #{status},
  reply_message = #{replyMessage},
  reply_time = CURRENT_TIMESTAMP,
  update_time = CURRENT_TIMESTAMP
WHERE id = #{applicationId}
  AND status = 'PENDING'
  AND project_id IN (
    SELECT project_id FROM biz_project WHERE user_id = #{currentUserId}
  )
```

#### 系统消息通知
```sql
INSERT INTO sys_message (id, receiver_id, title, content, type, ref_id)
VALUES (
  #{messageId},
  #{applicantId},
  '项目申请已处理',
  CONCAT('您申请加入的项目《', #{projectName}, '》已', 
    CASE WHEN #{action} = 'APPROVE' THEN '通过' ELSE '拒绝' END),
  2,
  #{applicationId}
)
```

### 4. 获取申请统计（可选）

**接口路径**: `/api/mobile/project/{projectId}/application-stats`  
**请求方式**: `GET`  
**接口描述**: 获取项目的申请统计数据

#### 响应数据
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "totalCount": 10,
    "pendingCount": 3,
    "approvedCount": 5,
    "rejectedCount": 2
  }
}
```

#### SQL查询示例
```sql
SELECT 
  COUNT(*) as total_count,
  SUM(CASE WHEN status = 'PENDING' THEN 1 ELSE 0 END) as pending_count,
  SUM(CASE WHEN status = 'APPROVED' THEN 1 ELSE 0 END) as approved_count,
  SUM(CASE WHEN status = 'REJECTED' THEN 1 ELSE 0 END) as rejected_count
FROM biz_project_application
WHERE project_id = #{projectId}
```

## 错误码说明

| 错误码 | 说明 |
|--------|------|
| 401 | 未登录或token过期 |
| 403 | 无权限（不是项目负责人） |
| 404 | 项目或申请不存在 |
| 409 | 申请状态不允许操作（已处理的申请不能再次处理） |
| 500 | 服务器内部错误 |

|------|------|------|

- 通过/拒绝申请前弹出确认对话框
- 建议填写回复消息

- 处理申请后立即更新列表
- 更新待审核数量徽章
