# 联系负责人功能说明

## 功能概述

在项目详情页面，学生可以点击"联系负责人"按钮查看项目负责人的联系方式，方便快速沟通。

## 设计方案

### 选择理由

我们采用了**显示联系方式**的方案，而不是站内消息系统，原因如下：

1. **符合创业项目特点**：创业项目需要快速、高效的沟通，电话和微信是最直接的方式
2. **开发成本低**：无需开发复杂的即时通讯系统，适合毕业设计的范围
3. **用户体验好**：一键复制手机号，方便用户通过电话、微信、短信等多种方式联系
4. **实用性强**：学生之间更习惯使用微信等社交工具沟通

### 功能流程

```
用户点击"联系负责人"
    ↓
检查负责人是否公开联系方式
    ↓
显示对话框（姓名、手机号、温馨提示）
    ↓
用户选择：
    - 点击"复制手机号" → 复制到剪贴板 → 提示"已复制"
    - 点击"关闭" → 关闭对话框
```

## 前端实现

### 1. 对话框内容

```
标题：负责人联系方式

内容：
姓名：张三
手机：13800138000

温馨提示：请礼貌沟通，说明来意

按钮：[关闭] [复制手机号]
```

### 2. 复制功能

支持两种复制方式：
- 现代浏览器：使用 `navigator.clipboard.writeText()`
- 兼容方案：使用 `document.execCommand('copy')`

### 3. 边界处理

- 如果负责人未公开手机号，显示提示"负责人未公开联系方式"
- 复制成功后显示提示"已复制到剪贴板"
- 用户点击"关闭"按钮不做任何操作

## 后端实现

### 数据库查询

在项目详情接口中，需要关联查询负责人的手机号：

```sql
SELECT 
  p.*,
  s.name as school_name,
  u.nickname as leader_name,
  u.phone as leader_phone,  -- 新增：负责人手机号
  t.name as mentor_name,
  CASE WHEN p.user_id = #{currentUserId} THEN true ELSE false END as is_owner,
  pa.status as application_status
FROM biz_project p
LEFT JOIN sys_school s ON p.school_id = s.id
LEFT JOIN sys_user u ON p.user_id = u.id
LEFT JOIN biz_teacher t ON p.mentor_id = t.teacher_id
LEFT JOIN biz_project_application pa ON pa.project_id = p.project_id 
  AND pa.applicant_id = #{currentUserId}
WHERE p.project_id = #{projectId}
```

### 接口响应

在项目详情接口的响应中添加 `leaderPhone` 字段：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "projectId": "1",
    "title": "智能校园管理系统",
    "leaderName": "张三",
    "leaderPhone": "13800138000",
    ...
  }
}
```

## 隐私保护建议

虽然我们选择了直接显示手机号的方案，但仍然可以考虑以下隐私保护措施：

### 1. 可选公开（推荐）

在项目发布时，让负责人选择是否公开联系方式：

```sql
ALTER TABLE biz_project ADD COLUMN show_contact boolean DEFAULT true;
```

如果 `show_contact = false`，则不返回手机号。

### 2. 脱敏显示（可选）

显示部分脱敏的手机号，点击"查看完整号码"后才显示完整号码：

```
显示：138****8000
点击后：13800138000
```

### 3. 查看记录（可选）

记录谁查看了联系方式，通知负责人：

```sql
CREATE TABLE biz_contact_view_log (
    id varchar(255) PRIMARY KEY,
    project_id varchar(255) NOT NULL,
    viewer_id varchar(255) NOT NULL,
    viewer_name varchar(255) NOT NULL,
    create_time timestamp DEFAULT CURRENT_TIMESTAMP
);
```

## 用户体验优化

### 1. 温馨提示

在对话框中添加温馨提示，引导用户礼貌沟通：
- "请礼貌沟通，说明来意"
- "建议先通过申请加入项目功能提交申请"

### 2. 快捷操作

- 一键复制手机号到剪贴板
- 支持点击遮罩层关闭对话框

### 3. 多渠道联系

如果后续需要扩展，可以添加更多联系方式：
- 微信号
- QQ号
- 邮箱

## 与申请功能的配合

"联系负责人"和"申请加入项目"是两个互补的功能：

| 功能 | 使用场景 | 特点 |
|------|---------|------|
| 申请加入项目 | 正式申请加入团队 | 正式、有记录、需要审核 |
| 联系负责人 | 咨询项目详情、快速沟通 | 非正式、即时、灵活 |

建议用户流程：
1. 浏览项目详情
2. 如有疑问，点击"联系负责人"咨询
3. 确定加入意向后，点击"申请加入项目"正式申请

## 总结

这个方案在**开发成本**、**用户体验**、**实用性**之间取得了很好的平衡，非常适合毕业设计项目。如果后续需要更完善的沟通功能，可以考虑集成第三方即时通讯SDK（如环信、融云等）。
