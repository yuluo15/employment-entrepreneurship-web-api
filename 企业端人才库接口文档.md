# 企业端人才库接口文档

## 📋 接口概述

人才库模块用于管理已录用和已拒绝的候选人，支持查看历史记录、统计数据、二次发放Offer等功能。

---

## 1. 获取人才库列表（分页）

### 接口信息
- **接口路径**: `GET /api/company/talent/list`
- **接口描述**: 分页查询已录用和已拒绝的人才列表
- **权限要求**: 企业HR角色

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| pageNum | number | 是 | 页码，从1开始 |
| pageSize | number | 是 | 每页条数 |
| studentName | string | 否 | 学生姓名（模糊查询） |
| status | string | 否 | 状态：OFFER=已录用/REJECTED=已拒绝 |
| jobId | string | 否 | 职位ID |

### 响应数据

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "data": [
      {
        "id": "delivery_123456",
        "studentId": "student_001",
        "studentName": "张三",
        "studentPhone": "13800138000",
        "email": "zhangsan@example.com",
        "jobId": "job_123",
        "jobName": "Java开发工程师",
        "resumeId": "resume_001",
        "school": "四川大学",
        "major": "计算机科学与技术",
        "education": "bachelor",
        "graduationYear": "2024",
        "status": "OFFER",
        "updateTime": "2026-02-10 16:30:00",
        "gender": 1
      }
    ],
    "total": 30
  }
}
```

### SQL 实现示例

```sql
SELECT 
  d.id,
  d.student_id AS "studentId",
  s.name AS "studentName",
  s.phone AS "studentPhone",
  s.email,
  d.job_id AS "jobId",
  j.job_name AS "jobName",
  d.resume_id AS "resumeId",
  r.school,
  r.major,
  r.education,
  r.graduation_year AS "graduationYear",
  d.status,
  d.update_time AS "updateTime",
  s.gender
FROM biz_job_delivery d
LEFT JOIN biz_student s ON d.student_id = s.student_id
LEFT JOIN biz_job j ON d.job_id = j.job_id
LEFT JOIN biz_resume r ON d.resume_id = r.resume_id
WHERE d.company_id = #{companyId}
  AND d.status IN ('OFFER', 'REJECTED')
  AND (#{studentName} IS NULL OR s.name LIKE CONCAT('%', #{studentName}, '%'))
  AND (#{status} IS NULL OR d.status = #{status})
  AND (#{jobId} IS NULL OR d.job_id = #{jobId})
ORDER BY d.update_time DESC
LIMIT #{pageSize} OFFSET (#{pageNum} - 1) * #{pageSize};

-- 总数查询
SELECT COUNT(*) 
FROM biz_job_delivery d
LEFT JOIN biz_student s ON d.student_id = s.student_id
WHERE d.company_id = #{companyId}
  AND d.status IN ('OFFER', 'REJECTED')
  AND (#{studentName} IS NULL OR s.name LIKE CONCAT('%', #{studentName}, '%'))
  AND (#{status} IS NULL OR d.status = #{status})
  AND (#{jobId} IS NULL OR d.job_id = #{jobId});
```

### 业务规则

1. **权限控制**: 只能查询自己公司的人才库记录
2. **状态过滤**: 只显示状态为 `OFFER` 或 `REJECTED` 的记录
3. **排序规则**: 按更新时间降序排列

## 2. 获取人才库统计

### 接口信息
- **接口路径**: `GET /api/company/talent/statistics`
- **接口描述**: 获取人才库统计数据（已录用、已拒绝、总计）
- **权限要求**: 企业HR角色

### 请求参数

无

### 响应数据

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "offerCount": 15,
    "rejectedCount": 25,
    "totalCount": 40
  }
}
```

### SQL 实现示例

```sql
SELECT 
  COUNT(CASE WHEN status = 'OFFER' THEN 1 END) AS "offerCount",
  COUNT(CASE WHEN status = 'REJECTED' THEN 1 END) AS "rejectedCount",
  COUNT(*) AS "totalCount"
FROM biz_job_delivery
WHERE company_id = #{companyId}
  AND status IN ('OFFER', 'REJECTED');
```

### 业务规则

1. **权限控制**: 只统计自己公司的数据
2. **状态范围**: 只统计 `OFFER` 和 `REJECTED` 状态的记录

## 3. 发放Offer

### 接口信息
- **接口路径**: `POST /api/company/offer/send`
- **接口描述**: 为已拒绝的候选人重新发放Offer（二次机会）
- **权限要求**: 企业HR角色

### 请求体

```json
{
  "deliveryId": "delivery_123456",
  "entryDate": "2026-03-01",
  "salary": "8000-10000",
  "notes": "欢迎加入我们团队"
}
```

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| deliveryId | string | 是 | 投递记录ID |
| entryDate | string | 是 | 入职时间 YYYY-MM-DD |
| salary | string | 是 | 薪资待遇 |
| notes | string | 否 | 备注信息 |

### 响应数据

```json
{
  "code": 200,
  "msg": "Offer发放成功",
  "data": null
}
```

### SQL 实现示例

```sql
-- 1. 更新投递状态
UPDATE biz_job_delivery
SET 
  status = 'OFFER',
  update_time = CURRENT_TIMESTAMP
WHERE id = #{deliveryId}
  AND company_id = #{companyId};

-- 2. 记录Offer信息（可选：如果有单独的Offer表）
INSERT INTO biz_offer (
  id,
  delivery_id,
  company_id,
  student_id,
  job_id,
  entry_date,
  salary,
  notes,
  status,
  create_time
) VALUES (
  #{offerId},
  #{deliveryId},
  #{companyId},
  (SELECT student_id FROM biz_job_delivery WHERE id = #{deliveryId}),
  (SELECT job_id FROM biz_job_delivery WHERE id = #{deliveryId}),
  #{entryDate},
  #{salary},
  #{notes},
  'PENDING',  -- 待学生确认
  CURRENT_TIMESTAMP
);

-- 3. 发送消息通知学生
INSERT INTO sys_message (
  id,
  receiver_id,
  title,
  content,
  type,
  is_read,
  ref_id,
  create_time
) VALUES (
  #{messageId},
  (SELECT student_id FROM biz_job_delivery WHERE id = #{deliveryId}),
  'Offer通知',
  CONCAT('恭喜您获得Offer！入职时间：', #{entryDate}, '，薪资：', #{salary}),
  'OFFER',
  0,
  #{deliveryId},
  CURRENT_TIMESTAMP
);
```

### 业务规则

1. **权限验证**: 只能为投递给自己公司的候选人发放Offer
2. **状态变更**: 投递状态从 `REJECTED` 变为 `OFFER`
3. **消息通知**: 发放Offer后自动发送消息通知学生
4. **Offer记录**: 可选择是否创建单独的Offer记录表

## 数据库表结构说明

### biz_job_delivery（投递记录表）

```sql
CREATE TABLE biz_job_delivery (
  id VARCHAR(64) PRIMARY KEY,
  student_id VARCHAR(64) NOT NULL,
  job_id VARCHAR(64) NOT NULL,
  company_id VARCHAR(64) NOT NULL,
  resume_id VARCHAR(64) NOT NULL,
  status VARCHAR(20) NOT NULL,  -- DELIVERED/INTERVIEW/OFFER/REJECTED
  reject_reason TEXT,
  create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  update_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_delivery_company ON biz_job_delivery(company_id, status);
CREATE INDEX idx_delivery_student ON biz_job_delivery(student_id);
CREATE INDEX idx_delivery_job ON biz_job_delivery(job_id);
```

**status 字段说明**：
- `DELIVERED`: 已投递（待处理）
- `INTERVIEW`: 面试中
- `OFFER`: 已录用
- `REJECTED`: 已拒绝

### biz_offer（Offer记录表）- 可选

```sql
CREATE TABLE biz_offer (
  id VARCHAR(64) PRIMARY KEY,
  delivery_id VARCHAR(64) NOT NULL,
  company_id VARCHAR(64) NOT NULL,
  student_id VARCHAR(64) NOT NULL,
  job_id VARCHAR(64) NOT NULL,
  entry_date DATE NOT NULL,  -- 入职时间
  salary VARCHAR(100) NOT NULL,  -- 薪资待遇
  notes TEXT,  -- 备注
  status VARCHAR(20) NOT NULL,  -- PENDING/ACCEPTED/REJECTED
  create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  update_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_offer_company ON biz_offer(company_id);
CREATE INDEX idx_offer_student ON biz_offer(student_id);
CREATE INDEX idx_offer_delivery ON biz_offer(delivery_id);
```

**status 字段说明**：
- `PENDING`: 待学生确认
- `ACCEPTED`: 学生已接受
- `REJECTED`: 学生已拒绝

## 业务流程说明

### 人才库流程

```
简历处理完成 → 
  ├─ 已录用 (status=OFFER) → 归档到人才库
  │                           ↓
  │                    可查看历史记录
  │
  └─ 已拒绝 (status=REJECTED) → 归档到人才库
                                ↓
                         可重新发放Offer → status=OFFER
```

### 权限控制

1. **HR身份验证**:
   ```sql
   SELECT hr_id, company_id, name
   FROM biz_hr
   WHERE user_id = #{userId} AND status = 1;
   ```

2. **操作权限**:
   - 查看人才库：只能查看自己公司的记录
   - 发放Offer：只能为投递给自己公司的候选人发放
   - 统计数据：只统计自己公司的数据

## 后端实现示例

### Controller 层

```java
package com.example.controller.company;

import com.example.service.TalentPoolService;
import com.example.vo.TalentPoolVO;
import com.example.vo.TalentStatisticsVO;
import com.example.dto.OfferDTO;
import com.example.common.Result;
import com.example.common.PageResult;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.web.bind.annotation.*;

/**
 * 人才库管理 - 企业端
 */
@RestController
@RequestMapping("/api/company/talent")
public class TalentPoolController {
    
    @Autowired
    private TalentPoolService talentPoolService;
    
    /**
     * 获取人才库列表
     */
    @GetMapping("/list")
    public Result<PageResult<TalentPoolVO>> getTalentPoolList(
        @RequestParam Integer pageNum,
        @RequestParam Integer pageSize,
        @RequestParam(required = false) String studentName,
        @RequestParam(required = false) String status,
        @RequestParam(required = false) String jobId
    ) {
        PageResult<TalentPoolVO> result = talentPoolService.getTalentPoolList(
            pageNum, pageSize, studentName, status, jobId
        );
        return Result.success(result);
    }
    
    /**
     * 获取人才库统计
     */
    @GetMapping("/statistics")
    public Result<TalentStatisticsVO> getTalentStatistics() {
        TalentStatisticsVO statistics = talentPoolService.getTalentStatistics();
        return Result.success(statistics);
    }
    
    /**
     * 发放Offer
     */
    @PostMapping("/offer/send")
    public Result<Void> sendOffer(@RequestBody OfferDTO offerDTO) {
        talentPoolService.sendOffer(offerDTO);
        return Result.success();
    }
}
```

### Service 层

```java
package com.example.service;

import com.example.vo.TalentPoolVO;
import com.example.vo.TalentStatisticsVO;
import com.example.dto.OfferDTO;
import com.example.common.PageResult;
import com.example.mapper.JobDeliveryMapper;
import com.example.util.SecurityUtils;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

import java.util.List;

@Service
public class TalentPoolService {
    
    @Autowired
    private JobDeliveryMapper jobDeliveryMapper;
    
    @Autowired
    private MessageService messageService;
    
    /**
     * 获取人才库列表
     */
    public PageResult<TalentPoolVO> getTalentPoolList(
        Integer pageNum, Integer pageSize, 
        String studentName, String status, String jobId
    ) {
        // 获取当前企业ID
        String companyId = SecurityUtils.getCurrentCompanyId();
        
        // 查询列表
        List<TalentPoolVO> list = jobDeliveryMapper.selectTalentPoolList(
            companyId, studentName, status, jobId, 
            (pageNum - 1) * pageSize, pageSize
        );
        
        // 查询总数
        Integer total = jobDeliveryMapper.countTalentPool(
            companyId, studentName, status, jobId
        );
        
        return new PageResult<>(list, total);
    }
    
    /**
     * 获取人才库统计
     */
    public TalentStatisticsVO getTalentStatistics() {
        String companyId = SecurityUtils.getCurrentCompanyId();
        return jobDeliveryMapper.selectTalentStatistics(companyId);
    }
    
    /**
     * 发放Offer
     */
    @Transactional
    public void sendOffer(OfferDTO offerDTO) {
        String companyId = SecurityUtils.getCurrentCompanyId();
        
        // 1. 验证权限
        JobDelivery delivery = jobDeliveryMapper.selectById(offerDTO.getDeliveryId());
        if (!delivery.getCompanyId().equals(companyId)) {
            throw new BusinessException("无权操作");
        }
        
        // 2. 更新投递状态
        jobDeliveryMapper.updateStatus(offerDTO.getDeliveryId(), "OFFER");
        
        // 3. 创建Offer记录（可选）
        Offer offer = new Offer();
        offer.setDeliveryId(offerDTO.getDeliveryId());
        offer.setCompanyId(companyId);
        offer.setStudentId(delivery.getStudentId());
        offer.setJobId(delivery.getJobId());
        offer.setEntryDate(offerDTO.getEntryDate());
        offer.setSalary(offerDTO.getSalary());
        offer.setNotes(offerDTO.getNotes());
        offer.setStatus("PENDING");
        offerMapper.insert(offer);
        
        // 4. 发送消息通知
        messageService.sendOfferNotification(
            delivery.getStudentId(),
            offerDTO.getEntryDate(),
            offerDTO.getSalary()
        );
    }
}
```

## 注意事项

### 1. 数据权限
- 所有操作都需要验证 `company_id`
- 确保HR只能操作自己公司的数据
- 查询时必须加上公司ID过滤条件

### 2. 状态管理
- 人才库只显示 `OFFER` 和 `REJECTED` 状态
- 发放Offer会将状态从 `REJECTED` 改为 `OFFER`
- 状态变更需要记录操作日志

### 3. 消息通知
- 发放Offer后需要发送消息通知学生
- 消息内容包含入职时间、薪资等信息
- 消息类型为 `OFFER`

### 4. 性能优化
- 列表查询添加索引
- 统计查询使用聚合函数
- 考虑添加缓存

### 5. 业务逻辑
- 只有已拒绝的候选人才能重新发放Offer
- 发放Offer需要填写入职时间和薪资
- 可以添加备注信息

## 常见问题

### Q1: 人才库列表为空？
**A**: 检查是否有状态为 `OFFER` 或 `REJECTED` 的投递记录

### Q2: 无法发放Offer？
**A**: 检查：
- 候选人状态是否为 `REJECTED`
- 是否有权限操作该投递记录
- 入职时间和薪资是否填写

### Q3: 统计数据不准确？
**A**: 检查SQL聚合查询是否正确，确保只统计 `OFFER` 和 `REJECTED` 状态
