## 🔧 核心实现

### 权限控制

所有接口都使用 `@PreAuthorize("hasRole('ROLE_COMPANY')")` 进行权限控制，确保只有企业端可以访问。

```java
@GetMapping("/list")
@PreAuthorize("hasRole('ROLE_COMPANY')")
public Result<PageResult<TalentPoolVo>> getTalentPoolList(...)
```

### 公司ID获取

通过当前登录用户获取HR信息，再获取公司ID：

```java
String userId = UserContext.getUserId();
LambdaQueryWrapper<HrEntity> hrWrapper = new LambdaQueryWrapper<>();
hrWrapper.eq(HrEntity::getUserId, userId);
HrEntity hr = hrMapper.selectOne(hrWrapper);
String companyId = hr.getCompanyId();
```

### 数据过滤

只查询状态为OFFER或REJECTED的投递记录：

```java
wrapper.eq(JobDeliveryEntity::getCompanyId, companyId);
wrapper.in(JobDeliveryEntity::getStatus, "OFFER", "REJECTED");
```

### 消息通知

发放Offer后自动发送消息通知学生：

```java
MessageEntity message = new MessageEntity();
message.setId(EntityHelper.uuid());
message.setReceiverId(delivery.getStudentId());
message.setTitle("Offer通知");
message.setContent(String.format("恭喜您获得Offer！入职时间：%s，薪资：%s。%s", 
    offerDto.getEntryDate(), 
    offerDto.getSalary(),
    StringUtils.hasText(offerDto.getNotes()) ? offerDto.getNotes() : ""));
message.setType(2);
message.setIsRead(0);
message.setRefId(offerDto.getDeliveryId());
message.setCreateTime(EntityHelper.now());
messageMapper.insert(message);
```

## ⚠️ 注意事项

### 1. 数据权限
- 所有操作都验证了公司ID
- 确保HR只能操作自己公司的数据
- 查询时必须加上公司ID过滤条件

### 2. 状态管理
- 人才库只显示OFFER和REJECTED状态
- 发放Offer会将状态从REJECTED改为OFFER
- 状态变更会更新updateTime字段

### 3. 消息通知
- 发放Offer后自动发送消息通知学生
- 消息内容包含入职时间、薪资等信息
- 消息类型为2（面试通知，可以考虑定义新类型）

### 4. 事务控制
- 发放Offer使用 `@Transactional` 注解
- 确保状态更新和消息发送的原子性

### 5. 性能优化建议
- 列表查询已按updateTime降序排列
- 可以考虑添加索引优化查询性能
- 统计查询使用了MyBatis-Plus的count方法

## 📊 业务流程

### 人才库流程

```
简历处理完成 → 
  ├─ 已录用 (status=OFFER) → 归档到人才库
  │                           ↓
  │                    可查看历史记录
  │
  └─ 已拒绝 (status=REJECTED) → 归档到人才库
                                ↓
                         可重新发放Offer → status=OFFER
                                            ↓
                                     发送消息通知学生
```
