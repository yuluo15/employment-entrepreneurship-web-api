# AI简历筛选功能 - 实现总结

## 📋 功能概述

企业端AI智能简历筛选功能,帮助HR快速找到最匹配的候选人,基于向量相似度算法自动分析职位要求和候选人简历的匹配程度。

## 🎯 功能位置说明

### ✅ 正确位置: 新简历页面 (new-resumes.vue)

AI简历筛选功能已正确放置在"新简历"页面,原因:
- **业务场景匹配**: 新简历页面展示的是状态为 `DELIVERED`(待处理)的简历
- **工作流程合理**: 这是HR需要筛选和处理的主要场景
- **提高效率**: 企业可以为特定职位快速找到最匹配的候选人
- **符合实际**: 收到简历 → AI筛选 → 安排面试/拒绝

### ❌ 不适合位置: 人才库页面 (talent-pool.vue)

人才库页面展示的是已处理的简历(已录用/已拒绝),不需要AI筛选功能:
- 人才库是归档性质的页面
- 简历已经过筛选和面试流程
- 主要用于查看历史记录和二次发放Offer

## 🔧 后端实现要点

### 1. 核心算法

```java
// 向量相似度搜索
SELECT 
    r.*,
    1 - (r.embedding <=> job_vector) as similarity
FROM biz_student_resume r
WHERE r.is_public = 1
  AND r.embedding IS NOT NULL
ORDER BY r.embedding <=> job_vector
LIMIT 20;
```

### 2. 匹配度计算

```java
// 余弦相似度转换为百分比
float similarity = calculateSimilarity(resumeVector, jobVector);
int matchScore = (int) (similarity * 100);
```

### 3. 匹配理由生成

```java
if (matchScore >= 90) {
    return "候选人的技能和经验与职位要求高度匹配";
} else if (matchScore >= 80) {
    return "候选人的专业背景与职位要求相符";
} else if (matchScore >= 70) {
    return "候选人具备职位所需的基本能力";
}
```

## 🚀 使用流程

### HR使用步骤

1. **登录企业端**
   - 进入"新简历"页面

2. **选择职位**
   - 在搜索栏选择要招聘的职位
   - 例如: "Java后端开发工程师"

3. **点击AI筛选**
   - 点击"AI智能筛选"按钮
   - 确认筛选操作

4. **查看结果**
   - 系统返回匹配度最高的20份简历
   - 按匹配度从高到低排序
   - 显示匹配理由

5. **处理简历**
   - 查看候选人详细简历
   - 安排面试
   - 或标记为不合适

## ⚠️ 注意事项

### 1. 数据准备
- 确保职位已生成向量
- 确保简历已生成向量
- 向量数据需要定期更新

### 2. 权限控制
- 只有企业端可以访问
- 只能筛选公开的简历
- 联系方式需要脱敏

### 3. 性能优化
- 限制返回数量(默认20)
- 添加缓存机制
- 异步处理大量数据

### 4. 用户体验
- 提供加载状态
- 显示筛选进度
- 空结果友好提示

## 🐛 常见问题

### Q1: 点击AI筛选没有反应?
**A**: 检查是否选择了职位,AI筛选需要先选择职位

### Q2: 筛选结果为空?
**A**: 可能原因:
- 职位没有向量
- 简历没有向量
- 匹配度都很低

### Q3: 匹配度都很低(<70)?
**A**: 可能原因:
- 职位要求过于特殊
- 简历库中没有合适候选人
- 向量生成质量问题
