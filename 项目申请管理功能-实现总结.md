## 需要完成的后端接口

### 1. 修改：获取我的项目列表

**接口路径**: `/api/mobile/project/my/list`  
**修改内容**: 返回每个项目的待审核申请数量

**SQL示例**:
```sql
SELECT 
  p.*,
  COALESCE(pa.pending_count, 0) as pending_application_count
FROM biz_project p
LEFT JOIN (
  SELECT project_id, COUNT(*) as pending_count
  FROM biz_project_application
  WHERE status = 'PENDING'
  GROUP BY project_id
) pa ON p.project_id = pa.project_id
WHERE p.user_id = #{currentUserId}
ORDER BY p.create_time DESC
```

**响应示例**:
```json
{
  "code": 200,
  "data": {
    "total": 5,
    "data": [
      {
        "id": "1",
        "name": "智能校园管理系统",
        "pendingApplicationCount": 3
      }
    ]
  }
}
```

### 2. 新增：获取项目的申请列表

**接口路径**: `/api/mobile/project/{projectId}/applications`  
**请求方式**: `GET`

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| projectId | String | 是 | 项目ID（路径参数） |
| pageNum | Number | 是 | 页码 |
| pageSize | Number | 是 | 每页数量 |
| status | String | 否 | 筛选状态：PENDING/APPROVED/REJECTED |

**权限验证**:
```sql
-- 验证是否是项目负责人
SELECT COUNT(*) FROM biz_project
WHERE project_id = #{projectId}
  AND user_id = #{currentUserId}
```

**SQL查询**:
```sql
SELECT 
  pa.*,
  st.student_name as applicant_name,
  u.avatar as applicant_avatar,
  s.name as applicant_school,
  st.major_name as applicant_major
FROM biz_project_application pa
LEFT JOIN biz_student st ON pa.applicant_id = st.student_id
LEFT JOIN sys_user u ON st.user_id = u.id
LEFT JOIN sys_school s ON st.school_id = s.id
WHERE pa.project_id = #{projectId}
  AND EXISTS (
    SELECT 1 FROM biz_project p 
    WHERE p.project_id = #{projectId} 
    AND p.user_id = #{currentUserId}
  )
  AND (#{status} IS NULL OR pa.status = #{status})
ORDER BY 
  CASE pa.status 
    WHEN 'PENDING' THEN 1 
    WHEN 'APPROVED' THEN 2 
    WHEN 'REJECTED' THEN 3 
    ELSE 4 
  END,
  pa.create_time DESC
LIMIT #{offset}, #{pageSize}
```

**响应示例**:
```json
{
  "code": 200,
  "data": {
    "total": 10,
    "data": [
      {
        "id": "app_001",
        "projectId": "1",
        "applicantId": "201",
        "applicantName": "李四",
        "applicantAvatar": "https://example.com/avatar.jpg",
        "applicantSchool": "四川大学",
        "applicantMajor": "计算机科学与技术",
        "applicationReason": "我对这个项目非常感兴趣...",
        "skills": "Vue3, TypeScript, Node.js",
        "status": "PENDING",
        "replyMessage": null,
        "createTime": "2024-01-15 10:30:00",
        "replyTime": null
      }
    ]
  }
}
```

### 3. 新增：处理申请（通过/拒绝）

**接口路径**: `/api/mobile/project/application/{applicationId}/handle`  
**请求方式**: `PUT`

**请求参数**:
```json
{
  "action": "APPROVE",
  "replyMessage": "欢迎加入我们的团队！"
}
```

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| applicationId | String | 是 | 申请ID（路径参数） |
| action | String | 是 | 操作：APPROVE=通过，REJECT=拒绝 |
| replyMessage | String | 否 | 回复消息 |

**权限验证**:
```sql
-- 验证是否是项目负责人
SELECT COUNT(*) FROM biz_project_application pa
LEFT JOIN biz_project p ON pa.project_id = p.project_id
WHERE pa.id = #{applicationId}
  AND p.user_id = #{currentUserId}
```

**SQL更新**:
```sql
UPDATE biz_project_application
SET 
  status = #{status},
  reply_message = #{replyMessage},
  reply_time = CURRENT_TIMESTAMP,
  update_time = CURRENT_TIMESTAMP
WHERE id = #{applicationId}
  AND status = 'PENDING'
  AND project_id IN (
    SELECT project_id FROM biz_project WHERE user_id = #{currentUserId}
  )
```

**系统消息通知**:
```sql
INSERT INTO sys_message (id, receiver_id, title, content, type, ref_id)
VALUES (
  #{messageId},
  #{applicantId},
  '项目申请已处理',
  CONCAT('您申请加入的项目《', #{projectName}, '》已', 
    CASE WHEN #{action} = 'APPROVE' THEN '通过' ELSE '拒绝' END),
  2,
  #{applicationId}
)
```

**响应示例**:
```json
{
  "code": 200,
  "message": "处理成功",
  "data": null
}
```

## 错误码说明

| 错误码 | 说明 |
|--------|------|
| 401 | 未登录或token过期 |
| 403 | 无权限（不是项目负责人） |
| 404 | 项目或申请不存在 |
| 409 | 申请状态不允许操作（已处理的申请不能再次处理） |
| 500 | 服务器内部错误 |

## 测试场景

### 场景1：查看申请列表

1. 登录学生账号A
2. 发布一个项目
3. 登录学生账号B
4. 申请加入项目
5. 切换回学生账号A
6. 进入"我的项目"
7. 点击"申请(1)"按钮

1. 点击"待审核"Tab，只显示待审核的申请
2. 点击"已通过"Tab，只显示已通过的申请

## 注意事项

1. **权限控制**：
   - 只有项目负责人可以查看和处理申请
   - 每次操作都要验证权限

2. **状态控制**：
   - 只有待审核的申请可以处理
   - 已处理的申请不能再次处理

3. **消息通知**：
   - 处理申请后要发送系统消息通知申请人
   - 消息内容要包含项目名称和处理结果

4. **数据一致性**：
   - 处理申请时要更新申请状态和回复时间
   - 要记录回复消息
