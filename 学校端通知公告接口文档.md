# 学校端通知公告接口文档

## 接口概述

学校端通知公告模块提供以下功能：
- 查看管理员发布的公告（只读）
- 发布、编辑、删除本校公告
- 支持草稿和发布状态管理

---

## 权限说明

### 学校端权限规则
1. **可查看**：管理员发布的公告 + 本校发布的公告
2. **可编辑/删除**：仅本校发布的公告
3. **不可操作**：管理员发布的公告（只读）

### 数据隔离
- 学校端只能操作本校发布的公告（通过 `school_id` 或 `create_by` 判断）
- 管理员公告对所有学校可见但不可编辑

---

## 1. 获取学校通知列表（分页）

### 接口信息
- **接口路径**: `GET /api/school/notice/list`
- **接口描述**: 分页查询通知列表，包含管理员发布的和本校发布的公告
- **请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| pageNum | number | 是 | 页码，从1开始 |
| pageSize | number | 是 | 每页条数 |
| noticeTitle | string | 否 | 标题（模糊查询） |
| noticeType | string | 否 | 类型：1=通知, 2=公告, 3=政策, 4=新闻 |
| source | string | 否 | 来源：admin=管理员发布, school=本校发布 |

### 响应数据

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "data": [
      {
        "noticeId": "1",
        "noticeTitle": "关于2026届毕业生就业工作的通知",
        "noticeType": "1",
        "noticeContent": "各高校...",
        "status": 1,
        "isTop": 1,
        "viewCount": 1523,
        "createBy": "admin",
        "createTime": "2026-02-01 10:00:00",
        "updateTime": "2026-02-01 10:00:00",
        "publishTime": "2026-02-01 10:30:00"
      },
      {
        "noticeId": "2",
        "noticeTitle": "本校2026年春季招聘会通知",
        "noticeType": "1",
        "noticeContent": "各学院...",
        "status": 1,
        "isTop": 0,
        "viewCount": 256,
        "createBy": "school_001",
        "createTime": "2026-02-10 14:00:00",
        "updateTime": "2026-02-10 14:00:00",
        "publishTime": "2026-02-10 14:30:00"
      }
    ],
    "total": 25
  }
}
```

### SQL 实现示例

```sql
-- 基础查询（简化版：通过 create_by 判断来源）
SELECT 
  notice_id AS "noticeId",
  notice_title AS "noticeTitle",
  notice_type AS "noticeType",
  notice_content AS "noticeContent",
  status,
  is_top AS "isTop",
  view_count AS "viewCount",
  create_by AS "createBy",
  create_time AS "createTime",
  update_time AS "updateTime",
  publish_time AS "publishTime"
FROM sys_notice
WHERE 1=1
  -- 只显示已发布的公告，或者本校的草稿
  AND (status = 1 OR create_by = #{schoolId})
  -- 只显示管理员发布的或本校发布的
  AND (create_by = 'admin' OR create_by = #{schoolId})
  -- 标题筛选
  AND (#{noticeTitle} IS NULL OR notice_title LIKE CONCAT('%', #{noticeTitle}, '%'))
  -- 类型筛选
  AND (#{noticeType} IS NULL OR notice_type = #{noticeType})
  -- 来源筛选
  AND (
    #{source} IS NULL 
    OR (#{source} = 'admin' AND create_by = 'admin')
    OR (#{source} = 'school' AND create_by = #{schoolId})
  )
ORDER BY 
  is_top DESC,
  publish_time DESC,
  create_time DESC
LIMIT #{pageSize} OFFSET (#{pageNum} - 1) * #{pageSize};

-- 总数查询
SELECT COUNT(*) 
FROM sys_notice
WHERE 1=1
  AND (status = 1 OR create_by = #{schoolId})
  AND (create_by = 'admin' OR create_by = #{schoolId})
  AND (#{noticeTitle} IS NULL OR notice_title LIKE CONCAT('%', #{noticeTitle}, '%'))
  AND (#{noticeType} IS NULL OR notice_type = #{noticeType})
  AND (
    #{source} IS NULL 
    OR (#{source} = 'admin' AND create_by = 'admin')
    OR (#{source} = 'school' AND create_by = #{schoolId})
  );
```

### 业务逻辑说明

1. **来源判断**：
   - `create_by = 'admin'` → 管理员发布
   - `create_by = #{schoolId}` → 本校发布
   
2. **可见性规则**：
   - 管理员发布的已发布公告：所有学校可见
   - 本校发布的公告：草稿和已发布都可见
   - 其他学校发布的公告：不可见

3. **前端判断逻辑**：
   ```typescript
   // 判断是否为管理员公告
   const isAdminNotice = (row) => {
     return row.createBy === 'admin' || row.createBy !== currentSchoolId
   }
   ```

## 2. 新增学校通知

### 接口信息
- **接口路径**: `POST /api/school/notice/add`
- **接口描述**: 学校发布新通知（可保存为草稿或直接发布）
- **请求体**:

```json
{
  "noticeTitle": "本校2026年春季招聘会通知",
  "noticeType": "1",
  "noticeContent": "各学院...",
  "targetAudience": "school",
  "status": 0
}
```

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| noticeTitle | string | 是 | 标题，最大200字符 |
| noticeType | string | 是 | 类型：1=通知, 2=公告, 3=政策, 4=新闻 |
| noticeContent | string | 是 | 内容 |
| targetAudience | string | 是 | 目标受众：school=本校教职工, student=本校学生 |
| status | number | 否 | 状态：0=草稿, 1=已发布，默认0 |

### 响应数据

```json
{
  "code": 200,
  "msg": "新增成功",
  "data": null
}
```

### SQL 实现示例

```sql
INSERT INTO sys_notice (
  notice_id,
  notice_title,
  notice_type,
  notice_content,
  status,
  is_top,
  view_count,
  create_by,
  create_time,
  update_time,
  publish_time
) VALUES (
  #{noticeId},           -- 使用UUID或雪花算法生成
  #{noticeTitle},
  #{noticeType},
  #{noticeContent},
  #{status},
  0,                     -- 学校发布的公告默认不置顶
  0,                     -- 初始阅读量为0
  #{schoolId},           -- 从登录用户的 owner_id 获取学校ID
  CURRENT_TIMESTAMP,
  CURRENT_TIMESTAMP,
  CASE WHEN #{status} = 1 THEN CURRENT_TIMESTAMP ELSE NULL END
);
```

### 业务逻辑说明

1. **create_by 字段**：存储学校ID（从登录用户的 `owner_id` 获取）
2. **targetAudience 字段**：
   - 当前数据库表不支持此字段，建议后续扩展
   - 临时方案：可以在 `notice_content` 开头添加标记，或使用备注字段
3. **置顶权限**：学校发布的公告不支持置顶（`is_top` 固定为 0）

## 3. 更新学校通知

### 接口信息
- **接口路径**: `PUT /api/school/notice/update`
- **接口描述**: 更新本校发布的通知
- **请求体**:

```json
{
  "noticeId": "2",
  "noticeTitle": "本校2026年春季招聘会通知（修改）",
  "noticeType": "1",
  "noticeContent": "各学院...",
  "targetAudience": "student",
  "status": 1
}
```

### 响应数据

```json
{
  "code": 200,
  "msg": "更新成功",
  "data": null
}
```

### SQL 实现示例

```sql
UPDATE sys_notice
SET 
  notice_title = #{noticeTitle},
  notice_type = #{noticeType},
  notice_content = #{noticeContent},
  status = #{status},
  update_time = CURRENT_TIMESTAMP,
  publish_time = CASE 
    WHEN #{status} = 1 AND publish_time IS NULL 
    THEN CURRENT_TIMESTAMP 
    ELSE publish_time 
  END
WHERE notice_id = #{noticeId}
  AND create_by = #{schoolId};  -- 权限校验：只能更新本校发布的公告
```

### 业务逻辑说明

1. **权限校验**：通过 `create_by = #{schoolId}` 确保只能更新本校公告
2. **返回值**：如果 SQL 影响行数为 0，说明无权限或公告不存在，返回错误

|--------|------|------|------|
| noticeId | string | 是 | 公告ID |

```json

  "code": 200,
  "msg": "发布成功",
  "data": null
}
```

```sql

  status = 1,
```

## 数据库扩展建议（可选）

### 当前方案的局限性

当前通过 `create_by` 字段判断公告来源，存在以下问题：
1. 无法区分公告的目标受众（教职工 vs 学生）
2. 无法支持多级权限管理
3. 数据查询逻辑较复杂

### 推荐的数据库扩展方案

如果需要更完善的权限管理，建议添加以下字段：

```sql
-- 扩展 sys_notice 表
ALTER TABLE sys_notice ADD COLUMN publisher_type VARCHAR(20);  -- 发布者类型：admin, school
ALTER TABLE sys_notice ADD COLUMN publisher_id VARCHAR(255);   -- 发布者ID：school_id
ALTER TABLE sys_notice ADD COLUMN target_audience VARCHAR(20); -- 目标受众：all, school, student

-- 添加索引
CREATE INDEX idx_notice_publisher ON sys_notice(publisher_type, publisher_id);
CREATE INDEX idx_notice_audience ON sys_notice(target_audience);
```

### 扩展后的权限逻辑

1. **管理员发布** (`publisher_type='admin'`):
   - `target_audience='all'` → 所有学校和学生可见
   
2. **学校发布** (`publisher_type='school'`):
   - `target_audience='school'` → 仅本校教职工可见
   - `target_audience='student'` → 本校教职工和学生可见
   - `publisher_id` 存储学校ID

3. **查询逻辑**:
   ```sql
   -- 学校端查询
   WHERE (publisher_type = 'admin' AND target_audience = 'all')
      OR (publisher_type = 'school' AND publisher_id = #{schoolId})
   
   -- 学生端查询
   WHERE (publisher_type = 'admin' AND target_audience = 'all')
      OR (publisher_type = 'school' AND publisher_id = #{schoolId} AND target_audience = 'student')
   ```

### 是否需要扩展？

- **不扩展**：适合毕业设计项目，功能简单，开发快速
- **扩展**：适合实际生产环境，权限清晰，易于维护

建议：**毕业设计阶段使用当前方案，答辩后如需优化再扩展**

## 注意事项

1. **权限控制**：所有接口都需要校验 `create_by = #{schoolId}`，防止跨校操作
2. **学校ID获取**：从登录用户的 `owner_id` 字段获取学校ID
3. **置顶功能**：学校发布的公告不支持置顶，只有管理员可以置顶
4. **响应格式**：列表接口返回的数据在 `data.data` 中，不是 `data.list`
5. **前端判断**：前端通过 `createBy` 字段判断是否为管理员公告，控制操作按钮显示
6. **目标受众**：当前版本 `targetAudience` 字段仅前端使用，后端可暂不处理，待数据库扩展后再实现
