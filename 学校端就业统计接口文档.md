# 学校端就业统计接口文档

## 概述

本接口用于学校端查看本校学生的就业统计数据，包括各学院、各专业的就业率，学生就业去向，薪资分布，行业分布等。

**与管理员端就业统计的区别**:
- 管理员端：查看所有学校的就业数据，按学校维度统计
- 学校端：只查看本校学生的就业数据，按学院、专业维度统计

---

## 接口地址

`GET /api/school/statistics/employment`

---

## 请求头

```
Authorization: Bearer {token}
```

---

## 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| graduationYear | int | 是 | 毕业年份（如：2026） |
| collegeName | string | 否 | 学院名称（用于筛选特定学院的数据） |

## 字段说明

### KPI 指标

| 字段名 | 类型 | 说明 |
|--------|------|------|
| totalGraduates | int | 毕业生总数 |
| employedCount | int | 已就业人数（employment_status='1'） |
| employmentRate | string | 就业率（百分比，如"90.0"） |
| entrepreneurshipCount | int | 创业项目总数 |
| incubatingCount | int | 孵化中的项目数（status='1'） |
| avgSalary | string | 平均薪资（元/月） |

### 学院列表 (collegeList)

返回本校所有学院名称的数组，用于前端筛选下拉框。

### 学院统计 (collegeStats)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| collegeName | string | 学院名称 |
| totalCount | int | 该学院毕业生总数 |
| employedCount | int | 该学院已就业人数 |
| employmentRate | number | 该学院就业率（百分比数值，如95.0） |

### 专业统计 (majorStats)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| majorName | string | 专业名称 |
| totalCount | int | 该专业毕业生总数 |
| employedCount | int | 该专业已就业人数 |
| employmentRate | number | 该专业就业率（百分比数值） |

**说明**: 返回就业率最高的前10个专业

### 就业去向 TOP10 (topCompanies)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| companyName | string | 公司名称 |
| count | int | 就业人数 |

**说明**: 统计本校学生就业最多的前10家公司

### 薪资分布 (salaryDistribution)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| range | string | 薪资范围（如"8-12K"） |
| count | int | 该薪资范围的人数 |

**薪资范围分类**:
- 5K以下
- 5-8K
- 8-12K
- 12-20K
- 20K以上

### 行业分布 (industryDistribution)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| industry | string | 行业名称 |
| count | int | 该行业就业人数 |

**说明**: 返回就业人数最多的前8个行业

### 月度趋势 (monthlyTrend)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| month | string | 月份（格式：YYYY-MM） |
| count | int | 该月新增就业人数 |

**说明**: 返回最近6个月的就业趋势数据

## 后端实现参考

### SQL查询示例

```sql
-- 1. KPI指标查询
SELECT 
  COUNT(*) AS total_graduates,
  SUM(CASE WHEN COALESCE(s.employment_status, '0') = '1' THEN 1 ELSE 0 END) AS employed_count,
  ROUND(
    (SUM(CASE WHEN COALESCE(s.employment_status, '0') = '1' THEN 1 ELSE 0 END)::NUMERIC / 
     NULLIF(COUNT(*), 0)) * 100, 
    1
  ) AS employment_rate,
  (SELECT COUNT(*) FROM biz_project WHERE school_id = #{schoolId}) AS entrepreneurship_count,
  (SELECT COUNT(*) FROM biz_project WHERE school_id = #{schoolId} AND status = '1') AS incubating_count,
  COALESCE(
    ROUND(AVG(
      CASE 
        WHEN j.salary_range ~ '^[0-9]+-[0-9]+K$' THEN 
          ((CAST(SPLIT_PART(j.salary_range, '-', 1) AS NUMERIC) + 
            CAST(REPLACE(SPLIT_PART(j.salary_range, '-', 2), 'K', '') AS NUMERIC)) / 2) * 1000
        ELSE NULL
      END
    )), 0
  ) AS avg_salary
FROM biz_student s
LEFT JOIN biz_job_delivery d ON s.student_id = d.student_id AND d.status = 'OFFER'
LEFT JOIN biz_job j ON d.job_id = j.job_id
WHERE s.school_id = #{schoolId}
  AND s.graduation_year = #{graduationYear}
  AND (#{collegeName} IS NULL OR s.college_name = #{collegeName});

-- 2. 获取学院列表
SELECT DISTINCT college_name
FROM biz_student
WHERE school_id = #{schoolId}
  AND graduation_year = #{graduationYear}
  AND college_name IS NOT NULL
ORDER BY college_name;

-- 3. 各学院就业率统计
SELECT 
  s.college_name,
  COUNT(*) AS total_count,
  SUM(CASE WHEN COALESCE(s.employment_status, '0') = '1' THEN 1 ELSE 0 END) AS employed_count,
  ROUND(
    (SUM(CASE WHEN COALESCE(s.employment_status, '0') = '1' THEN 1 ELSE 0 END)::NUMERIC / 
     NULLIF(COUNT(*), 0)) * 100, 
    1
  ) AS employment_rate
FROM biz_student s
WHERE s.school_id = #{schoolId}
  AND s.graduation_year = #{graduationYear}
  AND (#{collegeName} IS NULL OR s.college_name = #{collegeName})
  AND s.college_name IS NOT NULL
GROUP BY s.college_name
ORDER BY employment_rate DESC;

-- 4. 各专业就业率统计 TOP10
SELECT 
  s.major_name,
  COUNT(*) AS total_count,
  SUM(CASE WHEN COALESCE(s.employment_status, '0') = '1' THEN 1 ELSE 0 END) AS employed_count,
  ROUND(
    (SUM(CASE WHEN COALESCE(s.employment_status, '0') = '1' THEN 1 ELSE 0 END)::NUMERIC / 
     NULLIF(COUNT(*), 0)) * 100, 
    1
  ) AS employment_rate
FROM biz_student s
WHERE s.school_id = #{schoolId}
  AND s.graduation_year = #{graduationYear}
  AND (#{collegeName} IS NULL OR s.college_name = #{collegeName})
  AND s.major_name IS NOT NULL
GROUP BY s.major_name
ORDER BY employment_rate DESC
LIMIT 10;

-- 5. 就业去向 TOP10（学生就业最多的公司）
SELECT 
  c.name AS company_name,
  COUNT(*) AS count
FROM biz_student s
INNER JOIN biz_job_delivery d ON s.student_id = d.student_id AND d.status = 'OFFER'
INNER JOIN sys_company c ON d.company_id = c.id
WHERE s.school_id = #{schoolId}
  AND s.graduation_year = #{graduationYear}
  AND (#{collegeName} IS NULL OR s.college_name = #{collegeName})
GROUP BY c.name
ORDER BY count DESC
LIMIT 10;

-- 6. 薪资分布统计
SELECT 
  CASE 
    WHEN j.salary_range ~ '^[0-9]+-[0-9]+K$' THEN
      CASE 
        WHEN CAST(SPLIT_PART(j.salary_range, '-', 1) AS NUMERIC) < 5 THEN '5K以下'
        WHEN CAST(SPLIT_PART(j.salary_range, '-', 1) AS NUMERIC) >= 5 
         AND CAST(REPLACE(SPLIT_PART(j.salary_range, '-', 2), 'K', '') AS NUMERIC) <= 8 THEN '5-8K'
        WHEN CAST(SPLIT_PART(j.salary_range, '-', 1) AS NUMERIC) >= 8 
         AND CAST(REPLACE(SPLIT_PART(j.salary_range, '-', 2), 'K', '') AS NUMERIC) <= 12 THEN '8-12K'
        WHEN CAST(SPLIT_PART(j.salary_range, '-', 1) AS NUMERIC) >= 12 
         AND CAST(REPLACE(SPLIT_PART(j.salary_range, '-', 2), 'K', '') AS NUMERIC) <= 20 THEN '12-20K'
        ELSE '20K以上'
      END
    ELSE '未知'
  END AS range,
  COUNT(*) AS count
FROM biz_student s
INNER JOIN biz_job_delivery d ON s.student_id = d.student_id AND d.status = 'OFFER'
INNER JOIN biz_job j ON d.job_id = j.job_id
WHERE s.school_id = #{schoolId}
  AND s.graduation_year = #{graduationYear}
  AND (#{collegeName} IS NULL OR s.college_name = #{collegeName})
  AND j.salary_range IS NOT NULL
GROUP BY range
ORDER BY 
  CASE range
    WHEN '5K以下' THEN 1
    WHEN '5-8K' THEN 2
    WHEN '8-12K' THEN 3
    WHEN '12-20K' THEN 4
    WHEN '20K以上' THEN 5
    ELSE 6
  END;

-- 7. 行业分布统计 TOP8
SELECT 
  c.industry,
  COUNT(*) AS count
FROM biz_student s
INNER JOIN biz_job_delivery d ON s.student_id = d.student_id AND d.status = 'OFFER'
INNER JOIN sys_company c ON d.company_id = c.id
WHERE s.school_id = #{schoolId}
  AND s.graduation_year = #{graduationYear}
  AND (#{collegeName} IS NULL OR s.college_name = #{collegeName})
  AND c.industry IS NOT NULL
GROUP BY c.industry
ORDER BY count DESC
LIMIT 8;

-- 8. 月度就业趋势（最近6个月）
SELECT 
  TO_CHAR(d.update_time, 'YYYY-MM') AS month,
  COUNT(*) AS count
FROM biz_student s
INNER JOIN biz_job_delivery d ON s.student_id = d.student_id AND d.status = 'OFFER'
WHERE s.school_id = #{schoolId}
  AND s.graduation_year = #{graduationYear}
  AND (#{collegeName} IS NULL OR s.college_name = #{collegeName})
  AND d.update_time >= CURRENT_DATE - INTERVAL '6 months'
GROUP BY TO_CHAR(d.update_time, 'YYYY-MM')
ORDER BY month ASC;
```

### Java Controller示例

```java
@RestController
@RequestMapping("/api/school/statistics")
public class SchoolStatisticsController {

    @Autowired
    private SchoolStatisticsService statisticsService;

    @GetMapping("/employment")
    public Result<EmploymentStatsVO> getEmploymentStats(
            @RequestParam Integer graduationYear,
            @RequestParam(required = false) String collegeName) {
        
        String schoolId = SecurityUtils.getSchoolId();
        
        EmploymentStatsVO stats = statisticsService.getEmploymentStats(
            schoolId, graduationYear, collegeName
        );
        
        return Result.success(stats);
    }
}
```

## 注意事项

1. **权限控制**: 学校端只能查看本校（school_id）的就业数据
2. **数据来源**: 
   - 就业状态从 `biz_student.employment_status` 字段获取
   - 就业详情从 `biz_job_delivery` 表关联获取（status='OFFER'）
   - 创业项目从 `biz_project` 表统计
3. **学院筛选**: 当传入 `collegeName` 参数时，所有统计数据都只统计该学院的数据
4. **薪资计算**: 
   - 从 `biz_job.salary_range` 字段解析薪资（格式如"8-12K"）
   - 平均薪资 = (最低薪资 + 最高薪资) / 2
5. **就业率计算**: 就业率 = 已就业人数 / 毕业生总数 × 100
6. **数据排序**: 
   - 学院/专业按就业率降序排列
   - 公司/行业按就业人数降序排列
7. **TOP限制**: 
   - 专业统计返回前10名
   - 公司统计返回前10名
   - 行业统计返回前8名
8. **月度趋势**: 统计最近6个月的数据，按月份升序排列
9. **空值处理**: 对于 NULL 值要做适当处理，避免统计错误
10. **性能优化**: 建议在 `biz_student` 表的 `school_id`, `graduation_year`, `college_name`, `major_name` 字段上建立索引

## 与管理员端就业统计的对比

| 维度 | 管理员端 | 学校端 |
|------|---------|--------|
| 数据范围 | 所有学校 | 仅本校 |
| 主要统计维度 | 按学校统计 | 按学院、专业统计 |
| 就业去向 | 不统计 | 统计TOP10公司 |
| 学院筛选 | 无 | 支持按学院筛选 |
| 红绿灯监管 | 有（低于85%标红） | 无 |
| 创业统计 | 项目总数、审核状态 | 项目总数、孵化中数量 |
