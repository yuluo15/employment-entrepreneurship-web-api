# 项目详情申请状态查询 - 后端实现指南

## 问题描述

用户申请加入项目后，再次查看项目详情时，"申请加入项目"按钮应该根据申请状态变化：
- 未申请：显示"申请加入项目"（可点击）
- 待审核：显示"申请审核中"（不可点击，橙色）
- 已通过：显示"已加入项目"（不可点击，绿色）
- 已拒绝：显示"申请未通过"（不可点击，红色）

## 核心问题

后端需要在项目详情接口中返回当前用户对该项目的申请状态。

## 后端实现方案

### 1. SQL查询（推荐方案）

#### MyBatis XML 实现

```xml
<select id="getProjectDetail" resultType="ProjectDetailVO">
    SELECT 
        p.project_id,
        p.project_name as title,
        p.logo,
        p.slogan,
        p.description,
        p.needs,
        p.team_size,
        p.status,
        p.domain,
        p.user_id as leader_id,
        
        -- 学校信息
        s.name as school_name,
        
        -- 负责人信息
        u.nickname as leader_name,
        u.phone as leader_phone,
        
        -- 导师信息
        t.name as mentor_name,
        
        -- 是否是项目负责人
        CASE WHEN p.user_id = #{currentUserId} THEN true ELSE false END as is_owner,
        
        -- 当前用户的申请状态（关键）
        pa.status as application_status
        
    FROM biz_project p
    LEFT JOIN sys_school s ON p.school_id = s.id
    LEFT JOIN sys_user u ON p.user_id = u.id
    LEFT JOIN biz_teacher t ON p.mentor_id = t.teacher_id
    
    -- 关键：关联申请表，只查询当前用户的申请记录
    LEFT JOIN biz_project_application pa 
        ON pa.project_id = p.project_id 
        AND pa.applicant_id = #{currentUserId}
    
    WHERE p.project_id = #{projectId}
</select>
```

#### 关键点说明

1. **LEFT JOIN 条件**：
   ```sql
   LEFT JOIN biz_project_application pa 
       ON pa.project_id = p.project_id 
       AND pa.applicant_id = #{currentUserId}
   ```
   - 必须在 `ON` 条件中加上 `pa.applicant_id = #{currentUserId}`
   - 不能放在 `WHERE` 条件中，否则会过滤掉没有申请记录的项目

2. **返回值**：
   - 如果用户没有申请过：`application_status = null`
   - 如果用户申请过：`application_status = 'PENDING'/'APPROVED'/'REJECTED'/'CANCELLED'`

3. **currentUserId 获取**：
   - 从 JWT token 中解析
   - 或从 Spring Security 的 SecurityContextHolder 中获取

### 2. Java Service 层实现

#### 方案A：直接在 SQL 中查询（推荐）

```java
@Service
public class ProjectServiceImpl implements ProjectService {
    
    @Autowired
    private ProjectMapper projectMapper;
    
    @Override
    public ProjectDetailVO getProjectDetail(String projectId) {
        // 1. 获取当前登录用户ID
        String currentUserId = SecurityUtils.getUserId();
        
        // 2. 查询项目详情（包含申请状态）
        ProjectDetailVO detail = projectMapper.getProjectDetail(projectId, currentUserId);
        
        if (detail == null) {
            throw new BusinessException("项目不存在");
        }
        
        // 3. 处理其他业务逻辑（如收藏状态）
        // detail.setIsCollected(checkIfCollected(projectId, currentUserId));
        
        return detail;
    }
}
```

#### 方案B：分两次查询（不推荐，但更灵活）

```java
@Service
public class ProjectServiceImpl implements ProjectService {
    
    @Autowired
    private ProjectMapper projectMapper;
    
    @Autowired
    private ProjectApplicationMapper applicationMapper;
    
    @Override
    public ProjectDetailVO getProjectDetail(String projectId) {
        // 1. 获取当前登录用户ID
        String currentUserId = SecurityUtils.getUserId();
        
        // 2. 查询项目基本信息
        ProjectDetailVO detail = projectMapper.getProjectDetail(projectId);
        
        if (detail == null) {
            throw new BusinessException("项目不存在");
        }
        
        // 3. 判断是否是项目负责人
        detail.setIsOwner(detail.getLeaderId().equals(currentUserId));
        
        // 4. 查询申请状态
        ProjectApplication application = applicationMapper.selectOne(
            new LambdaQueryWrapper<ProjectApplication>()
                .eq(ProjectApplication::getProjectId, projectId)
                .eq(ProjectApplication::getApplicantId, currentUserId)
        );
        
        if (application != null) {
            detail.setApplicationStatus(application.getStatus());
        } else {
            detail.setApplicationStatus(null);
        }
        
        return detail;
    }
}
```

### 3. Controller 层实现

```java
@RestController
@RequestMapping("/api/mobile/project")
public class MobileProjectController {
    
    @Autowired
    private ProjectService projectService;
    
    /**
     * 获取项目详情
     * 
     * @param id 项目ID
     * @return 项目详情（包含申请状态）
     */
    @GetMapping("/detail/{id}")
    public Result<ProjectDetailVO> getProjectDetail(@PathVariable String id) {
        ProjectDetailVO detail = projectService.getProjectDetail(id);
        return Result.success(detail);
    }
}
```

## 常见问题

### Q1: 为什么 LEFT JOIN 的条件要放在 ON 中，而不是 WHERE 中？

**错误写法**：
```sql
LEFT JOIN biz_project_application pa ON pa.project_id = p.project_id
WHERE pa.applicant_id = #{currentUserId}
```

**问题**：如果用户没有申请过该项目，`pa.applicant_id` 为 `NULL`，WHERE 条件会过滤掉这条记录，导致查询不到项目。

**正确写法**：
```sql
LEFT JOIN biz_project_application pa 
    ON pa.project_id = p.project_id 
    AND pa.applicant_id = #{currentUserId}
```

**原因**：ON 条件在 JOIN 时执行，WHERE 条件在 JOIN 后执行。

---

### Q2: 如果用户申请了多次（虽然有唯一索引），如何处理？

虽然数据库有唯一索引 `uk_project_applicant (project_id, applicant_id)`，但为了保险，可以在 SQL 中加上排序和限制：

```sql
LEFT JOIN (
    SELECT project_id, applicant_id, status
    FROM biz_project_application
    WHERE applicant_id = #{currentUserId}
    ORDER BY create_time DESC
    LIMIT 1
) pa ON pa.project_id = p.project_id
```

### Q3: 如何获取当前登录用户ID？

#### Spring Security 方式
```java
public class SecurityUtils {
    public static String getUserId() {
        Authentication authentication = SecurityContextHolder.getContext().getAuthentication();
        if (authentication != null && authentication.getPrincipal() instanceof UserDetails) {
            UserDetails userDetails = (UserDetails) authentication.getPrincipal();
            // 假设 UserDetails 实现类中有 getUserId 方法
            return ((CustomUserDetails) userDetails).getUserId();
        }
        throw new BusinessException("未登录");
    }
}
```

#### JWT Token 方式
```java
public class JwtUtils {
    public static String getUserIdFromToken(String token) {
        Claims claims = Jwts.parser()
            .setSigningKey(SECRET_KEY)
            .parseClaimsJws(token)
            .getBody();
        return claims.getSubject(); // 或 claims.get("userId", String.class)
    }
}
```

## 总结

### 核心要点

1. **SQL 关键**：在 LEFT JOIN 的 ON 条件中加上 `pa.applicant_id = #{currentUserId}`
2. **返回值**：`applicationStatus` 为 `null` 表示未申请，否则为具体状态
3. **前端处理**：根据 `isOwner` 和 `applicationStatus` 决定按钮状态
