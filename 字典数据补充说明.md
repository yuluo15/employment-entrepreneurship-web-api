## ⚠️ 重要说明

### 1. 数据一致性
- 代码中使用的状态值必须与字典数据保持一致
- 例如：面试方式 1=现场、2=视频、3=电话

### 2. 字典值格式
- **数字类型**: 用于数据库存储的状态字段（如 type=1, status=0）
- **字符串类型**: 用于前后端交互的枚举值（如 "ONSITE", "DELIVERED"）

### 3. 前端展示
- 前端应该通过字典接口获取标签文本
- 不要在前端硬编码状态文本

## 🔧 后续优化建议

### 1. 字典管理接口
建议实现字典管理的CRUD接口，方便动态维护字典数据：
- 获取字典类型列表
- 根据字典类型获取字典数据

建议使用Redis缓存字典数据，提高查询性能：

```java
public List<DictDataVo> getDictData(String dictType) {
    // ...
}
```

### 3. 枚举类优化
建议将硬编码的状态值改为使用字典：
```java
// 不推荐
interview.setType(1);

// 推荐
String typeValue = dictService.getDictValue("sys_interview_type", "ONSITE");
interview.setType(Integer.parseInt(typeValue));
```

建议封装字典选择组件，自动从后端获取字典数据：

## 📞 问题排查

### 问题1: 字典数据不显示
**原因**: 字典数据的 status 字段为 0（禁用状态）
**解决**: 确保 status=1（启用状态）

### 问题2: 字典值不匹配
**原因**: 代码中使用的值与字典数据不一致
**解决**: 检查代码中的硬编码值，确保与字典数据一致

### 问题3: 前端显示字典编码而非标签
**原因**: 前端没有正确调用字典转换接口
**解决**: 使用字典服务将编码转换为标签文本
