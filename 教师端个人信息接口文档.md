# 教师端个人信息接口文档

## 1. 获取教师详细信息

### 接口信息
- **接口路径**: `/api/teacher/info`
- **请求方式**: `GET`
- **接口描述**: 获取当前登录教师的详细信息，包括教师特有的职称、学院、专业领域等信息

### 请求参数
无（从token中获取教师ID）

### 请求头
```
Authorization: Bearer {token}
```

### 响应数据
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "teacherId": "5",
    "userId": "5",
    "name": "赵六",
    "avatar": "https://example.com/avatar.jpg",
    "employeeNo": "T20230001",
    "schoolId": "1",
    "schoolName": "四川大学",
    "collegeName": "计算机学院",
    "title": "教授",
    "expertise": "人工智能、大数据",
    "phone": "138****8888",
    "email": "zhao@scu.edu.cn",
    "guidanceCount": 45,
    "projectCount": 12,
    "ratingScore": 4.8,
    "status": 1,
    "createTime": "2023-01-01 10:00:00"
  }
}
```

### 响应字段说明
| 字段名 | 类型 | 说明 |
|--------|------|------|
| teacherId | String | 教师ID |
| userId | String | 关联的用户ID |
| name | String | 教师姓名 |
| avatar | String | 头像URL |
| employeeNo | String | 工号 |
| schoolId | String | 学校ID |
| schoolName | String | 学校名称 |
| collegeName | String | 学院名称 |
| title | String | 职称（教授、副教授、讲师等） |
| expertise | String | 专业领域 |
| phone | String | 手机号（脱敏显示） |
| email | String | 邮箱 |
| guidanceCount | Number | 总指导次数 |
| projectCount | Number | 指导项目数 |
| ratingScore | Number | 平均评分（满分5分） |
| status | Number | 状态：0=禁用，1=正常 |
| createTime | String | 创建时间 |

### 业务逻辑说明
1. 从token中解析出用户ID
2. 查询教师基本信息
3. 统计指导次数、指导项目数、平均评分
4. 手机号脱敏处理（中间4位显示为*）

### 数据库查询示例
```sql
-- 1. 查询教师基本信息
SELECT 
  t.teacher_id,
  t.user_id,
  t.name,
  u.avatar,
  t.employee_no,
  t.school_id,
  s.school_name,
  t.college_name,
  t.title,
  t.expertise,
  u.phone,
  u.email,
  t.status,
  t.create_time
FROM teacher t
LEFT JOIN sys_user u ON t.user_id = u.id
LEFT JOIN school s ON t.school_id = s.school_id
WHERE t.user_id = #{userId}

-- 2. 统计指导次数
SELECT COUNT(*) as guidance_count
FROM guidance
WHERE teacher_id = #{teacherId}

-- 3. 统计指导项目数
SELECT COUNT(DISTINCT project_id) as project_count
FROM guidance
WHERE teacher_id = #{teacherId}

-- 4. 计算平均评分
SELECT AVG(rating) as rating_score
FROM guidance
WHERE teacher_id = #{teacherId} AND rating IS NOT NULL
```

### 错误码说明
| 错误码 | 说明 |
|--------|------|
| 401 | 未登录或token过期 |
| 403 | 无权限访问（非教师角色） |
| 404 | 教师信息不存在 |
| 500 | 服务器内部错误 |

### 注意事项
1. 该接口需要验证用户角色为教师（role = 'ROLE_TEACHER'）
2. 手机号需要脱敏处理（中间4位显示为*）
3. 评分保留一位小数
4. 如果教师信息不存在，返回404错误

### 前端调用示例
```typescript
import { getTeacherInfo } from '@/api/teacher'

const loadTeacherInfo = async () => {
  try {
    const res = await getTeacherInfo()
    teacherInfo.value = res.data
  } catch (error) {
    console.error('加载教师信息失败', error)
  }
}
```

## 数据库表结构参考

### teacher 表
```sql
CREATE TABLE teacher (
  teacher_id VARCHAR(50) PRIMARY KEY COMMENT '教师ID',
  user_id VARCHAR(50) NOT NULL COMMENT '关联的用户ID',
  name VARCHAR(50) NOT NULL COMMENT '教师姓名',
  employee_no VARCHAR(50) COMMENT '工号',
  school_id VARCHAR(50) NOT NULL COMMENT '学校ID',
  college_name VARCHAR(100) COMMENT '学院名称',
  title VARCHAR(50) COMMENT '职称',
  expertise VARCHAR(200) COMMENT '专业领域',
  status TINYINT DEFAULT 1 COMMENT '状态：0=禁用，1=正常',
  create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  FOREIGN KEY (user_id) REFERENCES sys_user(id),
  FOREIGN KEY (school_id) REFERENCES school(school_id)
) COMMENT='教师表';
```

### guidance 表
```sql
CREATE TABLE guidance (
  guidance_id VARCHAR(50) PRIMARY KEY COMMENT '指导记录ID',
  project_id VARCHAR(50) NOT NULL COMMENT '项目ID',
  student_id VARCHAR(50) NOT NULL COMMENT '学生ID',
  teacher_id VARCHAR(50) NOT NULL COMMENT '教师ID',
  content TEXT NOT NULL COMMENT '指导内容',
  rating DECIMAL(3,1) COMMENT '评分（1-5分）',
  create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '指导时间',
  FOREIGN KEY (project_id) REFERENCES project(project_id),
  FOREIGN KEY (student_id) REFERENCES sys_user(id),
  FOREIGN KEY (teacher_id) REFERENCES teacher(teacher_id)
) COMMENT='指导记录表';
```
