# 学校端创业项目接口文档

## 1. 获取项目列表

**接口地址**: `GET /api/school/project/list`

**请求头**:
```
Authorization: Bearer {token}
```

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| pageNum | int | 是 | 页码 |
| pageSize | int | 是 | 每页数量 |
| projectName | string | 否 | 项目名称（模糊查询） |
| studentName | string | 否 | 学生姓名（模糊查询） |
| domain | string | 否 | 项目领域（模糊查询） |
| status | string | 否 | 项目状态（0=待审核, 1=孵化中, 2=已驳回, 3=已落地） |

**响应示例**:
```json
{
  "message": "success",
  "data": {
    "total": 80,
    "data": [
      {
        "projectId": "proj001",
        "projectName": "智能校园管理系统",
        "logo": "https://example.com/logo1.png",
        "studentName": "张三",
        "studentNo": "2021001",
        "domain": "互联网",
        "teamSize": 5,
        "jobsCreated": 3,
        "status": "0",
        "createTime": "2026-02-01 10:00:00"
      },
      {
        "projectId": "proj002",
        "projectName": "在线教育平台",
        "logo": "https://example.com/logo2.png",
        "studentName": "李四",
        "studentNo": "2021002",
        "domain": "教育",
        "teamSize": 8,
        "jobsCreated": 5,
        "status": "1",
        "createTime": "2026-01-15 14:30:00"
      }
    ]
  }
}
```

**字段说明**:
- `projectId`: 项目ID
- `projectName`: 项目名称
- `logo`: 项目Logo图片URL
- `studentName`: 学生姓名（项目创建者）
- `studentNo`: 学号
- `domain`: 项目领域
- `teamSize`: 团队规模（人数）
- `jobsCreated`: 创造岗位数量
- `status`: 项目状态
  - `0` = 待审核（学生刚发布，等待学校审核）
  - `1` = 孵化中（审核通过，项目正在孵化）
  - `2` = 已驳回（审核不通过）
  - `3` = 已落地（项目成功落地）
- `createTime`: 创建时间

## 3. 审核项目

**接口地址**: `PUT /api/school/project/audit`

**请求头**:
```
Authorization: Bearer {token}
Content-Type: application/json
```

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| projectId | string | 是 | 项目ID |
| status | string | 是 | 审核结果（1=通过, 2=驳回） |
| auditReason | string | 否 | 驳回原因（status=2时必填） |

**请求示例**（通过）:
```json
{
  "projectId": "proj001",
  "status": "1",
  "auditReason": ""
}
```

**请求示例**（驳回）:
```json
{
  "projectId": "proj001",
  "status": "2",
  "auditReason": "项目描述不够详细，缺少市场分析和可行性论证"
}
```

**响应示例**:
```json
{
  "message": "审核成功"
}
```

**业务规则**:
1. 只有状态为 `0`（待审核）的项目才能进行审核
2. 审核通过后，项目状态变为 `1`（孵化中）
3. 审核驳回后，项目状态变为 `2`（已驳回），需要填写驳回原因
4. 审核后记录审核时间 `audit_time`
5. 审核操作会发送系统消息通知学生

SELECT

  p.project_name,

### Java Controller示例

```java
@RestController
@RequestMapping("/api/school/project")
public class SchoolProjectController {

    @Autowired
    private SchoolProjectService projectService;

    @GetMapping("/list")
    public Result<PageResult<ProjectVO>> getProjectList(ProjectQuery query) {
        // 从token获取当前学校ID
        String schoolId = SecurityUtils.getSchoolId();
        query.setSchoolId(schoolId);
        
        PageResult<ProjectVO> result = projectService.getProjectList(query);
        return Result.success(result);
    }

    @GetMapping("/detail/{projectId}")
    public Result<ProjectDetailVO> getProjectDetail(@PathVariable String projectId) {
        String schoolId = SecurityUtils.getSchoolId();
        ProjectDetailVO detail = projectService.getProjectDetail(projectId, schoolId);
        
        if (detail == null) {
            return Result.error("项目不存在或无权访问");
        }
        
        return Result.success(detail);
    }

    @PutMapping("/audit")
    public Result<Void> auditProject(@RequestBody AuditProjectDTO dto) {
        // 参数校验
        if (!"1".equals(dto.getStatus()) && !"2".equals(dto.getStatus())) {
            return Result.error("审核状态参数错误");
        }
        
        // 驳回时必须填写原因
        if ("2".equals(dto.getStatus()) && StringUtils.isBlank(dto.getAuditReason())) {
            return Result.error("驳回时必须填写驳回原因");
        }
        
        String schoolId = SecurityUtils.getSchoolId();
        String userId = SecurityUtils.getUserId();
        
        boolean success = projectService.auditProject(dto, schoolId, userId);
        
        if (!success) {
            return Result.error("审核失败，项目不存在或已审核");
        }
        
        return Result.success("审核成功");
    }
}
```

### Service实现示例

```java
@Service
public class SchoolProjectServiceImpl implements SchoolProjectService {
    
    @Autowired
    private BizProjectMapper projectMapper;
    
    @Autowired
    private SysMessageMapper messageMapper;
    
    @Override
    public PageResult<ProjectVO> getProjectList(ProjectQuery query) {
        // 计算分页偏移量
        int offset = (query.getPageNum() - 1) * query.getPageSize();
        query.setOffset(offset);
        
        // 查询列表
        List<ProjectVO> list = projectMapper.selectProjectList(query);
        
        // 查询总数
        int total = projectMapper.countProjectList(query);
        
        return new PageResult<>(list, total);
    }
    
    @Override
    public ProjectDetailVO getProjectDetail(String projectId, String schoolId) {
        return projectMapper.selectProjectDetail(projectId, schoolId);
    }
    
    @Transactional(rollbackFor = Exception.class)
    @Override
    public boolean auditProject(AuditProjectDTO dto, String schoolId, String auditorId) {
        // 更新项目状态
        int rows = projectMapper.auditProject(dto.getProjectId(), dto.getStatus(), 
            dto.getAuditReason(), schoolId);
        
        if (rows == 0) {
            return false;
        }
        
        // 查询项目信息，用于发送通知
        BizProject project = projectMapper.selectById(dto.getProjectId());
        
        // 发送系统消息通知学生
        SysMessage message = new SysMessage();
        message.setId(UUID.randomUUID().toString().replace("-", ""));
        message.setReceiverId(project.getUserId());
        message.setType(1); // 系统消息
        message.setIsRead(0);
        message.setRefId(dto.getProjectId());
        message.setCreateTime(new Date());
        
        if ("1".equals(dto.getStatus())) {
            message.setTitle("项目审核通过");
            message.setContent("您的创业项目《" + project.getProjectName() + "》已通过审核，现在可以开始孵化了！");
        } else {
            message.setTitle("项目审核未通过");
            message.setContent("您的创业项目《" + project.getProjectName() + "》审核未通过。驳回原因：" + dto.getAuditReason());
        }
        
        messageMapper.insert(message);
        
        return true;
    }
}
```

## 注意事项

1. **权限控制**: 学校端只能查看和审核本校学生的项目，需要在SQL中加上 `school_id` 过滤条件
2. **数据隔离**: 从token中获取当前登录用户所属的学校ID，确保数据隔离
3. **项目状态流转**:
   - 学生创建项目 → status = '0'（待审核）
   - 学校审核通过 → status = '1'（孵化中）
   - 学校审核驳回 → status = '2'（已驳回）
   - 项目成功落地 → status = '3'（已落地，可能由学生或学校更新）
4. **审核限制**: 只有状态为 `0`（待审核）的项目才能进行审核操作
5. **驳回原因**: 驳回时必须填写驳回原因，方便学生了解问题并改进
6. **消息通知**: 审核后需要发送系统消息通知学生审核结果
7. **关联查询**: 需要关联 `biz_project` 和 `biz_student` 表获取学生信息
8. **图片字段**: `logo` 字段存储图片URL，前端需要处理图片显示和预览
9. **统计字段**: 
   - `team_size`: 团队规模
   - `jobs_created`: 创造岗位数量
   - 这些字段可用于后续的统计分析
10. **导师信息**: 
    - `mentor_id`、`mentor_name`、`mentor_comment` 用于记录导师指导信息
    - 导师评价可能来自教师端的指导功能

## 业务说明

### 项目生命周期

1. **创建阶段**: 学生在学生端发布创业项目，初始状态为 `0`（待审核）
2. **审核阶段**: 学校端查看项目详情，决定通过或驳回
   - 通过：项目进入孵化阶段，状态变为 `1`
   - 驳回：项目被驳回，状态变为 `2`，学生可以修改后重新提交
3. **孵化阶段**: 项目正在孵化中，教师可以提供指导意见
4. **落地阶段**: 项目成功落地，状态变为 `3`

### 为什么合并项目管理和项目评审？

1. **避免重复请求**: 两个页面都需要请求项目列表，合并后只需一次请求
2. **操作便捷**: 在同一个页面可以查看所有项目并进行审核操作
3. **状态筛选**: 通过状态筛选可以快速找到待审核的项目
4. **减少维护成本**: 一个页面比两个页面更容易维护和扩展

### 扩展功能建议

如果后续需要扩展功能，可以考虑：

1. **批量审核**: 支持批量通过或驳回多个项目
2. **导出功能**: 导出项目列表为Excel文件
3. **统计分析**: 统计各领域项目数量、审核通过率等
4. **项目评分**: 对项目进行评分和排名
5. **项目跟踪**: 记录项目的进展和里程碑
