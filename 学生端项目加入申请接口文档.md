# 学生端项目加入申请接口文档

## 数据库表设计

### 1. 项目加入申请表 (biz_project_application)

```sql
CREATE TABLE public.biz_project_application (
    id varchar(255) NOT NULL,
    project_id varchar(255) NOT NULL,
    applicant_id varchar(255) NOT NULL,
    applicant_name varchar(255) NOT NULL,
    applicant_avatar varchar(500) NULL,
    applicant_school varchar(255) NULL,
    applicant_major varchar(255) NULL,
    application_reason text NULL,
    skills text NULL,
    status varchar(20) NOT NULL DEFAULT 'PENDING',
    reply_message varchar(500) NULL,
    reply_time timestamp NULL,
    create_time timestamp NULL DEFAULT CURRENT_TIMESTAMP,
    update_time timestamp NULL DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT biz_project_application_pkey PRIMARY KEY (id),
    CONSTRAINT uk_project_applicant UNIQUE (project_id, applicant_id)
);

-- 索引
CREATE INDEX idx_application_project ON public.biz_project_application USING btree (project_id);
CREATE INDEX idx_application_applicant ON public.biz_project_application USING btree (applicant_id);
CREATE INDEX idx_application_status ON public.biz_project_application USING btree (status);

-- 状态说明
-- PENDING: 待审核
-- APPROVED: 已通过
-- REJECTED: 已拒绝
-- CANCELLED: 已取消
```

### 字段说明
| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | varchar(255) | 申请ID（主键） |
| project_id | varchar(255) | 项目ID |
| applicant_id | varchar(255) | 申请人ID（学生ID） |
| applicant_name | varchar(255) | 申请人姓名 |
| applicant_avatar | varchar(500) | 申请人头像 |
| applicant_school | varchar(255) | 申请人学校 |
| applicant_major | varchar(255) | 申请人专业 |
| application_reason | text | 申请理由 |
| skills | text | 个人技能 |
| status | varchar(20) | 申请状态 |
| reply_message | varchar(500) | 回复消息 |
| reply_time | timestamp | 回复时间 |
| create_time | timestamp | 创建时间 |
| update_time | timestamp | 更新时间 |

## 接口设计

### 1. 获取项目详情（学生端）

**接口路径**: `/api/mobile/project/{id}`  
**请求方式**: `GET`  
**接口描述**: 获取项目详细信息，包含当前用户的申请状态

#### 请求参数
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | String | 是 | 项目ID（路径参数） |

#### 请求头
```
Authorization: Bearer {token}
```

#### 响应数据
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "projectId": "1",
    "title": "智能校园管理系统",
    "logo": "https://example.com/logo.jpg",
    "slogan": "让校园管理更智能",
    "description": "基于物联网和人工智能技术...",
    "schoolName": "四川大学",
    "teamSize": 5,
    "leaderName": "张三",
    "leaderId": "101",
    "mentorName": "王老师",
    "status": "孵化中",
    "needs": "寻找前端开发工程师1名，要求熟悉Vue3...",
    "tags": ["物联网", "人工智能", "智慧校园"],
    "isCollected": true,
    "isOwner": false,
    "applicationStatus": "PENDING"
  }
}
```

#### 新增字段说明
| 字段名 | 类型 | 说明 |
|--------|------|------|
| isOwner | Boolean | 是否是项目负责人 |
| applicationStatus | String | 申请状态：null=未申请，PENDING=待审核，APPROVED=已通过，REJECTED=已拒绝 |

### 2. 申请加入项目

**接口路径**: `/api/mobile/project/apply`  
**请求方式**: `POST`  
**接口描述**: 学生申请加入项目

#### 请求参数
```json
{
  "projectId": "1",
  "applicationReason": "我对这个项目非常感兴趣，我有3年的前端开发经验...",
  "skills": "Vue3, TypeScript, Node.js, 微信小程序开发"
}
```

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| projectId | String | 是 | 项目ID |
| applicationReason | String | 是 | 申请理由（至少20字） |
| skills | String | 否 | 个人技能 |

#### 请求头
```
Authorization: Bearer {token}
```

#### 响应数据
```json
{
  "code": 200,
  "message": "申请已提交，请等待项目负责人审核",
  "data": {
    "applicationId": "app_123456",
    "status": "PENDING"
  }
}
```

#### 业务规则
1. 不能申请自己的项目（isOwner = true）
2. 同一个项目只能申请一次（通过唯一索引保证）
3. 申请理由至少20个字
4. 申请成功后，项目负责人会收到系统消息通知

#### 错误码
| 错误码 | 说明 |
|--------|------|
| 400 | 参数错误（申请理由太短等） |
| 401 | 未登录 |
| 403 | 不能申请自己的项目 |
| 409 | 已经申请过该项目 |
| 404 | 项目不存在 |
| 500 | 服务器错误 |

### 6. 处理申请（通过/拒绝）

**接口路径**: `/api/mobile/project/apply/{applicationId}/handle`  
**请求方式**: `PUT`  
**接口描述**: 项目负责人处理申请

#### 请求参数
```json
{
  "action": "APPROVE",
  "replyMessage": "欢迎加入我们的团队！"
}
```

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| action | String | 是 | 操作：APPROVE=通过，REJECT=拒绝 |
| replyMessage | String | 否 | 回复消息 |

#### 响应数据
```json
{
  "code": 200,
  "message": "处理成功",
  "data": null
}
```

#### 业务规则
1. 只有项目负责人可以处理申请
2. 通过申请后，申请人会收到系统消息通知
3. 拒绝申请时，建议填写拒绝理由

## 数据库查询示例

### 1. 查询项目详情时获取申请状态
```sql
SELECT 
  p.*,
  s.name as school_name,
  u.nickname as leader_name,
  t.name as mentor_name,
  CASE WHEN p.user_id = #{currentUserId} THEN true ELSE false END as is_owner,
  pa.status as application_status
FROM biz_project p
LEFT JOIN sys_school s ON p.school_id = s.id
LEFT JOIN sys_user u ON p.user_id = u.id
LEFT JOIN biz_teacher t ON p.mentor_id = t.teacher_id
LEFT JOIN biz_project_application pa ON pa.project_id = p.project_id 
  AND pa.applicant_id = #{currentUserId}
WHERE p.project_id = #{projectId}
```

### 2. 创建申请记录
```sql
INSERT INTO biz_project_application (
  id, project_id, applicant_id, applicant_name, applicant_avatar,
  applicant_school, applicant_major, application_reason, skills, status
) VALUES (
  #{id}, #{projectId}, #{applicantId}, #{applicantName}, #{applicantAvatar},
  #{applicantSchool}, #{applicantMajor}, #{applicationReason}, #{skills}, 'PENDING'
)
```

### 3. 查询我的申请记录
```sql
SELECT 
  pa.*,
  p.project_name,
  p.logo as project_logo,
  u.nickname as leader_name
FROM biz_project_application pa
LEFT JOIN biz_project p ON pa.project_id = p.project_id
LEFT JOIN sys_user u ON p.user_id = u.id
WHERE pa.applicant_id = #{currentUserId}
  AND (#{status} IS NULL OR pa.status = #{status})
ORDER BY pa.create_time DESC
LIMIT #{offset}, #{pageSize}
```

### 4. 查询项目的申请列表
```sql
SELECT 
  pa.*,
  s.school_name as applicant_school,
  st.major_name as applicant_major
FROM biz_project_application pa
LEFT JOIN biz_student st ON pa.applicant_id = st.student_id
LEFT JOIN sys_school s ON st.school_id = s.id
WHERE pa.project_id = #{projectId}
  AND (#{status} IS NULL OR pa.status = #{status})
ORDER BY pa.create_time DESC
LIMIT #{offset}, #{pageSize}
```

### 5. 处理申请
```sql
UPDATE biz_project_application
SET 
  status = #{status},
  reply_message = #{replyMessage},
  reply_time = CURRENT_TIMESTAMP,
  update_time = CURRENT_TIMESTAMP
WHERE id = #{applicationId}
  AND project_id IN (
    SELECT project_id FROM biz_project WHERE user_id = #{currentUserId}
  )
```

## 前端实现要点

### 1. 项目详情页按钮状态
```typescript
// 根据 isOwner 和 applicationStatus 决定按钮状态
const buttonConfig = computed(() => {
  if (detail.value.isOwner) {
    return { text: '这是我的项目', disabled: true, color: '#999' }
  }
  
  switch (detail.value.applicationStatus) {
    case 'PENDING':
      return { text: '申请审核中', disabled: true, color: '#ff9800' }
    case 'APPROVED':
      return { text: '已加入项目', disabled: true, color: '#4caf50' }
    case 'REJECTED':
      return { text: '申请未通过', disabled: true, color: '#f44336' }
    default:
      return { text: '申请加入项目', disabled: false, color: '#7c3aed' }
  }
})
```

## 注意事项

1. **权限控制**：
   - 只有学生角色可以申请加入项目
   - 只有项目负责人可以查看和处理申请
   - 不能申请自己的项目

2. **数据一致性**：
   - 使用唯一索引防止重复申请
   - 申请状态变更时同步发送系统消息

3. **用户体验**：
   - 申请理由至少20字，确保申请质量
   - 按钮状态根据申请状态动态变化
   - 提供取消申请功能（仅待审核状态）

4. **性能优化**：
   - 申请列表支持分页
   - 添加必要的索引提升查询性能
