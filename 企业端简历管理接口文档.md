# 企业端简历管理接口文档

## 接口概述

企业端简历管理模块提供查看学生投递简历、安排面试、拒绝简历等功能。

---

## 1. 获取投递简历列表（分页）

### 接口信息
- **接口路径**: `GET /api/company/delivery/list`
- **接口描述**: 分页查询学生投递的简历列表，支持按职位、学生姓名、投递时间筛选
- **权限要求**: 企业HR角色

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| pageNum | number | 是 | 页码，从1开始 |
| pageSize | number | 是 | 每页条数 |
| jobId | string | 否 | 职位ID |
| studentName | string | 否 | 学生姓名（模糊查询） |
| startDate | string | 否 | 投递开始日期 YYYY-MM-DD |
| endDate | string | 否 | 投递结束日期 YYYY-MM-DD |

### 响应数据

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "data": [
      {
        "id": "delivery_123456",
        "studentId": "student_001",
        "studentName": "张三",
        "studentPhone": "13800138000",
        "email": "zhangsan@example.com",
        "jobId": "job_123",
        "jobName": "Java开发工程师",
        "resumeId": "resume_001",
        "education": "bachelor",
        "school": "四川大学",
        "major": "计算机科学与技术",
        "graduationYear": "2024",
        "deliveryTime": "2026-02-10 14:30:00",
        "gender": 1
      }
    ],
    "total": 50
  }
}
```

### SQL 实现示例

```sql
SELECT 
  d.id,
  d.student_id AS "studentId",
  s.name AS "studentName",
  s.phone AS "studentPhone",
  s.email,
  d.job_id AS "jobId",
  j.job_name AS "jobName",
  d.resume_id AS "resumeId",
  r.education,
  r.school,
  r.major,
  r.graduation_year AS "graduationYear",
  d.create_time AS "deliveryTime",
  s.gender
FROM biz_job_delivery d
LEFT JOIN biz_student s ON d.student_id = s.student_id
LEFT JOIN biz_job j ON d.job_id = j.job_id
LEFT JOIN biz_resume r ON d.resume_id = r.resume_id
WHERE d.company_id = #{companyId}  -- 只查询自己公司的投递
  AND d.status = 'DELIVERED'  -- 只查询待处理的简历
  AND (#{jobId} IS NULL OR d.job_id = #{jobId})
  AND (#{studentName} IS NULL OR s.name LIKE CONCAT('%', #{studentName}, '%'))
  AND (#{startDate} IS NULL OR DATE(d.create_time) >= #{startDate})
  AND (#{endDate} IS NULL OR DATE(d.create_time) <= #{endDate})
ORDER BY d.create_time DESC
LIMIT #{pageSize} OFFSET (#{pageNum} - 1) * #{pageSize};

-- 总数查询
SELECT COUNT(*) 
FROM biz_job_delivery d
LEFT JOIN biz_student s ON d.student_id = s.student_id
WHERE d.company_id = #{companyId}
  AND d.status = 'DELIVERED'
  AND (#{jobId} IS NULL OR d.job_id = #{jobId})
  AND (#{studentName} IS NULL OR s.name LIKE CONCAT('%', #{studentName}, '%'))
  AND (#{startDate} IS NULL OR DATE(d.create_time) >= #{startDate})
  AND (#{endDate} IS NULL OR DATE(d.create_time) <= #{endDate});
```

### 业务规则

1. **权限控制**: 只能查询自己公司的投递记录（通过company_id过滤）
2. **状态过滤**: 只显示状态为 `DELIVERED`（待处理）的简历
3. **排序规则**: 按投递时间降序排列

## 3. 安排面试

### 接口信息
- **接口路径**: `POST /api/company/interview/arrange`
- **接口描述**: 为投递的学生安排面试，并发送通知
- **权限要求**: 企业HR角色

### 请求体

```json
{
  "deliveryId": "delivery_123456",
  "interviewTime": "2026-02-15 14:00:00",
  "duration": 60,
  "type": "ONSITE",
  "location": "成都市高新区天府软件园A座10楼",
  "notes": "请携带身份证和学生证"
}
```

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| deliveryId | string | 是 | 投递记录ID |
| interviewTime | string | 是 | 面试时间 YYYY-MM-DD HH:mm:ss |
| duration | number | 是 | 面试时长（分钟） |
| type | string | 是 | 面试方式：ONSITE=现场/VIDEO=视频/PHONE=电话 |
| location | string | 是 | 面试地点或视频链接 |
| notes | string | 否 | 备注信息 |

### 响应数据

```json
{
  "code": 200,
  "msg": "面试安排成功",
  "data": {
    "interviewId": "interview_123456"
  }
}
```

### SQL 实现示例

```sql
-- 1. 插入面试记录
INSERT INTO biz_interview (
  id,
  delivery_id,
  company_id,
  student_id,
  job_id,
  hr_id,
  interview_time,
  duration,
  type,
  location,
  notes,
  status,
  create_time
) VALUES (
  #{interviewId},
  #{deliveryId},
  #{companyId},
  (SELECT student_id FROM biz_job_delivery WHERE id = #{deliveryId}),
  (SELECT job_id FROM biz_job_delivery WHERE id = #{deliveryId}),
  #{hrId},
  #{interviewTime},
  #{duration},
  #{type},
  #{location},
  #{notes},
  'SCHEDULED',  -- 状态：已安排
  CURRENT_TIMESTAMP
);

-- 2. 更新投递状态
UPDATE biz_job_delivery
SET 
  status = 'INTERVIEW',  -- 状态变更为面试中
  update_time = CURRENT_TIMESTAMP
WHERE id = #{deliveryId};

-- 3. 发送消息通知学生
INSERT INTO sys_message (
  id,
  receiver_id,
  title,
  content,
  type,
  is_read,
  ref_id,
  create_time
) VALUES (
  #{messageId},
  (SELECT student_id FROM biz_job_delivery WHERE id = #{deliveryId}),
  '面试通知',
  CONCAT('您投递的职位已安排面试，面试时间：', #{interviewTime}),
  'INTERVIEW',
  0,
  #{interviewId},
  CURRENT_TIMESTAMP
);
```

### 业务规则

1. **权限验证**: 
   - 只能为投递给自己公司的简历安排面试
   - 验证投递状态必须是 `DELIVERED`

2. **状态变更**:
   - 投递状态从 `DELIVERED` 变为 `INTERVIEW`
   - 面试状态初始为 `SCHEDULED`（已安排）

3. **消息通知**:
   - 安排面试后自动发送消息通知学生
   - 消息类型为 `INTERVIEW`

4. **时间验证**:
   - 面试时间必须晚于当前时间
   - 面试时长建议15-240分钟

## 4. 拒绝简历

### 接口信息
- **接口路径**: `POST /api/company/delivery/reject`
- **接口描述**: 拒绝学生投递的简历
- **权限要求**: 企业HR角色

### 请求体

```json
{
  "deliveryId": "delivery_123456",
  "reason": "岗位要求不匹配"
}
```

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| deliveryId | string | 是 | 投递记录ID |
| reason | string | 否 | 拒绝原因 |

### 响应数据

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": null
}
```

### SQL 实现示例

```sql
-- 1. 更新投递状态
UPDATE biz_job_delivery
SET 
  status = 'REJECTED',  -- 状态变更为已拒绝
  reject_reason = #{reason},
  update_time = CURRENT_TIMESTAMP
WHERE id = #{deliveryId}
  AND company_id = #{companyId}  -- 确保只能操作自己公司的投递
  AND status = 'DELIVERED';  -- 只能拒绝待处理的简历

-- 2. 发送消息通知学生（可选）
INSERT INTO sys_message (
  id,
  receiver_id,
  title,
  content,
  type,
  is_read,
  ref_id,
  create_time
) VALUES (
  #{messageId},
  (SELECT student_id FROM biz_job_delivery WHERE id = #{deliveryId}),
  '投递结果通知',
  '很遗憾，您投递的职位暂不合适',
  'DELIVERY',
  0,
  #{deliveryId},
  CURRENT_TIMESTAMP
);
```

### 业务规则

1. **权限验证**: 只能拒绝投递给自己公司的简历
2. **状态限制**: 只能拒绝状态为 `DELIVERED` 的简历
3. **消息通知**: 拒绝后可选择是否通知学生

```json
{

    },
```

```sql

  job_id AS "id",
```

## 数据库表结构说明

### biz_job_delivery（投递记录表）

```sql
CREATE TABLE biz_job_delivery (
  id VARCHAR(64) PRIMARY KEY,
  student_id VARCHAR(64) NOT NULL,
  job_id VARCHAR(64) NOT NULL,
  company_id VARCHAR(64) NOT NULL,
  resume_id VARCHAR(64) NOT NULL,
  status VARCHAR(20) NOT NULL,  -- DELIVERED/INTERVIEW/OFFER/REJECTED
  reject_reason TEXT,
  create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  update_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_delivery_company ON biz_job_delivery(company_id, status);
CREATE INDEX idx_delivery_student ON biz_job_delivery(student_id);
CREATE INDEX idx_delivery_job ON biz_job_delivery(job_id);
```

**status 字段说明**：
- `DELIVERED`: 已投递（待处理）
- `INTERVIEW`: 面试中
- `OFFER`: 已录用
- `REJECTED`: 已拒绝

### biz_interview（面试记录表）

```sql
CREATE TABLE biz_interview (
  id VARCHAR(64) PRIMARY KEY,
  delivery_id VARCHAR(64) NOT NULL,
  company_id VARCHAR(64) NOT NULL,
  student_id VARCHAR(64) NOT NULL,
  job_id VARCHAR(64) NOT NULL,
  hr_id VARCHAR(64) NOT NULL,
  interview_time TIMESTAMP NOT NULL,
  duration INT NOT NULL,  -- 面试时长（分钟）
  type VARCHAR(20) NOT NULL,  -- ONSITE/VIDEO/PHONE
  location TEXT,
  notes TEXT,
  status VARCHAR(20) NOT NULL,  -- SCHEDULED/COMPLETED/CANCELLED
  interview_score INT,  -- 面试评分（1-5）
  interview_comment TEXT,  -- 面试评价
  create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  update_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_interview_company ON biz_interview(company_id, interview_time);
CREATE INDEX idx_interview_student ON biz_interview(student_id);
```

**status 字段说明**：
- `SCHEDULED`: 已安排
- `COMPLETED`: 已完成
- `CANCELLED`: 已取消

## 业务流程说明

### 简历处理流程

```
学生投递简历 → status=DELIVERED(待处理) → HR查看简历 → 
  ├─ 安排面试 → status=INTERVIEW → 面试完成 → 
  │   ├─ 发放Offer → status=OFFER
  │   └─ 不合适 → status=REJECTED
  └─ 直接拒绝 → status=REJECTED
```

### 权限控制

1. **HR身份验证**:
   ```sql
   SELECT hr_id, company_id, name
   FROM biz_hr
   WHERE user_id = #{userId} AND status = 1;
   ```

2. **操作权限**:
   - 查看简历：只能查看投递给自己公司的简历
   - 安排面试：只能为投递给自己公司的简历安排面试
   - 拒绝简历：只能拒绝投递给自己公司的简历

## 注意事项

1. **消息通知**:
   - 安排面试后需要发送消息通知学生
   - 拒绝简历可选择是否通知学生

2. **时间处理**:
   - 面试时间必须晚于当前时间
   - 日期范围查询需要处理时区问题

3. **数据权限**:
   - 所有操作都需要验证company_id
   - 确保HR只能操作自己公司的数据

4. **状态流转**:
   - 投递状态只能单向流转，不能回退
   - 状态变更需要记录操作日志

5. **简历隐私**:
   - 只有投递给公司的简历才能查看
   - 学生可以设置简历隐私选项

## 6. 获取面试列表（分页）

### 接口信息
- **接口路径**: `GET /api/company/interview/list`
- **接口描述**: 分页查询面试安排列表，支持按学生姓名、面试状态、面试时间筛选
- **权限要求**: 企业HR角色

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| pageNum | number | 是 | 页码，从1开始 |
| pageSize | number | 是 | 每页条数 |
| studentName | string | 否 | 学生姓名（模糊查询） |
| status | string | 否 | 面试状态：SCHEDULED=待面试/COMPLETED=已完成/CANCELLED=已取消 |
| startDate | string | 否 | 面试开始日期 YYYY-MM-DD |
| endDate | string | 否 | 面试结束日期 YYYY-MM-DD |

### 响应数据

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "data": [
      {
        "id": "interview_123456",
        "deliveryId": "delivery_001",
        "studentId": "student_001",
        "studentName": "张三",
        "studentPhone": "13800138000",
        "jobId": "job_123",
        "jobName": "Java开发工程师",
        "interviewTime": "2026-02-15 14:00:00",
        "duration": 60,
        "type": "ONSITE",
        "location": "成都市高新区天府软件园A座10楼",
        "notes": "请携带身份证和学生证",
        "status": "SCHEDULED",
        "interviewScore": null,
        "interviewComment": null
      }
    ],
    "total": 20
  }
}
```

### SQL 实现示例

```sql
SELECT 
  i.id,
  i.delivery_id AS "deliveryId",
  i.student_id AS "studentId",
  s.name AS "studentName",
  s.phone AS "studentPhone",
  i.job_id AS "jobId",
  j.job_name AS "jobName",
  i.interview_time AS "interviewTime",
  i.duration,
  i.type,
  i.location,
  i.notes,
  i.status,
  i.interview_score AS "interviewScore",
  i.interview_comment AS "interviewComment"
FROM biz_interview i
LEFT JOIN biz_student s ON i.student_id = s.student_id
LEFT JOIN biz_job j ON i.job_id = j.job_id
WHERE i.company_id = #{companyId}
  AND (#{studentName} IS NULL OR s.name LIKE CONCAT('%', #{studentName}, '%'))
  AND (#{status} IS NULL OR i.status = #{status})
  AND (#{startDate} IS NULL OR DATE(i.interview_time) >= #{startDate})
  AND (#{endDate} IS NULL OR DATE(i.interview_time) <= #{endDate})
ORDER BY i.interview_time DESC
LIMIT #{pageSize} OFFSET (#{pageNum} - 1) * #{pageSize};
```

## 7. 完成面试

### 接口信息
- **接口路径**: `POST /api/company/interview/complete`
- **接口描述**: 标记面试为已完成，并记录面试结果和评价
- **权限要求**: 企业HR角色

### 请求体

```json
{
  "interviewId": "interview_123456",
  "result": "PASS",
  "score": 5,
  "comment": "表现优秀，技术能力强，沟通能力好"
}
```

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| interviewId | string | 是 | 面试ID |
| result | string | 是 | 面试结果：PASS=通过/FAIL=不通过 |
| score | number | 是 | 面试评分（1-5分） |
| comment | string | 是 | 面试评价 |

### 响应数据

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": null
}
```

### SQL 实现示例

```sql
-- 1. 更新面试记录
UPDATE biz_interview
SET 
  status = 'COMPLETED',
  interview_score = #{score},
  interview_comment = #{comment},
  update_time = CURRENT_TIMESTAMP
WHERE id = #{interviewId}
  AND company_id = #{companyId};

-- 2. 根据面试结果更新投递状态
UPDATE biz_job_delivery
SET 
  status = CASE 
    WHEN #{result} = 'PASS' THEN 'OFFER'  -- 通过则发放Offer
    WHEN #{result} = 'FAIL' THEN 'REJECTED'  -- 不通过则拒绝
  END,
  update_time = CURRENT_TIMESTAMP
WHERE id = (SELECT delivery_id FROM biz_interview WHERE id = #{interviewId});

-- 3. 发送消息通知学生
INSERT INTO sys_message (
  id,
  receiver_id,
  title,
  content,
  type,
  is_read,
  ref_id,
  create_time
) VALUES (
  #{messageId},
  (SELECT student_id FROM biz_interview WHERE id = #{interviewId}),
  '面试结果通知',
  CASE 
    WHEN #{result} = 'PASS' THEN '恭喜您，面试通过！'
    WHEN #{result} = 'FAIL' THEN '很遗憾，本次面试未通过'
  END,
  'INTERVIEW',
  0,
  #{interviewId},
  CURRENT_TIMESTAMP
);
```

### 业务规则

1. **状态变更**:
   - 面试状态从 `SCHEDULED` 变为 `COMPLETED`
   - 投递状态根据面试结果变更：
     - `PASS` → `OFFER`（已录用）
     - `FAIL` → `REJECTED`（已拒绝）

2. **消息通知**: 完成面试后自动发送消息通知学生

## 10. 获取人才库列表（分页）

### 接口信息
- **接口路径**: `GET /api/company/talent/list`
- **接口描述**: 分页查询已录用和已拒绝的人才列表
- **权限要求**: 企业HR角色

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| pageNum | number | 是 | 页码，从1开始 |
| pageSize | number | 是 | 每页条数 |
| studentName | string | 否 | 学生姓名（模糊查询） |
| status | string | 否 | 状态：OFFER=已录用/REJECTED=已拒绝 |
| jobId | string | 否 | 职位ID |

### 响应数据

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "data": [
      {
        "id": "delivery_123456",
        "studentId": "student_001",
        "studentName": "张三",
        "studentPhone": "13800138000",
        "email": "zhangsan@example.com",
        "jobId": "job_123",
        "jobName": "Java开发工程师",
        "resumeId": "resume_001",
        "school": "四川大学",
        "major": "计算机科学与技术",
        "education": "bachelor",
        "graduationYear": "2024",
        "status": "OFFER",
        "updateTime": "2026-02-10 16:30:00",
        "gender": 1
      }
    ],
    "total": 30
  }
}
```

### SQL 实现示例

```sql
SELECT 
  d.id,
  d.student_id AS "studentId",
  s.name AS "studentName",
  s.phone AS "studentPhone",
  s.email,
  d.job_id AS "jobId",
  j.job_name AS "jobName",
  d.resume_id AS "resumeId",
  r.school,
  r.major,
  r.education,
  r.graduation_year AS "graduationYear",
  d.status,
  d.update_time AS "updateTime",
  s.gender
FROM biz_job_delivery d
LEFT JOIN biz_student s ON d.student_id = s.student_id
LEFT JOIN biz_job j ON d.job_id = j.job_id
LEFT JOIN biz_resume r ON d.resume_id = r.resume_id
WHERE d.company_id = #{companyId}
  AND d.status IN ('OFFER', 'REJECTED')
  AND (#{studentName} IS NULL OR s.name LIKE CONCAT('%', #{studentName}, '%'))
  AND (#{status} IS NULL OR d.status = #{status})
  AND (#{jobId} IS NULL OR d.job_id = #{jobId})
ORDER BY d.update_time DESC
LIMIT #{pageSize} OFFSET (#{pageNum} - 1) * #{pageSize};
```

## 完整业务流程图

```
学生投递简历
    ↓
status = DELIVERED (待处理)
    ↓
HR查看简历 → 
    ├─ 直接拒绝 → status = REJECTED (已拒绝) → 归档到人才库
    │                                           ↓
    │                                    可重新发放Offer → status = OFFER
    └─ 安排面试 → status = INTERVIEW (面试中)
                      ↓
                  面试进行中
                      ↓
                  ├─ 取消面试 → status = DELIVERED (回退到待处理)
                  └─ 完成面试 → status = COMPLETED
                                  ↓
                              面试结果评估
                                  ↓
                              ├─ 通过 → status = OFFER (已录用) → 归档到人才库
                              └─ 不通过 → status = REJECTED (已拒绝) → 归档到人才库
```
