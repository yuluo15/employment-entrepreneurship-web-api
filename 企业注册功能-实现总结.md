# 企业注册功能 - 实现总结

## 功能概述

将企业注册流程从手机号+短信验证改为邮箱+邮箱验证码方式，提供更简单可靠的注册体验。

---

## 前端实现

### 修改的文件

1. **src/views/register/index.vue** - 注册页面
2. **src/api/auth.ts** - API接口文件

### 主要变更

#### 1. 表单字段调整

**原来**:
- 管理员账号：手机号（11位）
- 无验证码字段

**现在**:
- 管理员邮箱：邮箱格式
- 邮箱验证码：6位数字
- 添加"发送验证码"按钮

#### 2. 验证规则更新

```typescript
// 新增邮箱格式验证
const validateEmail = (rule: any, value: any, callback: any) => {
  if (value === '') {
    callback(new Error('请输入邮箱'))
  } else if (!/^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(value)) {
    callback(new Error('邮箱格式不正确'))
  } else {
    callback()
  }
}

// 更新验证规则
adminAccount: [{ validator: validateEmail, trigger: 'blur' }]
emailCode: [
  { required: true, message: '请输入验证码', trigger: 'blur' }, 
  { len: 6, message: '验证码为6位', trigger: 'blur' }
]
```

#### 3. 发送验证码功能

```typescript
// 倒计时状态
const countdown = ref(0)

// 发送验证码
const sendEmailCode = async () => {
  // 1. 验证邮箱格式
  if (!form.adminAccount) {
    ElMessage.warning('请先输入邮箱')
    return
  }
  if (!/^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(form.adminAccount)) {
    ElMessage.warning('邮箱格式不正确')
    return
  }

  // 2. 调用API发送验证码
  try {
    await sendEmailCodeApi(form.adminAccount)
    ElMessage.success('验证码已发送，请查收邮件')
    
    // 3. 开始60秒倒计时
    countdown.value = 60
    const timer = setInterval(() => {
      countdown.value--
      if (countdown.value <= 0) {
        clearInterval(timer)
      }
    }, 1000)
  } catch (error) {
    console.error('发送验证码失败', error)
  }
}
```

## 后端实现要点

### 需要实现的接口

#### 1. 发送邮箱验证码

**接口**: `POST /api/sendEmail/code`

**功能**:
- 验证邮箱格式
- 检查邮箱是否已注册
- 生成6位随机验证码
- 存储到Redis（5分钟有效）
- 发送邮件

**Redis Key**: `email:code:{email}`

#### 2. 企业注册

**接口**: `POST /api/company/register`

**功能**:
- 验证邮箱验证码（从Redis读取）
- 检查邮箱和信用代码是否重复
- 创建企业记录（sys_company，status=0待审核）
- 创建HR账号（sys_user，role=ROLE_COMPANY）
- 创建HR信息（biz_hr）
- 删除Redis验证码
- 使用事务确保数据一致性

### 数据库表

#### sys_company
- 企业信息
- status: 0=待审核, 1=已通过, 2=已驳回

#### sys_user
- 用户账号
- login_name: 邮箱（唯一）
- role: ROLE_COMPANY
- company_id: 关联企业

#### biz_hr
- HR信息
- 关联user_id和company_id

## 注册流程

```
1. 用户输入邮箱
   ↓
2. 点击"发送验证码"
   ↓
3. 后端发送6位验证码到邮箱（存Redis，5分钟有效）
   ↓
4. 前端开始60秒倒计时
   ↓
5. 用户填写完整注册信息（包括验证码）
   ↓
6. 点击"提交审核"
   ↓
7. 后端验证验证码
   ↓
8. 创建企业、用户、HR记录（事务）
   ↓
9. 企业状态=待审核
   ↓
10. 显示成功页面
   ↓
11. 管理员审核通过后，企业才能登录
```

## 与原方案的对比

| 项目 | 原方案（手机号） | 新方案（邮箱） |
|------|----------------|---------------|
| 登录账号 | 手机号 | 邮箱 |
| 验证方式 | 短信验证码 | 邮箱验证码 |
| 验证码位数 | 6位 | 6位 |
| 验证码有效期 | 5分钟 | 5分钟 |
| 倒计时 | 无 | 60秒 |
| 成本 | 需要短信服务（收费） | 邮件服务（免费/低成本） |
| 可靠性 | 依赖短信服务商 | 更稳定 |
| 重复问题 | 手机号可能重复 | 邮箱唯一性更好 |

---

## 优势

1. **成本更低**: 邮件服务免费或低成本，短信服务需要付费
2. **更可靠**: 不依赖短信服务商，邮件送达率更高
3. **唯一性更好**: 邮箱作为企业联系方式更正式，重复概率更低
4. **国际化友好**: 邮箱是国际通用的联系方式
5. **实现简单**: 不需要对接短信服务商API

## 配置要求

### 邮件服务配置

需要在 `application.yml` 中配置：

```yaml
spring:
  mail:
    host: smtp.qq.com
    port: 587
    username: your@qq.com
    password: your-auth-code  # QQ邮箱授权码
    properties:
      mail:
        smtp:
          auth: true
          starttls:
            enable: true
            required: true
```

### Redis配置

```yaml
spring:
  redis:
    host: localhost
    port: 6379
    database: 0
```
