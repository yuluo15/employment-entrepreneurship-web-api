# 岗位审核接口实现说明

## 已实现的接口

### 1. 获取岗位列表
**接口路径**: `GET /api/jobAudit/list`

**请求参数**:
- pageNum (必填): 页码
- pageSize (必填): 每页数量
- jobName (可选): 职位名称(模糊查询)
- companyName (可选): 企业名称(模糊查询)
- audit (可选): 审核状态 (0=待审核, 1=已通过, 2=已驳回)

### 2. 审核岗位
**接口路径**: `POST /api/jobAudit/audit`

**请求体**:
```json
{
  "jobId": "1",
  "audit": 1,
  "reason": "驳回原因(audit=2时必填)"
}
```

**返回示例**:
```json
{
  "message": "success",
  "data": null
}
```

### 3. 下架岗位
**接口路径**: `POST /api/jobAudit/offline/{jobId}`

**返回示例**:
```json
{
  "message": "success",
  "data": null
}
```

### 字段映射关系

| 数据库字段 (JobEntity) | VO字段 | 说明 |
|----------------------|--------|------|
| id | jobId | 岗位ID |
| job_name | jobName | 职位名称 |
| salary_range | salaryRange | 薪资范围(字典值) |
| - | salaryRangeLabel | 薪资范围(字典标签) |
| education | education | 学历要求(字典值) |
| - | educationLabel | 学历要求(字典标签) |
| city | city | 城市 |
| experience | experience | 工作经验 |
| tags | tags | 标签(逗号分隔) |
| description | description | 岗位描述 |
| requirement | requirement | 任职要求 |
| audit | audit | 审核状态 |
| view_count | viewCount | 浏览次数 |
| hr_id | hrId | HR的ID |
| contact_phone | contactPhone | 联系电话 |
| reason | reason | 驳回原因 |
| create_time | createTime | 创建时间 |

### 关联查询
- 通过 `companyId` 关联 `sys_company` 表获取公司信息
- 通过 `hrId` 关联 `biz_hr` 表获取HR信息
- 通过字典表 `sys_dict_data` 获取薪资范围和学历的标签

### 权限控制
所有接口都需要 `ROLE_ADMIN` 或 `ROLE_SCHOOL_ADMIN` 角色权限

### 业务逻辑
1. **列表查询**: 支持按职位名称、公司名称、审核状态筛选，按创建时间倒序排列
2. **审核**: 驳回时必须填写原因，审核状态只能是1(通过)或2(驳回)
3. **下架**: 将岗位状态设置为0(下架)

### 异常处理
- 岗位不存在时抛出 `BusinessException`
- 审核状态不正确时抛出 `BusinessException`
- 驳回时未填写原因抛出 `BusinessException`
- 所有异常由全局异常处理器统一处理并返回友好提示
